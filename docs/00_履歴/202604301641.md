# Integral block error 表示と skill template 修正

## 背景

- `baseline_correction.py` 実行時に `params must be a JSON Schema-like mapping.` が出た。
- 直接原因は Python block 側の `@integral_block(..., params=[...])` が旧形式で、現行 SDK は `params={"type": "object", "properties": {...}}` を要求していたこと。
- 併せて、Integral block 実行エラーが UI 上でコピーしにくく、LLM からも見えにくい問題があった。

## 対応

- Integral block 実行エラーを `<pre><code>` 表示にし、Copy ボタンでコピーできるようにした。
- 実行失敗時、対象 `itg-notes` block 直下に `integral-error` fenced code block として traceback を Markdown へ反映するようにした。
- `C:\Users\shimadzu\Documents\Notes\scripts\ai_blocks\baseline_correction.py` の `params` を現行 JSON Schema-like object 形式へ修正した。
- workspace 展開元を `Notes/` 配下の template truth と整理する方針を確認した。
- `Notes/.codex/skills/implement-integral-block` と `Notes/.claude/skills/implement-integral-block` の skill/reference を現行 params schema 契約へ更新した。
- 削除済みの demo script / test workspace file を skill 参照から外し、`scripts/integral` を source of truth とする説明へ寄せた。

## 確認

- `python C:\Users\shimadzu\Documents\Notes\scripts\ai_blocks\baseline_correction.py` が成功した。
- `npm run build` が成功した。
- skill template の `params schema system` / demo file 参照などの矛盾を確認し、否定例を除いて解消した。
