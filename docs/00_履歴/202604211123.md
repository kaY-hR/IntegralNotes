## 今回まとめたこと

- AI Chat は `Activity Bar` から開く main workspace tab とする
- renderer は chat UI、main process は agent runtime / provider call / tool 実装を持つ
- model transport の第一候補は `Vercel AI Gateway`
- local workspace の検索 / read / write は remote sandbox 前提ではなく、main process 側で local workspace を mount した `bash-tool` runtime を使う
- `bash-tool` だけでは project runtime を使う command 実行を完全には代替しにくいため、別に host command tool を持つ
- repo の `.codex/skills` を skill source として使う
- MCP は `@ai-sdk/mcp` client で generic に受け、remote HTTP 系と local stdio 系を扱う
- `Vercel MCP` は 2026-01-30 時点 docs の client review 制約があるため、first version の必須前提には置かない
