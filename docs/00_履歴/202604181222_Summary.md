## 背景

- 解析の in/out 単位を dataset 固定にしている現状は直感的ではないため、managed file 中心へ再設計したいという相談。
- original data / dataset を基盤概念から外し、slot ごとに `name / extension(s) / format(kind)` を持つ方向で整理する方針を確認。
- provenance は `createdByBlockId: null | BLK-*` と format 管理で代替し、過剰な概念は削る方針。

## 合意した方向性

- cwd 内の通常ファイルを managed file として追跡し、パス変更は hash と再スキャンで補正する。
- block の input/output slot は path ベースにする。
- `.idts` は dataset/bundle を表す拡張子の一つとして残すが、唯一の入出力表現にはしない。
- output が単一ファイルなら、そのファイル自体を managed data として扱う方向で進める。
- Markdown のリンク記法は path のまま維持し、外部 rename/move 検知時に本文側を書き換える。
