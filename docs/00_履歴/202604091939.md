## 背景

- エクスプローラー修正は確認できたが、ズームは `Ctrl + -` のみ効いて `Ctrl + +` は引き続き効かなかった。
- ユーザー指示に従い、Electron / Web の公式資料を見ながらズーム入力の経路を見直した。

## 調査

- Electron の keyboard shortcuts / `before-input-event` 公式資料を確認。
  - main process 側で shortcut を拾う方法自体は正しい。
- ただし `+` 側はキーボード配列差の影響を受けやすく、物理キー寄りの判定だけだと取りこぼす可能性があると判断した。
- Electron のサンプルは renderer 側でも capture で key を拾える前提なので、こちらで実際の `KeyboardEvent.key` を見る方が堅いと判断した。

## 実施内容

- `src/main/main.ts`
  - keyboard shortcut の zoom 処理を外し、`zoom-changed` のみを残した。
  - これにより wheel / gesture 系は main、keyboard 系は renderer に責務分離した。
- `src/main/preload.ts`
  - `webFrame.getZoomLevel()` / `setZoomLevel()` を使う `zoomIn` / `zoomOut` / `resetZoom` API を追加。
- `src/shared/workspace.ts`
  - preload で公開する zoom API の型を追加。
- `src/renderer/App.tsx`
  - capture phase の `keydown` で `Ctrl/Cmd +`, `Ctrl/Cmd -`, `Ctrl/Cmd 0` を処理するよう変更。
  - `+`, `=`, numpad `Add`, および shift 付き `Semicolon` 系も zoom in として扱うようにした。
  - `preventDefault()` と `stopPropagation()` を入れ、Milkdown 側へ流れないようにした。

## 効果

- キーボード配列依存が強い `Ctrl/Cmd + +` を renderer の実キー判定で拾えるようになった。
- `Ctrl/Cmd + -` と `Ctrl/Cmd + 0` も同じ経路に揃えた。
