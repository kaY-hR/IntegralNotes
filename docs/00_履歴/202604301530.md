# 解析 block input 参照の ID 正規化修正

## 背景

- 解析 block の `in:` は仕様上、保存値として managed file / dataset ID を正とする。
- 一方で、LLM や手書きなどで `in:` に workspace path が入った場合、UI 上は managed data として解決できても YAML source は path のまま残る可能性があった。

## 実装

- `itg-notes` block の input 参照を asset catalog で解決し、path から managed file / dataset ID へ正規化する helper を追加した。
- WYSIWYG の解析 block 表示時に、解決可能な `in:` path を即 ID へ書き戻すようにした。
- raw Markdown 編集で path が残っていても、ノート保存直前に `itg-notes` block の `in:` を ID へ正規化するようにした。
- `out:` は仕様どおり実行前 path / 実行後 ID の扱いなので、今回の正規化対象は `in:` のみとした。
