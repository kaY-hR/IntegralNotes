## 背景

- ユーザーから、Milkdown の
  - drag handle (`[+]` / menu handle)
  - slash menu
  - placeholder (`[Please Enter...]`)
  が dialog を貫通して前面に見える、という報告が追加で入った。
- あわせて、chunk viewer 内をクリックしたときに block 全体が選択されて薄灰色になるのが視認性を大きく落としていた。
- 途中で別件として `shimadzu-lc migration` もキュー追加され、`standard-graphs` は一旦削除してよい前提が共有された。

## 実装したこと

### dialog open 中の Milkdown 浮遊 UI 抑制

- `src/renderer/App.tsx`
  - modal dialog が開いているかを `hasBlockingDialog` として集約
  - `body.integral-dialog-open` を付け外しする effect を追加
  - dialog open 時には active element を一旦 blur するよう調整
- `src/renderer/styles.css`
  - `dialog-backdrop` に高い `z-index` を付与
  - dialog open 中は
    - `.milkdown-block-handle`
    - `.milkdown-slash-menu`
    - `.crepe-placeholder::before`
    を非表示化

### chunk viewer の選択グレー化を緩和

- `src/renderer/styles.css`
  - Milkdown/ProseMirror の `.ProseMirror-selectednode` 既定背景を Integral block に対してだけ打ち消す override を追加
  - block 選択そのものは残しつつ、灰背景だけ外す方針にした

### キュー追加

- `docs/Issues.md`
  - `## 6. shimadzu-lc migration を進めたい` を追加
  - `standard-graphs は一旦削除してよい` を明記
  - 優先重みは仮で `6` に設定

## 調査メモ

- upstream の GitHub Discussion `#1907` では、maintainer が `milkdown-slash-menu` は stylesheet override で調整する前提だと案内していた
- ローカルで追うと、Milkdown の `SlashProvider` / `BlockProvider` は浮遊 UI を editor parent へ append する実装になっており、app 側で dialog レイヤと suppress を持つのが自然と判断した
