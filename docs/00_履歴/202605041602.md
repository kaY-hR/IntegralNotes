# Link picker ranking 実装

## 要望

- Link picker の候補順を Relation Graph の近さに基づける。
- 3hop 以内は hop 数が小さい順に優先し、3hop より遠い候補は通常順位へ寄せる。
- 通常 link と embed link で拡張子の優先順位を分ける。
- 拡張子優先順位は決め打ちではなく設定可能にする。

## 実装

- `RelationGraphService` に active path 起点の path 距離を返す `getPathDistances` を追加。
  - node 探索は graph view と同じく `note -> data-note -> file -> folder -> python-callable -> block` の優先順で起点を解決。
  - edge は proximity 用に無向として扱い、指定 hop 以内の path ごとの最短 hop を返す。
  - data-note path も alias として扱い、datafile / datanote の一体運用に寄せた。
- renderer API に `getRelationPathDistances` を追加。
- `MilkdownEditor` の link picker ranking を変更。
  - 文字列 query で候補を絞った後、`graph hop -> link/embed 別 extension priority -> 既存文字列 score -> path` で並べる。
  - active editor の `relativePath` から 3hop 以内の距離だけを取得し、取得できない場合は従来相当の fallback で動く。
- App settings に `linkPickerRanking` を追加。
  - 通常 link の既定優先: `.md, .idts, .csv, .tsv, .json, .py, .txt, .html, .htm`
  - embed link の既定優先: `.png, .jpg, .jpeg, .webp, .gif, .svg, .bmp, .html, .htm, .md`
  - 設定 dialog から comma / whitespace 区切りで編集可能。

## 確認

- `npm run build` 成功。

## 追加調整

- active note 自身は link picker ranking 用の hop 距離から除外した。
  - 候補 list から消すのではなく、graph proximity による top 表示だけを外す。
- folder/file の `contains` edge は ranking 用 hop 計算にも残す方針に戻した。
  - 同じ folder 配下の file は `child -> parent -> child` で 2hop になり得る。
  - Python callable も `note -> block -> callable` で 2hop になり得るため、同 hop の場合は link/embed 別の拡張子優先順位で並ぶ。
- `npm run build` 成功。
