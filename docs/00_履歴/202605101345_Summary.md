# 2026-05-10 13:45 AI commit validation 経路の補強

## 背景

重複 block id / output id を含む Markdown は保存時に拒否できるようになったが、AI Inline Action に試させると、保存できない内容を挿入したまま作業完了扱いになるケースがあった。

## 原因

- `insertMarkdownAtCursor` には Markdown validator を通していたが、`canEditWorkspaceFiles: true` の action では AI が `writeWorkspaceFile` で `.md` を直接保存でき、この経路が validator を通っていなかった。
- `insertMarkdownAtCursor` が validation error を返したあとに AI が成功 commit せず終了した場合、呼び出し側が validation error を通常の失敗理由として surfaced していなかった。

## 変更

- `src/main/aiAgentService.ts`
  - `writeWorkspaceFile` が `.md` file を保存する場合、保存前に `validateMarkdownDocument()` を実行するようにした。
  - validation error の場合は保存せず、`validationStatus: "error"` と error 内容を tool result として返すようにした。
  - tool trace でも `writeWorkspaceFile` validation error を失敗扱いにした。
- `src/main/aiChatService.ts`
  - `insertMarkdownAtCursor` / `writeWorkspaceFile` の validation error が未解消のまま AI が終了した場合、通常完了扱いにせず validation error を throw して popup 側へ返すようにした。
  - Inline Action instructions に `writeWorkspaceFile` の Markdown validation 契約を追加した。
- `docs/10_要求/10_MVP要件.md`
  - AI の `.md` 保存 tool も共通 validator を使う仕様を追記した。
- `docs/30_設計/40_InlineAction.md`
  - `writeWorkspaceFile` と validation error 未解消時の扱いを追記した。
