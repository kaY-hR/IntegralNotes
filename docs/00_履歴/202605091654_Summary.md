# 2026-05-09 16:54 plugin / package / global extension 配置方針

## 背景

plugin system について、Python block、AI skill、runtime plugin、viewer plugin が相互に依存するケースをどう扱うかを整理した。

`.icv` image compare のように、次の要素が一式で揃って初めて意味を持つ機能がある。

- `.icv` を生成する Python block / helper
- `.icv` の使い方や出力判断を AI に教える skill
- `.icv` を表示する viewer plugin
- template / sample

## 方針

Python / skill 側には個別 manifest を持ち込まない。

代わりに、上位の package 実体だけが manifest を持ち、package manager / scanner が中身を見て不足を検知する。

- Python block は従来どおり `.py` と `@integral_block` で成立する
- skill は従来どおり `SKILL.md` folder で成立する
- runtime plugin / viewer plugin は従来どおり `integral-plugin.json` を持つ
- package は「一式配布」「不足検知」「由来管理」のために存在する

dependency resolver のような本格的な依存解決は当面作らない。まずは package manifest に期待構成を書き、scanner が `missing / inactive / version mismatch` などを警告する程度でよい。

## 管理場所の整理

未リリースで利用者がまだいないため、後方互換は不要と判断した。

今後の global extension root は `%LocalAppData%\IntegralNotes` に寄せる方針とする。

```text
%LocalAppData%\IntegralNotes\
  global\
    skills\
    scripts\
  runtime-plugins\
  packages\
```

役割:

- `global/skills`
  - workspace に属さない IntegralNotes global skill
- `global/scripts`
  - workspace へコピーして使う Python script stock
- `runtime-plugins`
  - app が直接読む runtime plugin / viewer plugin の install 先
- `packages`
  - package manifest、install record、構成検査に使う package 実体

## Python の扱い

現状実装では global Python を直接実行できる。

現在の scan / execution 対象:

- `%LocalAppData%\IntegralNotes\analysis-stock`
- `%LocalAppData%\IntegralNotes\plugins`

ただし今後の方針としては、Python は解析の再現性に直結するため、global script は stock / template として扱い、実際に使うときは workspace (`pwd`) へコピーする。workspace 内の `.py` を source of truth とする。

## 現状の問題

現在は管理場所が分散している。

- Python stock: `%LocalAppData%\IntegralNotes\analysis-stock`
- Python vendor-like folder: `%LocalAppData%\IntegralNotes\plugins`
- Skill: `%LocalAppData%\IntegralNotes\skills`
- Runtime plugin: Electron `userData\plugins`
  - packaged: `%AppData%\IntegralNotes\plugins`
  - dev: `%AppData%\IntegralNotes-dev\plugins`

特に `%LocalAppData%\IntegralNotes\plugins` が Python vendor-like folder で、`%AppData%\IntegralNotes[-dev]\plugins` が runtime plugin という点は混乱しやすい。

## 次の設計メモ

- `plugins` という名前は runtime plugin 専用にするか、少なくとも Python vendor folder には使わない
- package 導入時は component 側に manifest を分散させず、package manifest だけが期待構成を持つ
- package scanner は新 path を正として扱う
- 旧 path 互換は不要だが、移行作業中の検出や cleanup 方針は別途決める
- app の設定や履歴は Electron `userData`、拡張資産は `%LocalAppData%\IntegralNotes` に寄せる
