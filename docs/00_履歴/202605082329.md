# 2026-05-08 23:29 AI command environment investigation

## 依頼

- AI が `bash: python: command not available in browser environments. Exclude 'python' from your commands or use the Node.js bundle.` と返す理由を確認したい。
- PowerShell 全般は許可しているはずなのに、なぜ弾かれるのかを知りたい。

## 調査

- 直近履歴と `docs/10_要求/00_プロダクト概要.md` を確認した。
- エラーメッセージ本文は repository 内には存在しなかった。
- `src/main/aiAgentService.ts` では `bash-tool` の `bash` 系 tool と、承認付き real host command の `runShellCommand` が併存している。
- `bash-tool` は browser/sandbox 側の workspace snapshot 実行環境であり、PowerShell 承認とは別経路。
- host PowerShell では `python --version` が成功し、`C:\Users\shimadzu\AppData\Local\Programs\Python\Python310\python.exe` が見えていることを確認した。

## 結論

- 原因は PowerShell 権限ではなく、AI が `runShellCommand` ではなく `bash` tool 内で `python` を実行していること。
- `bash` tool は host PowerShell ではないため、そこで `python` が使えず browser environment error になる。
- 改善するなら prompt/tool description に「Python 実行は必ず `runShellCommand` を使う。`bash` で `python` / `pip` を実行しない」を明示するのが有効。
