## 設計上の判断

- 今回の preview 実装は「JSON を独自 UI っぽく見せる」ための最小構成
- ただし、この方式は sanitize 済み HTML preview を返す前提であり、将来的な「ボタン押下で装置操作や解析を実行する」本格的な interactive UI の土台としては弱い
- 本命は Milkdown の custom node view で独自コンポーネントを editor 内に載せる構成
- JSON は Markdown 上に保持しつつ、表示時は `type` ごとに React component へ変換し、操作は preload 経由 IPC で main process / 実行基盤へ委譲するのが筋
