# 2026-04-21 13:17 AI Chat proxy対応とbash-tool挙動整理

## 対応内容

- `AI Chat` main process 起動時に network proxy を初期化するようにした。
- `HTTP_PROXY` / `HTTPS_PROXY` / `NO_PROXY` に加えて、社内利用想定の `PROXY_HTTP` / `PROXY_HTTPS` / `PROXY_NO` も読むようにした。
- custom proxy env が入っている場合は標準 env 名へも mirror し、`undici` の `EnvHttpProxyAgent` を global dispatcher として設定するようにした。
- これにより `AI Gateway` / direct provider ともに main process の `fetch` 系 request は proxy 経由で送る前提になった。
- `bash-tool` については未実装ではなく、`ToolLoopAgent + createBashTool + just-bash OverlayFs` の配線自体はすでにあることを確認した。
- 一方で user から見て「active file しか見ていない」挙動になりやすかったため、repository / code / file / implementation の質問では active excerpt があっても tool を先に使うよう agent instructions を強めた。
- `bash` runtime が `rg`, `find`, `grep`, `ls`, `head`, `tail`, `sed`, `jq`, `cat` などを使える前提であることを bash instructions に明記した。
