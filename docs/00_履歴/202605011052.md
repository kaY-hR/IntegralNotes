# 2026-05-01 10:52 Playwright MCP worktree registration scripts

## 背景

- Codex の MCP 設定は global なので、固定 path で `codex mcp add` すると worktree を切ったときに登録先がズレる。
- 開発者が worktree 作成時に 1 回だけ登録し、worktree 削除前に解除できる簡単な導線が必要になった。
- stdio MCP では stdout を JSON-RPC protocol 専用にする必要があるため、`npm run` ではなく direct `node` 起動で登録する方針にした。

## 対応

- `scripts/register-codex-playwright-mcp.bat` を追加した。
  - current directory を IntegralNotes worktree root として扱う。
  - 既定 MCP 名は `integralnotes-playwright-<カレントフォルダ名>`。
  - 同名登録があれば一度 remove してから add する。
  - 登録 command は `node <worktree>\src\dev\integral-playwright-mcp.mjs`。
- `scripts/unregister-codex-playwright-mcp.bat` を追加した。
  - register と同じ既定名で `codex mcp remove` する。
  - 任意名を第 1 引数で渡せる。
- `docs/30_設計/95_PlaywrightMCP.md` に worktree ごとの登録/解除手順を追記した。

## 確認

- ユーザー指示により bat の実行確認は未実施。
- 初回実行では bat 内から `codex.cmd` を `call` なしで起動していたため、`codex mcp remove` 後に親 bat へ戻らず、`codex mcp add` まで進まなかった。
- `scripts/register-codex-playwright-mcp.bat` と `scripts/unregister-codex-playwright-mcp.bat` の `codex mcp ...` 呼び出しを `call codex mcp ...` に修正した。
- 修正後に `scripts\register-codex-playwright-mcp.bat` を実行し、global MCP server `integralnotes-playwright-mcp-server-impl` の追加に成功した。
- `codex mcp get integralnotes-playwright-mcp-server-impl` で `enabled: true`、`transport: stdio`、`command: node`、script path が登録されていることを確認した。
- Codex 再起動後、`integralnotes-playwright-mcp-server-impl` の tools が利用可能になった。
- `integral_launch` で IntegralNotes を起動し、`Notes` workspace を開いた。
- `integral_evaluate` 経由で `window.integralNotes.createEntry` / `saveNote` / `readNote` を呼び、`Notes/playwright-mcp-live-202605010202.md` の作成・保存・読み戻しに成功した。
