# 2026-05-03 19:59 Inline Action 後の Python block registry refresh 修正

## 依頼

- @@continue / @@mkpy で挿入された block は IntegralNotes 独自 code-block としては認識されている。
- ただし Python block として解決されず、上位の default block 表示になる。
- Sync やカーソル移動で Python block に変わるため、itg-notes 認識ではなく un: から block definition を解決するタイミングの問題と判断した。

## 対応

- src/renderer/MilkdownEditor.tsx の前回修正で入れた itg-notes 直接 node 化 helper を削除し、通常の Markdown parse + Slice 挿入へ戻した。
- Inline Action 挿入 Markdown に itg-notes fence が含まれる場合、挿入後に window.integralNotes.getIntegralAssetCatalog() を呼ぶようにした。
- 取得した catalog を setIntegralPluginRuntimeCatalog と onIntegralAssetCatalogChanged に反映し、renderer 側の Python block definition registry を更新するようにした。
- catalog 更新後に軽い editor transaction を dispatch し、既に表示された IntegralNotes block が最新 registry で再描画されるようにした。
