# 2026-04-17 14:57

- エクスプローラーの外部 drag/drop と外部 clipboard 貼り付けが依然動かない件を再調査した。
- Electron 35 では `File.path` が削除されており、renderer 側で外部 drop から絶対パスを取るには `webUtils.getPathForFile(file)` を preload 経由で呼ぶ必要があることを確認した。
- 実装を次のように修正した。
  - `src/shared/workspace.ts` に `getPathForFile(file)` API を追加した。
  - `src/main/preload.ts` で `webUtils.getPathForFile(...)` を expose した。
  - `src/renderer/App.tsx` の外部 drop パス解決を `file.path` 依存から `window.integralNotes.getPathForFile(file)` へ置き換えた。
- 併せて、外部 drop / 外部 clipboard paste の切り分け用に renderer console へ debug / error log を追加した。
  - drop 時に `types`, `itemKinds`, `fileCount`, `sourceAbsolutePaths` を出す
  - paste 時に `externalClipboardPaths` を出す
  - 失敗時は `console.error(...)` で例外も出す
- `npm run build` を実行し、修正後に通過を確認した。
