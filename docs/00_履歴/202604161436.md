## 会話概要

- ユーザー要望:
  - data-note を保存すると、最初は `[~~ のノート]` と表示されるのに、保存後は `ORD-xxxxx.md` のようなファイル名へ戻ってしまう
  - 常に `[~~ のノート]` の表示を維持したい

## 原因

- `src/renderer/App.tsx`
  - data-note を開くときだけ `tabNameOverride` で `${displayName} のノート` を設定していた
  - ただし保存後は `window.integralNotes.saveNote()` の戻り値 `savedNote.name` でタブ名を更新していた
- `src/main/workspaceService.ts`
  - markdown 読み込み時の `name` は常に実ファイル名 `ORD-xxxxx.md` / `DS-xxxxx.md` を返していた
  - そのため保存直後に friendly name が失われていた

## 実装内容

- `src/main/dataNote.ts`
  - managed data note の frontmatter から `displayName` を読み、`${displayName} のノート` を返す `resolveManagedDataNoteTabName()` を追加
- `src/main/workspaceService.ts`
  - markdown ファイル読み込み時に managed data note なら `resolveManagedDataNoteTabName()` を優先して `document.name` に設定
  - 通常の markdown は従来どおり basename を返す

## 効果

- data-note は初回オープン時だけでなく、保存後・再読込後も一貫して `${displayName} のノート` 表示になる
- friendly name の解決を renderer の一時 override ではなく main 側の document 読み込みへ寄せたため、表示名の再設定経路が統一された
