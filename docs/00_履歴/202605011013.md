# 2026-05-01 10:13 inline AI Markdown 反映フリーズ対策

## 背景

- `@@` promptless continuation で、popup 内の生成表示は進むが、生成結果を実際の Markdown editor へ反映するタイミングで固まるように見える、という報告があった。
- `??` inline Markdown insertion / `>>` Python block insertion でも同じ可能性があるため、3 trigger 共通の editor 挿入経路を確認した。
- 関連履歴として `202604300931.md`, `202604301130.md`, `202605010118.md` と直近履歴を確認した。

## 対応

- `src/renderer/MilkdownEditor.tsx` の `insertMarkdownAtPosition` を修正した。
- これまでは marker cleanup のための selection dispatch と、Milkdown `insert(markdown)` による挿入 dispatch の 2 段階だった。
- 生成 Markdown を `parserCtx` で parse し、`Slice` / `replaceRange` を使って optional marker cleanup と挿入を 1 つの ProseMirror transaction にまとめた。
- 挿入後の selection は、`TextSelection` の doc 境界 warning を避けるため、挿入 slice 末尾付近に `Selection.near` で移動するようにした。
- 対象経路は `@@` / `??` / `>>` の共通挿入処理。
