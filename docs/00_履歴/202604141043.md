## 会話要約

- `Artifacts / blob / chunk` 系の命名を、`data-catalog / data-note / original data / dataset` ベースへ全面的に寄せる実装を行った。
- 対象は文書、ソースコード、workspace 配下の folder 命名、App 上の表示、サンプル workspace を含む。

## 実装したこと

- docs
  - `docs/10_要求`, `docs/20_アーキテクチャ`, `docs/30_設計` の主要文書で、
    - `artifact` -> `data note`
    - `blob` -> `original data`
    - `chunk` -> `dataset`
    - `chunkId` / `blobId` 例 -> `datasetId` / `originalDataId`
    に更新した。
- shared / main / renderer
  - 型名、API 名、戻り値名、workspace path 定数、metadata file 名を新命名へ更新した。
  - built-in display block の新 block type を `dataset-view` に変更した。
  - 既存ノート互換のため、renderer 側では旧 `chunk-view` も `dataset-view` として解決する互換層を残した。
- UI
  - データ登録、picker、block editor、標準表示まわりの表示文言を `original data` / `dataset` ベースへ更新した。
  - Python script 登録 dialog 内の残っていた `chunk` 系 class 名も `dataset` へ揃えた。
- workspace / sample data
  - migration 実装は維持しつつ、同梱サンプル workspace `Notes/vvv` 自体も
    - `Artifacts` -> `data-catalog`
    - `.blob` -> `.original-data`
    - `chunk` -> `dataset`
    に移行した。
  - 併せて `blob.json` -> `original-data.json`, `chunk.json` -> `dataset.json`,
    `links.json` の key / target, `data note` 内の payload path, `analysis-args.json` の dataset path を更新した。

## 確認結果

- `npm run build` は成功した。

  - legacy migration 用の定数
  - 旧 `chunk-view` 互換
  - 過去の議論履歴 / Issue 記録
  に限定した。
