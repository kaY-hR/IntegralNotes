# 2026-05-08 16:09 Ctrl+S dirty 解除遅延の軽量化

## 依頼

- Markdown 更新後に `Ctrl+S` を押しても dirty 表示が消えるのが遅い。

## 調査

- renderer の `saveNote()` は `window.integralNotes.saveNote()` の完了後に `savedContent` を更新し、dirty を解除している。
- main 側の `WorkspaceService.saveNote()` は保存後に `modify` mutation を発行する。
- 直前の高速化変更後も、managed `.md` の `modify` が Integral 側の full reconcile 判定に入る可能性が残っていた。

## 対応

- `IntegralWorkspaceService.handleWorkspaceMutations()` に modify 専用の軽量処理を追加した。
- direct target の managed file / dataset については、その metadata だけを更新するようにした。
  - managed file は対象 file の hash / visibility を更新。
  - dataset `.idts` は hash / visibility と manifest fields を更新。
- 軽量処理で更新済みの metadata ID は full reconcile 判定から除外した。
- `.md` managed file は data-note を持たないため、保存時に data-note 全件同期へ進まない。

## 確認

- `npm run build` 成功。
