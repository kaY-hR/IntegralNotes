## 目的

inline AI 挿入 (`??`) を AI Chat パネル相当の workspace agent として扱い、tool call 利用、開いている Markdown の context、HTML / 画像確認の指示を強化する。

## 実装内容

- `??` の tool loop 上限を 4 step から 8 step に変更し、通常 chat と同等に複数 tool call を使いやすくした。
- inline Markdown 挿入の system prompt に、workspace tools / `writeWorkspaceFile` / `readWorkspaceImage` / `renderWorkspaceDocument` の利用方針を明示した。
- HTML や画像の内容・見た目が関係する場合は、ファイル名や source text だけで判断せず tool で確認するよう明記した。
- `??` / `>>` の request に、現在開いている Markdown 全体 (`documentMarkdown`) と挿入開始位置 (`insertionPosition`) を追加した。
- prompt には現在の open editor state、挿入位置、カーソル前後 context を渡すようにした。open editor state は未保存変更を含む authoritative context として扱わせる。
- popup 内の tool message 表示を、tool 名・status・input・output summary が見える形にした。
