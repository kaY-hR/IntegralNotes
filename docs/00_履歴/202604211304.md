# 2026-04-21 13:04 AI Chat direct provider fallback

## 対応内容

- `AI Chat` の送信 runtime を `AI Gateway 固定` から `direct provider または AI Gateway` へ切り替えた。
- `anthropic/*` を選択中で `ANTHROPIC_API_KEY` が `process.env` / `.env.local` / `.env` のいずれかにあれば、`@ai-sdk/anthropic` 経由で Anthropic へ直接送るようにした。
- `openai/*` を選択中で `OPENAI_API_KEY` が `process.env` / `.env.local` / `.env` のいずれかにあれば、`@ai-sdk/openai` 経由で OpenAI へ直接送るようにした。
- direct provider が使えない場合のみ、保存済み key または `AI_GATEWAY_API_KEY` / `VERCEL_OIDC_TOKEN` による `AI Gateway` 実行へフォールバックするようにした。
- Anthropic の model ID は gateway 形式から direct provider 形式へ変換するようにした。
- `AI Chat` の settings UI を `Gateway Settings` 表記から `Runtime Settings` 表記へ寄せ、`Runtime Auth` と `AI Gateway API Key (optional)` の表示に修正した。
- `AI Chat` status に `runtimeAuthConfigured` と `implementationMode=direct|gateway|stub` を追加した。
- 設定ダイアログから保存した key の保存先を確認し、これは BYOK ではなく `AI Gateway` 用 saved key として `userData/ai-chat-settings.json` に保持されることを整理した。
- `@ai-sdk/anthropic` と `@ai-sdk/openai` を dependency に追加した。

- `AI Gateway` の saved key は app の `userData/ai-chat-settings.json` に保存され、`safeStorage` が使える環境では暗号化される。
