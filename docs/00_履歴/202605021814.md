# 2026-05-02 18:14 inline preview crash 切り分け

## 依頼

- `@@` inline 挿入後、再び画面が真っ白になった。
- 直前に追加したログが残っているか確認した。

## ログ確認

- ログは `C:\Users\shimadzu\AppData\Roaming\Electron\logs\integral-notes-20260502.jsonl` に残っていた。
- 最後の再発では次の流れだった。
  - `submit-result`: `@@continue` の AI 結果は返っている。
  - `preview-start`: 挿入前の疑似 streaming preview に入っている。
  - `preview-finished` は出ていない。
  - `insert-start` も出ていない。
  - main 側で `render-process-gone`, `reason: crashed` が記録されている。
- したがって、今回の白画面は ProseMirror / Milkdown への実挿入処理ではなく、挿入前 popup preview 表示中の renderer crash と判断した。

## 対応

- `src/renderer/MilkdownEditor.tsx` で、挿入前の疑似 streaming preview animation をスキップするようにした。
- live stream / preview 用の `streamingText` 表示を `AiMarkdown` ではなく plain `<pre>` に変更した。
- これにより、popup 内で増分 Markdown を繰り返し parse/render する経路を避ける。
- 既存の実挿入内容は変更していない。

## 確認

- `npm run build` 成功。
