## 背景

- plugin system を store 先行ではなく `interactive renderer` 先行へ切り替えた流れを、実装へ落とし込む必要があった。
- 追加要求として、plugin 開発者向けドキュメントも実装済み contract に更新する必要があった。
- さらに `plugins/shimadzu-lc/gradient.png` を使って、sample plugin の UI を `docs/10_要求/機能詳細イメージ/スライド3.PNG` に近づけたいという要望があった。
- 作業途中で、raw JSON を見せる textarea は不要とのフィードバックを受けた。

## 実施内容

- shared / SDK contract
  - `src/shared/plugins.ts`
    - `integral:set-block` と `integral:update-params` の message type を定数化。
    - renderer block / message payload 型を追加。
  - `plugin-sdk/src/renderer.js`
  - `plugin-sdk/src/renderer.d.ts`
    - `postIntegralPluginParamsUpdate(params)` helper を追加。
    - `bindIntegralPluginRenderer(render)` と合わせ、renderer bridge の最小 helper を揃えた。
- app 本体
  - `src/renderer/integralCodeBlockFeature.tsx`
    - plugin iframe から `integral:update-params` を受けて `params` を JSON に反映する reverse bridge を実装。
    - 反映後は app 側で再 serialize し、正規化済み block を再度 iframe へ送る流れにした。
    - raw JSON textarea と `整形` ボタンを外し、通常編集は plugin renderer 中心にした。
    - iframe click を ProseMirror 側で誤処理しないよう `stopEvent` に `iframe` を追加。
  - `src/main/pluginRegistry.ts`
    - plugin renderer HTML に `<base href="...">` を差し込み、`renderer/index.html` からの相対 asset path を使えるようにした。
- sample plugin
  - `plugins/shimadzu-lc/renderer/index.html`
    - preview 専用から interactive GUI へ作り替えた。
    - `analysis-time`, `time-prog` を編集できる UI とし、変更時に `integral:update-params` を送るようにした。
    - `../gradient.png` をテンプレート画像として使い、スライド3に近い見た目へ寄せた。
- ドキュメント
  - `docs/20_アーキテクチャ/pluginシステム.md`
    - reverse bridge 実装済み、asset path 解決、raw JSON textarea 廃止後の制約と次段階へ更新。
  - `plugin-sdk/docs/開発者向けアナウンス.md`
    - renderer contract, message type, asset path, minimal code sample を追加。
  - `plugin-sdk/README.md`
    - renderer helper の用途を追記。
  - `plugins/shimadzu-lc/README.md`
    - sample renderer が GUI 編集と `gradient.png` を使うことを追記。
