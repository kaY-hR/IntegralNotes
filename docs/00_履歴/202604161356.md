## 会話概要

- ユーザー要望:
  - Python block を `run: scripts/pca.py:main` のような人間可読な source にしたい
  - slash menu から callable を選んで block 挿入できるような設計を文書化したい
  - dataset は source / derived の扱い差を減らし、どちらも `.idts` 管理に統一したい
  - derived dataset は hidden `.store/.integral/` 配下にあるため通常見えない、という差だけに寄せたい

## 更新した方針

- block は user-facing source と internal normalized schema の二層構造にする
  - note source では `run:` / `use:` shorthand を使う
  - app 内部では従来どおり `plugin / block-type / params / inputs / outputs` に正規化する
- Python は `general-analysis` plugin が workspace の `.py` を scan し、decorator 付き関数を block 候補として公開する
  - block-type は `relative/path.py:function`
  - slash menu では display name を主表示し、callable path を補助表示する
- dataset は source / derived を問わず `.idts` manifest を canonical entrypoint に統一する
  - source dataset は visible `.idts` を default にする
  - derived dataset は hidden `.store/.integral/datasets/{datasetId}.idts` を default にする
  - 実データ directory は `.store/objects/{datasetId}/` 側に置いてよい
