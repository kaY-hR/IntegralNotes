# 2026-05-05 00:22 userData 保存先修正と skill 方針

## 相談

- IntegralNotes のローカル設定、AI Chat 履歴、API key 設定が `%APPDATA%\Electron` に保存されていることを確認した。
- 開発時でもアプリ固有の folder に保存したい。
- global / cwd の skill を両方使えるようにしたい。

## 対応

- `src/main/main.ts` で起動直後に `app.setName("IntegralNotes")` を設定するようにした。
- `INTEGRALNOTES_USER_DATA_DIR` が指定された場合は、その path を `userData` として使い、directory を作成するようにした。
- 通常起動時の `userData` を次に分けた。
  - packaged: `%APPDATA%\IntegralNotes`
  - dev: `%APPDATA%\IntegralNotes-dev`
- 旧 `%APPDATA%\Electron` から、設定・AI chat 履歴・plugin・log など必要な entry だけを target `userData` に初回コピーするようにした。
- cache / GPUCache / Local Storage などの Chromium cache 類は移行対象にしない方針にした。

## skill 方針

- skill は解析 script と違い、実行済み block の再現性 source of truth ではないため、global skill を直接使える設計でよい。
- 優先順は `cwd` の project skill、global/user skill、built-in/core skill が自然。
- UI では skill の出所を `project` / `global` のように見せると分かりやすい。

## 確認

- 初回 `npm run build` は `node_modules` に `graphology` が無く renderer build が失敗した。
- `npm install` で package.json に定義済みの不足 dependency を補完した。
- 再度 `npm run build` を実行し、renderer / main とも成功した。
