## 背景

- top menu から `blob 登録` と `chunk 作成` へ入れる導線が別れており、ユーザー操作としてまとまりが悪かった。
- あわせて、Python Script 登録時の input 指定は `chunk を直接選ぶ` よりも、設計どおり `blob を複数選び、app 側で source chunk を自動生成する` 形へ寄せる必要があった。

## 実装したこと

### データ登録導線

- `src/renderer/DataRegistrationDialog.tsx`
  - top menu の `データ登録` から開く dialog を追加
  - `ファイルを blob 登録`
  - `フォルダを blob 登録`
  - `source chunk を作成`
  の入口を集約
- `src/renderer/App.tsx`
  - top menu の `Import Files / Import Folders` を廃止し、`データ登録` ボタンへ置換
  - dialog 経由の blob 登録後も workspace snapshot を再読込するよう変更

### blob 選択 / source chunk 作成

- `src/renderer/IntegralAssetDialogs.tsx`
  - `BlobSelectionDialog` を追加
    - blob の複数選択だけを担当
    - 必要に応じてその場で file / directory blob を追加可能
  - 既存の source chunk 作成 dialog は `BlobPickerDialog` として維持しつつ、内部では `BlobSelectionDialog` を使う形に整理

### Python Script 登録

- `src/renderer/PythonScriptDialog.tsx`
  - input slot の UI を `chunk選択` から `blob複数選択` に変更
  - slot ごとに選択した blob 群を保持し、script 登録完了後に `createSourceChunk` を呼ぶ形へ変更
  - 生成された `CNK-...` を初期 block の `inputs` に埋めて editor へ挿入する実装に変更

### スタイル

- `src/renderer/styles.css`
  - `データ登録` dialog 用 card と action row のスタイルを追加
  - dialog の responsive 設定に新しい data registration dialog を追加

## 補足

- `createSourceChunk` は既存どおり `links.json` ベースの source chunk 生成を利用している
- `npm run build` は成功
