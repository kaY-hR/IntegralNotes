## 背景

- 独自 UI を将来的に plugin 化するため、実行ファイルの配置先、manifest 制約、IntegralNotes 本体の認識方法を先に整理する必要があった。
- 既存実装では `LC.Method.Gradient` と `StandardGraphs.Chromatogram` が renderer/main に hard-code されていた。

## 実施内容

- `docs/20_アーキテクチャ/pluginシステム.md`
  - plugin のインストール先を `app.getPath("userData")/plugins/<pluginId>` とする方針を文書化。
  - `integral-plugin.json` の必須項目、renderer / executable の役割分担、認識フロー、段階的移行方針を整理。
- `docs/20_アーキテクチャ/jsonスキーマ.md`
  - `itg-notes` block の JSON が plugin namespace とどう結びつくかを追記。
  - `type` 未登録時は generic preview へフォールバックする前提を明記。
- `src/shared/plugins.ts`
  - plugin manifest / action / block contribution / installed plugin summary の共有型を追加。
  - manifest 文字列の parse・簡易検証処理を追加。
  - 現在の `LC.Method.Gradient` と `StandardGraphs.Chromatogram` を組み込み plugin manifest として表現。
- `src/main/pluginRegistry.ts`
  - `userData/plugins` を走査して `integral-plugin.json` を読む plugin 発見処理を追加。
  - 組み込み plugin と外部 plugin をまとめて返す registry を追加。
- `src/main/main.ts`
  - `plugins:listInstalled` IPC を追加。
- `src/main/preload.ts`
  - renderer から利用できる `listInstalledPlugins()` API を追加。
- `src/shared/workspace.ts`
  - `IntegralNotesApi` に `listInstalledPlugins()` を追加。
- `src/renderer/integralBlockRegistry.tsx`
  - block 定義の一次情報を shared の組み込み plugin manifest から生成する形へ整理。

## 確認

- `npm run build` 成功。

## 残課題

- 外部 plugin の renderer bundle を実際に読み込む処理は未実装。
- plugin executable の起動 protocol は仕様化までで、実行経路への接続は次フェーズ。
