- managed data の消失対応を追加で整理した。

- `src/main/integralWorkspaceService.ts`
  - `original-data` を管理対象から外すとき、`memberIds` にその ID を含む `.idts` manifest を走査し、該当 ID を除去するようにした。
  - manifest 更新後は対応する dataset metadata の `memberIds` / `hash` も更新し、data-note sync で frontmatter も追従するようにした。
  - これにより、無くなった元データを含み続ける source dataset manifest が残らないようにした。
  - さらに、アプリ内の workspace delete 操作で managed data の実体を削除したときは、tracking issue に回さず、その場で metadata json と data-note も削除するようにした。
  - delete 対象が original-data の場合は、上記の `.idts` member cleanup も同時に走る。

- 方針
  - アプリ外でファイルが消えた場合は従来どおり `missing` issue + confirm を維持する。
  - アプリ内でユーザーが明示的に delete した場合だけ、metadata / data-note cleanup を即時実行する。
