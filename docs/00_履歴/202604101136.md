## 背景

- plugin block の外側 UI を本体が持つ構成だと、plugin ごとの UI/UX を一体で設計しにくいという懸念が出た。
- 特に `shimadzu-lc` のような操作系 plugin では、`装置操作を実行` のような privileged action も plugin renderer 内で自然に配置したいという要望があった。
- 設計判断では既存差分の少なさではなく、理想的な ownership を優先する方針が共有された。

## 実施内容

- `src/shared/plugins.ts`
  - `integral:request-action` と `integral:action-state` の message contract を追加した。
- `plugin-sdk/src/renderer.js`, `plugin-sdk/src/renderer.d.ts`
  - `bindIntegralPluginActionState`
  - `postIntegralPluginActionRequest`
  - を追加し、plugin renderer から privileged action を要求できる helper を用意した。
- `src/renderer/integralCodeBlockFeature.tsx`
  - renderer を持つ block では app 本体の外側 toolbar / action button / description を常設しない構成へ変更した。
  - plugin iframe を主 UI とし、app 側は JSON serialize / validation / action 実行 / state 中継に徹する構成へ寄せた。
  - `integral:update-params` に加えて `integral:request-action` を受け、結果を `integral:action-state` で iframe に返す bridge を追加した。
  - renderer 読込失敗や renderer 未登録時だけ、本体側 fallback UI を表示するようにした。
- `src/renderer/styles.css`
  - plugin renderer 主体に合わせて、Integral block 外枠と iframe shell の見た目を薄くした。
- `plugins/shimadzu-lc/renderer/index.html`
  - renderer 内に `装置操作を実行` ボタンを追加。
  - action 実行中 / 成功 / エラーの state を renderer 内で表示するようにした。
  - `詳細 / シンプル` 切替と編集 GUI は維持した。
- docs
  - `docs/20_アーキテクチャ/pluginシステム.md`
  - `plugin-sdk/docs/開発者向けアナウンス.md`
  - `plugin-sdk/README.md`
