# 2026-05-10 13:13 履歴

## 背景

- 拡張機能の管理が runtime plugin だけでなく、AI skill、Python script、package まで含むようになり、sidebar 表示では情報量と操作導線が足りなくなった。
- ユーザーと、管理画面は sidebar ではなく main workspace tab とし、`Current Workspace` と `Global` を横並びで比較する方針を確認した。

## 決定

- 拡張機能管理は `Extensions` tab として開く。
- 行は component 種別、列は `Current Workspace` / `Global` とする。
- top-level の `Skills` / `Scripts` / `Standalone Runtime` は package に属さない standalone component だけを表示する。
- package に属する component は `Packages` 行の child として read-only 表示し、操作 button は package 行にだけ出す。
- `Current Workspace` へ import できるのは standalone global script と package だけとし、skill / runtime plugin には import button を出さない。
- workspace standalone skill / script には `Open` / `Delete` / `Stock on Global` を用意する。
- standalone script の import / stock は file copy とし、複数 helper に依存するものは package 化する。

## 対応

- `docs/30_設計/56_拡張機能管理タブ.md` を追加し、管理タブの目的、レイアウト、location、表示単位、操作、状態表示、非目標を仕様化した。
- `docs/30_設計/55_Package管理.md` の package 由来 runtime plugin 表示先を拡張機能管理タブに合わせた。
- `docs/30_設計/80_ActivityBarとSidebarView.md` を更新し、Extensions を built-in sidebar view ではなく built-in workspace-tool item として main workspace tab を開く前提にした。
