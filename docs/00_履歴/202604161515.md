## 概要

- `.idts` の `![...](...idts)` 埋め込み preview を、通常の dataset manifest viewer と統一した。
- 旧 `.py-scripts` / `PYS-*` ベースの Python plugin 登録システムを除去した。
- Python 解析 block は workspace 上の `.py` を直接 scan し、decorator 付き関数を `relative/path.py:function` として公開する方式へ切り替えた。
- `itg-notes` block は JSON 互換を残しつつ、YAML shorthand (`run:` / `use:` / `in:` / `params:` / `out:`) を新規 canonical form として扱うようにした。
- editor の block 挿入導線は既存スラッシュメニューを外し、`>` popup から Python callable を選ぶ方式に変更した。

## 実装メモ

### 1. dataset / `.idts`

- `workspaceEmbedFeature.tsx` で `.idts` embed preview が `DatasetManifestFileViewer` を使うようにした。
- source / derived とも `.idts` manifest viewer で開ける前提を維持した。

### 2. Python 実行系

- `IntegralWorkspaceService` で `.py-scripts` scan / 登録 API / bundled copy を削除した。
- workspace 直下の `.py` を再帰 scan し、`@integral_block(...)` + `def name(...)` を regex ベースで検出するようにした。
- `general-analysis` block-type は `relative/path.py:function` をそのまま使う。
- 実行時は workspace root を `cwd` にし、`.store/.integral/runtime/BLK-.../analysis-args.json` を作って Python callable を直接呼び出す。
- user script から `from integral import integral_block` できるよう、runner 側で no-op decorator module を `sys.modules` へ差し込む方式にした。

### 3. `itg-notes` source

- `integralBlockRegistry.tsx` を差し替え、JSON と YAML の両方を parse できるようにした。
- 新規 block 生成は YAML code fence を返す。
- YAML では input slot に dataset ID ではなく `.idts` path を保持し、実行時に dataset metadata へ解決する。
- block 実行後に note source を dataset ID で書き戻さないようにした。
- output dataset 表示は `createdByBlockId` と `kind` から最新結果を引く。

### 4. editor UI

- `MilkdownEditor.tsx` に `>` popup completion を追加した。
- 候補は `source === "python-callable"` の block のみ表示する。
- 選択すると YAML `itg-notes` block を挿入する。
- 旧 block edit / slash menu の Integral snippet 導線は削除した。
- menubar の `Python Scripts` 導線と `PythonScriptDialog` も削除した。
