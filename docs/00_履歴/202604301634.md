# 2026-04-30 idts / output folder 整理

## 背景

- `.idts` 専用 viewer、hidden bundle directory、manifest file path 指定が混ざり、dataset の扱いが分かりにくくなっていた。
- 未リリース製品なので後方互換性は考慮せず、理想設計へ寄せる方針になった。
- 作業前に `origin/main` の更新を fast-forward で取り込んだ。

## 仕様更新

- `.idts` を開いたときは専用 viewer ではなく dataset の data-note を表示する方針にした。
- dataset の data-note 初期本文は、構成 file / folder への Markdown link 箇条書きにした。
- `.idts` input picker では `新しいデータセットを作る` を先頭に出し、任意 file 複数選択後に `.idts` 保存先を選ぶ方針にした。
- `.idts` output は file path ではなく output folder path を指定し、実行成功後に `{folder名}.idts` manifest をその folder 内へ作る方針にした。
- `.idts` output の既定 folder は `analysis-result/{解析表示名}_yyyyMMddHHmm_{英数字3文字}` とし、root folder は設定 dialog で変更できるようにした。

## 実装

- app settings に `analysisResultDirectory` を追加し、設定 dialog に `解析結果フォルダ` を追加した。
- `.idts` output の既定値生成を folder path へ変更し、block UI の placeholder / 新規 block 挿入 / main process 実行前補完で使うようにした。
- `.idts` output 実行では Python に visible output folder path を渡し、成功後に folder 内へ `.idts` manifest と dataset metadata を作るようにした。
- derived dataset の output folder 配下が auto managed file 登録されないよう、dataset `dataPath` を reserved path として扱うようにした。
- dataset data-note 生成時に、source dataset は member path、derived dataset は output folder 内 file への link 箇条書きを作るようにした。
- `.idts` 専用 viewer component を削除し、workspace / embed から `.idts` を開く場合は data-note Markdown を表示するようにした。
- `.idts` input popup と dataset picker dialog から、任意 file 選択 + `.idts` save dialog による dataset 作成を呼べるようにした。
- AI / skill 向けの Python block 実装指示に、`.idts` output は directory path であり manifest は app が作ることを追記した。
