## 調査と判断

- 既存実装では block source の `out:` は scalar 形式だったが、renderer / main process / shared model を拡張すれば slot ごとの output 設定を保持できる見込みだった
- `.idts path -> datasetId` 解決層は既に存在しており、user-facing path と internal ID の分離契約はそのまま維持できる
- そのため、旧 `auto` / scalar を読みつつ、新形式 `out.{dir,name,latest}` を保存する方針で問題なく実現可能と判断した

## 仕様整理

- Python 実行 block は `Inputs` / `Outputs` の section を分ける
- output slot ごとに次を保持する
  - `directory`: visible `.idts` manifest の保存先
  - `name`: dataset 名および file stem 候補
  - `latest`: 直近実行で生成された `.idts` path
- 既定値は `directory = /Data`, `name = slot名`
- 実行時に `{directory}/{name}.idts` が衝突した場合は `_1`, `_2`, ... を付けて既存 file を壊さず新規保存する
- internal canonical ID と `.store/objects/{datasetId}` は維持する
