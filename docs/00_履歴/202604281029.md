## 決定事項

- file の実体は ID で管理し、`.store/.integral/{ID}.json` を metadata の単位とする。
- metadata JSON には hash と file path の両方を持たせ、path または hash が変わっても同じ ID として追跡できるようにする。
- `itg-notes` block の `in:` は、authoring 時は path を書けるが、保存時または実行前に managed data ID へ正規化する。
- `out:` は実行前は出力予定 path、実行成功後は生成された managed data ID とする。
- 実行済み block は provenance として read-only 表示し、再実行は既存 block の再実行ではなく新しい block 作成で扱う。
- `outputConfigs` / `out.{dir,name,latest}` / `auto` output placeholder / 互換読み取りは削除対象とし、PoC・リリース前のため後方互換は残さない。
- Markdown link / image の記法は path のまま維持する。
- LLM 用 tool として path から ID、ID から path を解決する API を用意する。
