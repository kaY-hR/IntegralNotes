# 2026-04-09 00:22 Summary

## 概要

- `docs\00_履歴\202604090006_UserReq.md` を起点に、json を使った独自 UI 挿入の step2-1 に着手
- 要求は「trigger で候補を選び、選択すると json を挿入する」こと
- `/` トリガーを使うと標準スラッシュコマンドが消える懸念があるため、Milkdown の GitHub 実装も確認した

## 調査結果

- 2025-05-24 の Discussion #1907 では maintainer が「項目追加 API はない」と回答している
- ただし `Milkdown/milkdown` の `v7.17.1` にある `packages/crepe/src/feature/block-edit/menu/config.ts` では `config?.buildMenu?.(groupBuilder)` が実装されている
- 同じく `packages/crepe/src/feature/block-edit/index.ts` では `block-edit` feature が `configureMenu(ctx, config)` を経由して標準 `/` メニューを組み立てている
- そのため、少なくとも `v7.17.1` 系では「標準 `/` メニューを潰さずに項目追加」は可能と判断
- 一方で、別 plugin として独自の `/` slash plugin を足すと、標準メニューを消すのではなく競合させる可能性がある

## 実施した対応

- `src/renderer/integralSnippetMenu.ts`
  - Crepe の `block-edit` `buildMenu` を使い、標準スラッシュメニューへ `Integral` グループを追加
  - `LCのグラジエント設定`
  - `クロマトグラム表示`
  - 2 つの項目を追加し、選択時に fenced `json` code block を挿入するように実装
- `src/renderer/MilkdownEditor.tsx`
  - `Crepe` 初期化時に上記 `featureConfigs` を渡すよう変更

## 検証

- `npm run build` 成功

## 補足

- 今回は最小実装として既存 `/` メニュー拡張を採用した
- `>>` のような独自 trigger は、別 plugin を追加して `SlashProvider` をカスタムすれば実装できる見込み
- 将来的な独自 UI 置換を考えると、json は生テキストより fenced `json` code block で保持した方が識別しやすい
