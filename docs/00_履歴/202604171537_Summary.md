# 2026-04-17 15:37 Summary

## 会話要約

- `html` embed は正常だが、`md` embed だけ wheel でも動かず scrollbar も出ない、という追加切り分けが行われた。
- 調査の結果、readonly markdown preview に対しても通常 editor 用の `.editor-surface .milkdown .editor` スタイルが後勝ちで当たっており、巨大な `min-height` が混入して nested preview の overflow を壊していることが分かった。

## 実装

- `src/renderer/styles.css`
  - 通常 editor 用ルールの後段に、`.editor-surface .readonly-markdown-preview .milkdown` と `.editor-surface .readonly-markdown-preview .milkdown .editor` の明示 override を追加した。
  - これにより readonly markdown preview では
    - `max-width: none`
    - `min-height: 0`
    - `height: 100%`
    - `overflow-y: scroll`
    - `padding: 16px 18px 22px`
    - `box-shadow: none`
    が優先され、通常 editor の `min-height: calc(100vh - 230px)` などが混入しないようにした。
  - mobile 用 media query の中でも同じ override を追加し、狭い画面でも通常 editor の縮小 rule が nested preview に混ざらないようにした。

## 確認

- `npm run build` : 通過

## 補足

- 今回は `md` embed 専用の style 競合解消であり、`html` や plugin viewer には触れていない。
