# Python 解析 block の params schema / UI 対応

## 目的

Python 解析 block の parameter を、Python decorator 側の `params={...}` schema を正として定義し、block 上のフォーム UI と実行 payload に反映する。

## 決定事項

- `@integral_block(..., params={...})` に Python literal の JSON Schema subset を書く。
- root schema は `type: object` と `properties` のみ扱う。
- MVP の property type は `string / number / integer / boolean` と `enum`。
- `title / description / default / minimum / maximum` を UI metadata として扱う。
- `required` は MVP 対象外。
- `default` が無い property は `null` 初期値にする。
- decorator に schema が無い場合、有効 param は空とする。
- schema 外 key、未対応型、YAML 手書き param は削除してよい。

## 実装内容

- shared type に `IntegralParamsSchema` / primitive param schema 型と正規化 helper を追加した。
- Python callable discovery で decorator の `params` literal を静的解析し、block type catalog に載せるようにした。
- 実行前の block 正規化で `params` を decorator schema に沿って再生成するようにした。
- renderer の block YAML 生成で schema default から `params:` を生成し、有効 param が無い場合は `params:` 自体を出さないようにした。
- block UI に schema 由来の primitive param フォームを追加した。
- workspace Python SDK の `integral_block` decorator に `params` 引数を追加した。
- inline Python block system prompt と fallback skill prompt に params schema ルールを追加した。
- `.codex/skills/implement-integral-block/SKILL.md` を追加し、py block 実装時の params ルールを明文化した。
- `docs/10_要求` の MVP 要件と UX 文書を更新した。
