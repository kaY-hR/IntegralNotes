# 2026-05-01 01:48 py block input UX / Undo

## 背景

- 解析 block の input slot は、一度 file / dataset を選ぶと Backspace などで消せず、別 file へ変えたいときの導線が分かりにくかった。
- 実行後に params を変えて試したい場合、既存の結果 file / note embed が残り続け、作業 folder が散らかりやすかった。

## 対応

- `src/renderer/integralCodeBlockFeature.tsx` の input slot を、path 表示欄そのものを編集・クリアできる形にした。
  - 入力欄 focus / typing で既存の候補 popup と preview を使う。
  - 入力欄を空にすると slot を `null` に戻す。
  - `x` button で割り当て済み input を解除できる。
  - 完全一致する managed file / dataset path または ID は blur 時にも slot へ反映する。
- 実行済み block の `Delete` 左側に `Undo` を追加した。
  - Undo は同じ block 定義から新しい block ID / 初期 input / 初期 params / 新しい output path を作り直す。
  - 生成済み managed output の file / dataset folder を削除し、metadata / data-note を整理する。
  - workspace 内の Markdown link / embed から、削除対象 output への参照を削除する。
  - 現在開いている note の未保存 editor state からも output embed を取り除く。
- `src/shared/workspaceLinks.ts` に Markdown link / embed 削除 helper を追加した。
  - fenced code block 内は触らない。

## 確認

- `npm run build`
  - 成功。
  - Vite CJS API deprecated warning と chunk size warning は既存系の警告として残る。
