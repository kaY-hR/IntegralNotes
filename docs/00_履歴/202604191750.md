## 概要

- `share_note_with_input` と `auto_insert_to_work_note` / embed 周りの不具合調査を行い、shared note 解決と legacy slot 記法の互換を修正した
- ユーザー報告は「ノート共有はできていそうだが、`auto_insert_to_work_note: True` を付けても note へ `![]()` が入らないことがある」というもの

## 確認したこと

- 最新 commit の実装を確認した結果、shared note 実装には以下のズレがあった
  - output の `noteTargetId` が、file input を共有先にした場合に metadata ID ではなく workspace path になっていた
  - dataset 側の note open 導線が `App.tsx` の resolver で統一されておらず、導線ごとに挙動が揺れる可能性があった
  - Python block のサンプルや実データはまだ `project_to_inputs` を使っていたが、parser 側はそれを読まなくなっていた

## 修正

- `src/main/integralWorkspaceService.ts`
  - `resolveOutputSlotSharedNoteTargetId()` を async に変更
  - `share_note_with_input` の共有先は、input reference をそのまま保存せず、dataset / managed file metadata を解決して最終的な `noteTargetId` または self ID を保存するようにした
  - output dataset / managed file 作成時の `noteTargetId` 設定を `await` 経由で解決するようにした
  - Python slot parser に legacy 互換を追加し、`project_to_inputs` がある場合は
    - `share_note_with_input = <先頭要素>`
    - `embed_to_shared_note = true`
    として扱うようにした

- `src/renderer/App.tsx`
  - `findManagedDataTargetById()` に dataset 解決を追加
  - `findManagedDataTargetForPath()` に dataset 解決を追加
  - `collectManagedDataTargetsForPaths()` に dataset 解決を追加
  - note open が managed file だけでなく dataset の `noteTargetId` に対しても一貫して動くようにした

- `src/renderer/IntegralAssetDialogs.tsx`
  - dataset picker の `ノート` ボタンが `dataset.noteTargetId ?? dataset.datasetId` を開くようにした
