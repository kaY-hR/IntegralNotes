# 2026-05-09 22:50 履歴

## 背景

- Explorer の hidden 表示を ON にしても workspace `.packages` が表示されないことを確認した。
- 原因は renderer の hidden filter ではなく、main process の workspace snapshot 生成時点で `.packages` を固定除外していたこと。

## 対応

- `workspaceService` の snapshot 固定除外対象から `.packages` を外し、`.git` のみを常時除外するようにした。
- `.packages` は snapshot には含め、hidden 表示 OFF では renderer 側の既存 hidden filter で非表示、hidden 表示 ON では表示される形にした。
- `.packages` は引き続き managed file tracking の通常自動登録対象や search の通常対象からは除外する方針として、設計文書を更新した。

## 確認

- `npm run build` 成功。
- `WorkspaceService.readDirectoryEntries("")` の直接確認で `.packages` が snapshot entry に含まれ、`.git` は含まれないことを確認した。
