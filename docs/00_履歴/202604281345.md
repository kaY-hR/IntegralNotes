# 2026-04-28 13:45 会話まとめ

## 目的

新規挿入した Python block が、未実行なのに実行済み扱いになって read-only 表示される問題を修正する。

## 調査

- 直近履歴で、`out:` は実行前は希望 output path、実行後は generated managed data ID へ書き戻す方針であることを確認した。
- renderer の `isExecutedBlock` は、`out:` の値が managed data に解決できるかだけを見ていた。
- そのため、新規 block の既定 output path (`/Data/...`) が過去に作成済みの managed data path と一致すると、未実行 block でも実行済みと誤判定されていた。

## 修正

- `src/renderer/integralCodeBlockFeature.tsx` の実行済み判定を、path 解決ではなく stored output reference の ID 解決に変更した。
- さらに、解決した managed data の `createdByBlockId` が現在の block id と一致する場合だけ実行済みとみなすようにした。
- output 欄の表示や path 解決は変更せず、read-only 化の条件だけを provenance に沿う形へ絞った。

## 確認

- `npm run build` 成功。
- `git diff --check` 成功。

## 補足

- 共有 `node_modules` の `.bin` が不完全だったため、root 指示に従って `main_ws` で `npm install` を実行し、`python_block_readonly_fix_wt` から junction で共有している依存を復旧した。
