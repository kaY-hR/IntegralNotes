# Relation Graph managed Markdown scan 修正

## 発生した問題

`C:\Users\shimadzu\Documents\Notes\test.md` には以下の relation があるはずだったが、Relation Graph 上では孤立して見えていた。

- `![1.00](/image.png)`
- `![1.00](/Data/report_Z8J.html)`
- `itg-notes` block から `scripts/spheroid_detection.py:main`
- block input / output の managed data ID

DB を確認すると、`test.md` は managed file node として存在していたが、`file:test.md` の origin が存在せず、`test.md` 起点の edge が作られていなかった。

## 原因

Relation Graph scan で、visible path が managed data と一致する場合に `collectWorkspaceStructure()` が早期 `continue` していた。このため managed file になっている `.md` は node だけ作られ、Markdown link / `itg-notes` block extractor が呼ばれていなかった。

また、同じ `test.md` path に複数 managed file metadata があり、Graph View の active node 解決が孤立した duplicate node を拾う可能性もあった。

## 修正

- managed file として登録済みの `.md` / `.py` / `.idts` でも scan origin を作るようにした。
- managed file が directory の場合は従来通り単一 node として扱い、配下の詳細 scan は抑制する。
- data-note を持たない managed file で同じ visible path が重複する場合、graph node を 1 つに寄せ、複数 managed ID から同じ node に解決するようにした。
- `RELATION_GRAPH_SCHEMA_VERSION` を `2` に上げ、既存 DB を自動再構築させるようにした。

## 検証

- `npm run build` 成功。
