## 背景

- plugin の独立開発を考えると、bundled plugin とは別に external plugin を `userData/plugins` へ install する導線が必要になった。
- 「各 plugin の installer を作って install しないときちんと動かないのか」という確認があり、installer 作成まで進めることになった。

## 実施内容

- `plugin-sdk`
  - `src/installer.js` を追加し、local install / uninstall helper を実装。
  - `src/installer.d.ts` を追加。
  - `package.json`, `src/index.js`, `src/index.d.ts`, `README.md` を更新し、installer export を追加。
  - `docs/開発者向けアナウンス.md` に install の考え方とコマンド例を追記。
- repo 側 CLI
  - `src/dev/plugin-installer.mjs` を追加。
  - `where`, `install`, `uninstall` を実装。
  - `--target-root`, `--user-data` を受けられるようにした。
- root scripts
  - `package.json` に `plugins:where`, `plugins:install:all`, `plugins:uninstall:all` を追加。
- 各 sample plugin
  - `plugins/shimadzu-lc/package.json`
  - `plugins/standard-graphs/package.json`
  - 各 plugin に `install:local`, `uninstall:local`, `where:local` を追加。
  - `plugins/README.md` と各 plugin `README.md` に install 手順を追記。
- 設計文書
  - `docs/20_アーキテクチャ/pluginシステム.md`
    - bundled plugin は install 不要、external plugin は `userData/plugins` への install が必要と明記。
    - installer flow の mermaid と利用コマンドを追加。

## 判断

- 現状の bundled plugin は install 不要で動く。
- ただし third-party / 独立開発の観点では external plugin install 導線が必要。
- install target は既定で `%APPDATA%/IntegralNotes/plugins` とし、必要なら CLI option / env で上書きできるようにした。
