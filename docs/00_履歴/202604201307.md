## 概要

- 解析 block の input popup で候補が 0 件になる原因を調査し、`.idts` input の filter 条件を修正した

## 原因

- renderer 側の `input-dataset` popup は `assetCatalog.datasets` を対象にしつつ、Python slot 定義の `format` から生成された `acceptedKinds` だけで候補を絞っていた
- `Notes/src/lc_text_to_chromatogram_json.py` の input slot は
  - `extensions: [".idts"]`
  - `format: "bundle/idts"`
  を要求している
- 一方で、workspace 上の実サンプル `Notes/Data/my-LC-Samples.idts` の manifest / metadata は
  - `kind: "source-bundle"`
  だった
- そのため「拡張子は一致しているが dataset kind が不一致」という状態で、popup 候補が全部落ちていた

## 修正

- `src/renderer/integralCodeBlockFeature.tsx`
  - `input-dataset` picker state に `extensions` も持たせるよう変更
  - dataset 候補は
    - `kind` 一致
    - または dataset manifest path の拡張子一致
    のどちらかで通すように変更
