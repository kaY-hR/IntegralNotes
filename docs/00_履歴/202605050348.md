# web検索 tool_call 設計相談

## 相談内容

- web検索用の `tool_call` を準備するにあたり、一般的な LLM tool 実装パターンを確認したい。

## 確認した前提

- IntegralNotes の AI Chat は Vercel AI SDK v6 の `ToolLoopAgent` を使っている。
- direct Anthropic / direct OpenAI / AI Gateway の runtime 切替がある。
- 既存設計では typed workspace tools と承認付き `runShellCommand` があり、MCP registry は未実装。

## 整理した方針

- 一般的な tool call は `name` / `description` / `inputSchema` / `execute` / `toolCallId` / `tool result` の契約で実装する。
- web検索は大きく分けて provider built-in tool と自前検索 tool の 2 パターンがある。
- IntegralNotes では、最初は provider built-in の web search を runtime ごとに差し込むのが自然。
- 検索結果の citations / sources / query / errors を transcript に残せるよう、内部 trace model を先に揃えるのが重要。
