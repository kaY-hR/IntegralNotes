# 2026-04-08 18:33 作業ログ

## 依頼

- Milkdown 使用時に下記エラーが出る
  - `Uncaught MilkdownError: Context "editorView" not found, do you forget to inject it?`

## 原因

- `src/renderer/MilkdownEditor.tsx` が `initialValue` 依存で `useEffect` を回していた
- その結果、入力のたびに Milkdown エディタを破棄・再生成していた
- さらに dev 時は `React.StrictMode` によりマウント/アンマウントが二重に走り、Milkdown の初期化途中に破棄が入りやすかった

## 対応

- `src/renderer/MilkdownEditor.tsx`
  - Milkdown の生成をコンポーネント mount 時の 1 回だけに変更
  - 初期化完了前に unmount された場合は、create 完了後に破棄する形へ修正
- `src/renderer/App.tsx`
  - save 時の `refreshKey` による不要な editor remount を削除
  - `MilkdownEditor` の key を `relativePath` のみに変更
- `src/renderer/main.tsx`
  - `React.StrictMode` を外して、Milkdown との dev 時ライフサイクル競合を避けるように変更

## 検証

- `npm run build` 成功

## メモ

- すでに `npm run dev` を走らせている場合は、watch によって自動反映される想定
- もし反映が怪しければ Electron ウィンドウを閉じて `npm run dev` をやり直せばよい
