# 2026-04-19 18:17

## 背景

- ユーザー要求:
  - 未リリース PoC 前提で、後方互換を捨てて legacy 実装を除去したい
  - `share_note_with_input` / `embed_to_shared_note` / `noteTargetId` を中心に、仕様と実装を clean&clear に揃えたい
  - `auto_insert_to_work_note = True` でも `![]()` が作業 note に反映されない現象もあわせて直したい
  - `$doc-review` 観点で docs と実装の乖離も同時に潰したい

## doc-review で見つかった主な乖離

- Python SDK がまだ `project_to_inputs` を canonical として持っていた
- `Notes/src/*.py` のサンプル block が `project_to_inputs` を残していた
- `src/main/integralWorkspaceService.ts` に `project_to_inputs` の互換 parser が残っていた
- renderer 側に `hasDataNote` という古い意味づけが残り、shared note target を first-class に扱えていなかった
- note を開く導線の一部が `id` 前提で、`noteTargetId` を渡したときに解決できない箇所があった
- block 実行結果を note に戻す処理が `lastSyncedMarkdownRef` 依存で、`auto_insert_to_work_note` の `![]()` 反映に失敗しうる経路があった

## 今回の修正

- 実装
  - `src/main/integralWorkspaceService.ts`
    - Python slot parser から `project_to_inputs` 互換を削除
    - output slot は `share_note_with_input` / `embed_to_shared_note` のみ解釈するよう整理
    - summary の `hasDataNote` を `canOpenDataNote` に置換
  - `src/renderer/App.tsx`
    - `noteTargetId` 指定でも managed data note を解決できるよう修正
  - `src/renderer/integralCodeBlockFeature.tsx`
    - dataset / managed-file を統一的に note 解決するよう整理
    - note button が shared target を開けるよう修正
  - `src/renderer/workspaceEmbedFeature.tsx`
    - data-note fallback が dataset も拾うよう修正
  - `src/renderer/MilkdownEditor.tsx`
    - block 実行結果の反映に `editor.getMarkdown()` を使うよう変更
    - `auto_insert_to_work_note` の `![]()` 挿入が stale markdown で落ちる経路を解消
  - `src/main/dataNote.ts`
    - 未使用の `hasDataNoteFrontmatter` helper を削除

- Python SDK / サンプル
  - `scripts/integral/__init__.py`
  - `Notes/scripts/integral/__init__.py`
    - SDK schema を `share_note_with_input: str | None` / `embed_to_shared_note: bool` へ更新
    - `project_to_inputs` を見つけたら明示エラーに変更
  - `Notes/src/lc_text_to_chromatogram_json.py`
  - `Notes/src/chromatogram_pca.py`
    - `project_to_inputs` を除去し、必要箇所を `embed_to_shared_note: True` へ移行

- 補助文書
  - `Notes/.claude/skills/implement-integral-block/**`
  - `Notes/.codex/skills/implement-integral-block/**`
    - block contract の記述を新 schema に更新

## 検証

- `npm run build` 成功
- `project_to_inputs` の実装上の使用は除去し、SDK 側では「廃止済み」としてのみ残した
- `hasDataNote` の summary field は除去した

- 公式 docs (`docs/10_要求`, `docs/30_設計`, `docs/Issues.md`) は今回レビュー範囲では既に新 schema 側へ追随済みで、大きな矛盾は見つからなかった
- 残る `project_to_inputs` 文字列は、SDK の廃止エラーメッセージのみ
