## 今回の唯一の正

- block 共通 schema は `id / plugin / block-type / params / inputs / outputs`
- `plugin` と `block-type` を分離
- 元データ登録単位は `blob`
- 処理入出力単位は `chunk`
- 内部契約は `1 input slot = 1 chunk`, `1 output slot = 1 chunk`
- GUI 上だけ `1 input slot = N blobs` を許し、app が source chunk を自動生成
- Python は `general-analysis` plugin が `cwd/.py-scripts/` を走査して block-type を動的生成
- Python block の `params` は MVP では `{}` 固定
- renderable な `html / image / text` は標準表示 block で描画
- 可視化 plugin は renderable chunk を生成する処理 plugin とみなす
