## 背景

- external-only plugin runtime に切り替えた後も、editor の slash command は install 前から表示されていた。
- 実際には plugin 未導入時は block を挿入できない方が自然なので、snippet menu も install 状態に追従させる必要があった。

## 原因

- `src/renderer/integralSnippetMenu.ts`
  - `LC.Method.Gradient`
  - `StandardGraphs.Chromatogram`
  の snippet を固定配列で持っていた。
- そのため `userData/plugins` に plugin が 0 件でも、slash command だけは常に見えていた。

## 実施内容

- `src/renderer/integralPluginRuntime.ts`
  - install 済み plugin 一覧を renderer から参照するため `getInstalledIntegralPlugins()` を追加。
- `src/renderer/integralSnippetMenu.ts`
  - 固定 `INTEGRAL_SNIPPETS` を廃止。
  - install 済み plugin の `blocks` を走査して snippet を動的生成する構成へ変更。
  - plugin が 0 件なら `Integral` group 自体を menu に追加しないようにした。
  - 既知 block (`LC.Method.Gradient`, `StandardGraphs.Chromatogram`) だけは従来のサンプル JSON を維持し、それ以外は最小 `{ "type": "..." }` を挿入する。
- `src/renderer/MilkdownEditor.tsx`
  - plugin runtime 初期化完了後に `Crepe` を生成する順へ変更。
  - snippet feature config も runtime 初期化後の install 状態を使って構築するようにした。

## 効果

- plugin 未導入時は slash command に plugin block が出ない。
- plugin install / uninstall 後は、既存の runtime reset + editor remount と組み合わせて slash command 一覧も更新される。
