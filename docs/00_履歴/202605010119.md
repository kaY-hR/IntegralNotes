# 2026-05-01 01:19 dirty close confirmation

## 背景

- dirty のままタブを閉じたり、dirty がある状態でアプリを終了して未保存内容が失われることがあった。
- Windows + Electron の renderer `confirm()` 問題を避けるため、確認は main process の native dialog 経由にした。

## 対応

- `src/shared/workspace.ts` / `src/main/preload.ts` に未保存変更確認用 IPC を追加した。
- `src/main/main.ts` に未保存変更破棄確認 dialog と window close 前の renderer 問い合わせを追加した。
- `src/renderer/App.tsx` で FlexLayout の `DELETE_TAB` action を intercept し、dirty tab の close 前に確認するようにした。
- アプリ終了時は renderer が dirty tab 一覧を返し、dirty がある場合だけ終了確認を出すようにした。

## 確認

- `npm run build`
  - 成功。
  - Vite CJS API deprecated warning と chunk size warning は既存系の警告として残る。
