## `docs/Issues.md` に追記した要点

- 実現性は高く、主変更点は renderer 側の markdown tab に editor mode を足すこと。
- 実装は `OpenMarkdownTab` に `editorMode` を持たせ、`MilkdownEditor` と plain text editor を切り替える構成が自然。
- `raw -> WYSIWYG` 切替では Milkdown の再初期化が必要。
- 設計上の最大の論点は `data-catalog/` 配下の data-note frontmatter で、Issue 10 の「frontmatter を見せない」方針と full-file raw 編集は衝突する。
- そのため、まずは通常ノートだけ raw 切替を許可するか、data-note は本文 raw のみに限定するのがよい、という整理にした。
