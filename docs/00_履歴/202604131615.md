## 背景

- source chunk を chunk viewer で開くと、実体データではなく `links.json` だけが renderable として扱われていた。
- `flexlayout-react` の viewer も block 内の埋め込みとして閉じ込め切れておらず、表示が Markdown 上端側に張り付く崩れがあった。
- chunk picker dialog の confirm button も、配色の都合で `Select` が読みにくいという指摘があった。

## 実装したこと

### source chunk inspection の修正

- `src/main/integralWorkspaceService.ts`
  - `source-bundle` chunk の inspection 時は `links.json` を読んで、参照先 blob payload 配下の実ファイルを inspect 対象として展開するよう変更
  - `links.json` 自体は renderable 候補から除外
  - linked payload 内の `html / image / text` をそのまま viewer へ渡すよう変更
  - chunk summary 側の `renderableCount` も同じ inspection ロジックを使うよう整理

### chunk viewer の containment 修正

- `src/renderer/styles.css`
  - `integral-renderable-layout` に `position: relative`, `overflow: hidden`, `min-width: 0` などを追加
  - FlexLayout の tab/content container に `min-width: 0`, `min-height: 0` を付与
  - renderable card 側も block 内で収まるよう `overflow` と sizing を調整

### chunk picker dialog の可読性

- `src/renderer/styles.css`
  - asset picker dialog の primary button 配色を明示
- `src/renderer/IntegralAssetDialogs.tsx`
  - confirm button ラベルを `Select` から `選択` に変更
