# 2026-04-14 11:02

## 会話要約

- source dataset の `links.json` を廃止するなら、仕様文書の更新も Issue に含めたいという要望があった。
- また、`.inlk` 案とハードリンク案のどちらを主方針にするかを今決めたい、という相談があった。

## 実施したこと

- Issue 13 に、ハードリンク方式を採る場合は `docs/10_要求`, `docs/20_アーキテクチャ`, `docs/30_設計` の更新が必要である旨を追記した。
- 方針案としては、`.inlk` の custom 形式ではなく、`source dataset` を普通のフォルダとして扱えるハードリンク方式を第一候補にするのが妥当と整理した。

## 判断メモ

- `.inlk` は `links.json` よりは細かいが、結局 IntegralNotes 専用の解釈が必要で、viewer / 実行系 / UI の special case が残る。
- ハードリンクなら、source dataset が普通のファイル群として見えるため、Python 実行・標準表示・workspace 直閲覧との相性がよい。
- 一方で、書き込みが original data に波及する点は要注意なので、input dataset は read-only 契約で扱う前提を明確化したい。
