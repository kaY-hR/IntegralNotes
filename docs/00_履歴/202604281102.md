## 目的

仕様文書で決めた ID-backed slot I/O 方針を実装へ反映する。

## 実装内容

- `IntegralBlockDocument` から `outputConfigs` を削除し、`in:` / `out:` の slot map だけを canonical にした。
- 新規 `itg-notes` block の `out:` は slot ごとの既定 output path (`/Data/{slot}{ext}`) として生成するようにした。
- `in:` は renderer の picker で managed data ID を保存し、実行時にも path が残っていれば managed data ID へ正規化するようにした。
- Python 実行時は input ID を current path に解決し、output path は実行用 path として渡すようにした。
- 実行成功後、`out:` へは生成された dataset / managed file の ID を書き戻すようにした。
- 実行済み block は renderer 上で input/output を read-only にし、Run の代わりに Delete を表示するようにした。
- AI Chat の workspace tools に `resolveManagedDataByPath` と `resolveManagedDataById` を追加した。
