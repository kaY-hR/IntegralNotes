## 確認結果

- 正常に動く `lc_visualize.py` の input slot は `extensions: [".idts"]` を宣言している。
- 動かない2本は input slot に `datatype` だけを書いており、`extensions: [".idts"]` が無い。
- `Data/試料セット.idts` の dataset metadata は `datatype: null` だった。
- 現行 `src/renderer/integralCodeBlockFeature.tsx` の input picker は、dataset 候補を `datatype` 一致または slot `extensions` 一致で絞り込む。
- そのため、slot 側が `datatype` だけを要求し、対象 dataset 側の `datatype` が `null` の場合、`.idts` file path であっても候補から落ちる。

## 原因

- `.idts` は datatype ではなく bundle 表現なので、`.idts` dataset を受ける script input slot では `extensions: [".idts"]` を明示する必要がある。
- 問題の2本は「`.idts` を要求しているつもり」だが、実際の decorator metadata では `.idts` 表現を宣言していない。
