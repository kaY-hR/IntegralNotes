## 会話要約

- `[` による workspace file link 補完はすでに入っているが、`![` で任意 file を「リンク」ではなく「埋め込み preview」として貼りたい、という相談があった。
- 対象として
  - `png / jpg` など image
  - `html / svg`
  - `md / txt / json` など text
  - 表示非対応 file は対応する `data-note`
  を note 本文中で見られるようにしたい、という要件を整理した。
- 依頼に合わせて、先に `docs/Issues.md` へ `Issue 26` を起票し、そのまま実装まで進めた。

## 実装

- `docs/Issues.md`
  - `26. Milkdown で ![ から workspace file の埋め込み挿入と preview をしたい` を追加した。
  - 実装完了後、状態を `completed` に更新し、実装メモを追記した。
- `src/renderer/MilkdownEditor.tsx`
  - 補完 state を `link` と `embed` に分け、`![` 入力時も workspace file 候補 popup を出すようにした。
  - 候補選択時は `![](/path/from/cwd)` を Markdown image 記法として挿入するようにした。
  - 新しい workspace embed feature を editor 起動時に組み込むようにした。
- `src/renderer/workspaceEmbedFeature.tsx`
  - Milkdown の `image` / `image-block` node view を workspace embed 用に差し替える feature を追加した。
  - workspace path を解決して、
    - raster image は画像表示
    - `html` は `iframe srcDoc`
    - `svg` は iframe 表示
    - `text / markdown` は text preview 用 iframe
    - `unsupported` は対応する managed data があれば `data-note` preview
    へ振り分けるようにした。
  - 空の embed node でも、workspace path 手入力と既存 image upload hook 経由の画像選択を継続できるようにした。
- `src/renderer/styles.css`
  - editor 内 embed card, iframe, 画像, empty state の style を追加した。
