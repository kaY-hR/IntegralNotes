# 2026-05-09 22:17 履歴

## 背景

- `integral_image_compare.py` を `scripts/` に個別コピーする対応は、package 化した Python block / skill の共通 helper 管理として小手先だった。
- Python block と skill が共通利用する module は今後も発生するため、vendor package 側で version 管理しやすい共通領域が必要になった。

## 仕様合意

- package root に `shared/` を置く。
- Python block と skill が共通利用する helper module は `shared/` を正本にする。
- package stock 側では skill は `%LocalAppData%/IntegralNotes/packages/<packageId>/shared` を見る。
- workspace import 後の Python block は `.packages/<packageId>/shared` を見る。
- workspace import 時は `integral-package.json`, `scripts/`, `shared/` を copy する。
- `runtime-plugins/` と `skills/` は workspace へ copy しない。

## 対応

- `plugins/image-analysis-pack/shared/integral_image_compare.py` を追加し、`scripts/` 側の duplicate helper を削除した。
- `skills/image-analysis-tips/sdk/integral_image_compare.py` を削除し、skill scripts/templates/docs を `shared/` 前提に更新した。
- package import 処理で `shared/` subtree も workspace `.packages/<packageId>/shared` へ copy するようにした。
- Python callable runner が `.packages/<packageId>/scripts/*.py` 実行時に `.packages/<packageId>/shared` を `sys.path` へ追加するようにした。
- package stock に `shared/` があり、workspace import 側に `shared/` が欠けている場合は未 import 扱いにするようにした。
- 仕様文書を `shared/` contract に更新した。
