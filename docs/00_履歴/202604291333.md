# 2026-04-29 会話まとめ

## 目的

AI Chat panel と inline AI の両方で、LLM tool_call から承認付き CLI 実行を行えるようにする。

## 合意した仕様

- MVP の実体 shell は PowerShell 互換とする。
- AI Chat settings から shell executable path を指定できる。
- 未設定時は `pwsh` を優先し、なければ Windows PowerShell に fallback する。
- tool_call は `command` と `purpose` を必須にする。
- `workingDirectory` は workspace root 起点の相対 path に限定する。
- `env` と `stdin` は MVP では持たせない。
- 実行前に IntegralNotes 側の共通 dialog で command / purpose / timeout / warning を表示する。
- command は dialog 上で編集できる。
- 編集ありの場合だけ、LLM へ `executedCommand` を返す。
- reject はメッセージ付きで行え、LLM へ `status: "rejected"` と reason を返す。
- 危険そうな command、network / install 系 command は強警告にするが、許可されれば実行する。
- default timeout は 60 秒、LLM 指定可能、上限は 300 秒。
- stdout / stderr は LLM 返却時にそれぞれ 20,000 文字へ切り詰める。
- 実行中 cancel を可能にする。
- 実行後は成功/失敗/timeout/cancel に関係なく workspace sync 相当を走らせる。
- MVP は OS level sandbox ではなく、実行前確認と warning による guard とする。

## 実装内容

- 要求文書と AI Chat tooling 設計文書に CLI tool 仕様を追記した。
- `runShellCommand` typed tool を追加した。
- `AiHostCommandService` を追加し、approval request、PowerShell spawn、stdout/stderr streaming、cancel、timeout、post-command sync を担当させた。
- main/preload/shared API に host command approval / execution update / workspace synced IPC を追加した。
- `App` 直下に共通 approval dialog を追加し、inline AI / AI Chat panel のどちらからでも同じ dialog を使うようにした。
- AI Chat settings に PowerShell executable path 入力を追加した。
- AI Chat transcript の tool trace summary で `runShellCommand` の status / exit code / output 要約を表示できるようにした。
