# 2026-04-14 11:00

## 会話要約

- source dataset の擬似リンク表現として使っている `links.json` が、ユーザーにとって分かりにくく気持ち悪い、という指摘があった。
- 代替として、`{元ファイル名}.inlk` のような `1 file = 1 link` 形式や、可能なら `original-data` へのハードリンクで表現したい、という方向が出た。

## 実施したこと

- `docs/Issues.md` に Issue 13「source dataset の擬似リンク表現を `links.json` から見直したい」を追加した。
- Issue には以下の検討観点を残した。
  - `links.json` 廃止の是非
  - `.inlk` 形式の疑似リンク案
  - ハードリンク適用可否
  - 同一 volume 制約、directory 非対応、書き込み安全性、fallback 方針
