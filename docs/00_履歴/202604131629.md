## 背景

- explorer 上で `.chunk` が見えず `blob` だけ見えている構成に違和感があり、chunk を表に出したい要望があった。
- blob artifact についても、中身の見出しだけでなくファイル名そのものを `{元ファイル名}_{blob-id}.md` にしないと判別しづらい問題があった。
- 途中で Milkdown 系の前面表示バグ
  - drag handle
  - slash menu
  - placeholder (`[Please Enter...]`)
  が dialog を貫通する件も追加で観測されたが、まずは上の 2 件を片付ける方針に整理した。

## 実装したこと

### `.chunk` を explorer で可視化

- `src/main/workspaceService.ts`
  - `isVisibleEntry(...)` を調整し、ドット始まりでも `.chunk` ディレクトリだけは explorer に表示するよう変更

### blob artifact 名を `{元ファイル名}_{blob-id}.md` に統一

- `src/main/integralWorkspaceService.ts`
  - blob artifact note の実ファイル名を `{originalName}_{blobId}.md` に変更
  - blob summary が返す `artifactRelativePath` も同じ命名へ揃えた
  - ファイル名向け sanitization helper を追加
- `src/main/workspaceService.ts`
  - workspace 起動時に旧命名 `Artifacts/{blobId}.md` を新命名へ rename する migration を追加
  - blob metadata から新 artifact 名を再計算する helper を追加

### 設計メモの追随

- `docs/30_設計/20_ワークスペースレイアウト.md`
  - `Artifacts/BLB-....md` の例を `Artifacts/{元ファイル名}_{blob-id}.md` に更新

## 確認

- `npm run build` は成功
