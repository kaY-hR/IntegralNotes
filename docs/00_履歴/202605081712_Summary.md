# 2026-05-08 17:12 md 保存 dirty 解除の追加高速化

## 依頼

- md 更新後に `Ctrl+S` を押しても dirty が消えるのがまだ遅い。
- 直前の計画に沿って実装する。

## 対応

- `WorkspaceService.saveNote()` の完了条件を軽量化した。
  - `.md` 書き込み後の workspace mutation listener を background dispatch に変更。
  - 保存結果は書き込み済み content と `fs.stat()` から返し、保存後の `readNote()` 再読込を避けるようにした。
  - 明示的な `workspace:sync` では background mutation を flush するようにした。
- `RelationGraphService` の mutation 処理を debounce した。
  - mutation 受信時に即 full scan せず、250ms 後にまとめて処理する。
  - graph の明示的な snapshot / synchronize では pending incremental scan を捨て、full sync に一本化する。
- renderer の `saveNote()` 成功後に `refreshWorkspace()` を await する処理を削除した。

## 確認

- `npm run build` 成功。

## 注意

- dirty 解除は「Markdown のディスク書き込み成功」を基準にした。
- managed metadata / relation graph は background で追従するため、保存直後に数百ms程度の eventual consistency がある。
