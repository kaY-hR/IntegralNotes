# 2026-05-02 17:57 py block / @@ 自動保存と Undo 判定修正

## 依頼

- py block 実行後に自動保存する。
- `@@` による挿入の開始直後と終了直後に自動保存する。
- 解析実行が一部 output 未作成で失敗した場合でも、1 つでも output が生成済みなら実行済みとして Undo を出す。

## 対応

- `MilkdownEditor` に自動保存要求経路を追加し、次のタイミングで現在 Markdown を保存するようにした。
  - py block 実行結果を note に反映した直後
  - `@@` trigger 削除直後
  - `@@` 挿入内容を dispatch した直後
- `@@` の開始保存と終了保存が前後しないよう、editor 側で保存要求を queue 化した。
- `App.saveNote` が明示された Markdown を保存できるようにし、自動保存中にさらに編集された場合は新しい編集内容を上書きしないようにした。
- 実行済み block 判定を「全 output が実行済み」から「1 つ以上の output が実行済み」へ変更した。
- 実行済み判定で managed ID だけでなく workspace path / Markdown target からも generated output を解決するようにした。
- Python 実行後、複数 output の一部が未作成でも、存在している managed file metadata は作成してから missing output error を返すようにした。
- Undo 対象 metadata の照合で、block の `out:` が managed ID ではなく path のまま残っている場合も、metadata の id / path / dataset dataPath と照合できるようにした。
