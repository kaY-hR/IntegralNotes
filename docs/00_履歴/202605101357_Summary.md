# 2026-05-10 13:57 AI commit validation 設計の整理

## 背景

`??` auto-continue で renderer の最終 validation が止める状態になったが、ユーザーに validation error を出すのではなく、AI の commit tool error として返して AI に修正させるべきという指摘があった。

## 設計整理

- 共通 validator は `src/shared/markdownValidation.ts` に置き、保存時・AI commit 時・renderer final check で同じものを使う。
- AI commit の primary validation は main process の tool execution 内で行う。
- renderer は `??` / `@@...` trigger を削除した前後の Markdown source 差分から、Markdown source 上の置換 range を算出する。
- main process の `insertMarkdownAtCursor` は置換前の `documentMarkdown` と source range から candidate Markdown を作り、validator に通す。
- validate error は tool result として AI に返し、AI が再度 `insertMarkdownAtCursor` を呼ぶ。
- renderer final validation は invariant check として残す。通常ここで止まる場合は、candidate 生成または editor mapping の不整合として扱う。
