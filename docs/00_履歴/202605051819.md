# 2026-05-05 18:19 Codex global skill 除外と tool loop 上限引き上げ

## 依頼

- IntegralNotes で動かす LLM に、Codex 本体の global skill (`%UserProfile%/.codex/skills`) が入っているらしい。
- IntegralNotes と無関係の skill は呼んでも意味がないため、この仕様をなくしたい。
- 追加で、`@@continue` などが commit tool を呼ばずに止まる件は、tool call / step 上限を上げる方針にしたい。
- 2 件は別 commit として作業する。

## 対応 1: Codex global skill 除外

- `getGlobalSkillRootPaths()` を `getIntegralNotesGlobalSkillRootPaths()` に改め、返す root を `%LocalAppData%/IntegralNotes/skills` に限定した。
- AI Chat の available skill 一覧、`skill` tool 用の一時 merge directory、`implement-integral-block` fallback 読み込み対象から `%UserProfile%/.codex/skills` を除外した。
- AI Chat status / explicit skill instructions の表示文言を、単なる `global skills` ではなく `IntegralNotes global skills` と分かる表現へ寄せた。
- `docs/30_設計/32_GlobalPwd解析スクリプトとSkill.md` と `docs/30_設計/86_AIChatPanelとAgentTooling.md` に、Codex 本体の global skill は IntegralNotes の LLM に渡さない方針を明記した。
- commit: `c6048a7 Exclude Codex global skills from AI runtime`

## 対応 2: tool loop 上限引き上げ

- 通常 AI agent の `TOOL_LOOP_MAX_STEPS` を 8 から 50 に引き上げた。
- inline insertion / Python block / promptless continuation の明示 `maxSteps` も 50 に揃えた。
- `@@continue` は `TOOL_LOOP_PROMPTLESS_CONTINUATION_MAX_STEPS` を明示指定しているため、通常既定値だけでなく inline task 側の定数も更新した。
- `docs/30_設計/86_AIChatPanelとAgentTooling.md` の stop 条件説明を `stepCountIs(50)` に更新した。
- commit: `a57d281 Raise AI tool loop step limit`

## 確認

- 1 件目の commit 前に `npm run build` 成功。
- 2 件目の commit 前に `npm run build` 成功。
