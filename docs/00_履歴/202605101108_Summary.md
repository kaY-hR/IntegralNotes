# 2026-05-10 11:08 Markdown validation 実装

## 背景

Integral block の `id` や実行済み `out` ID がコピー&ペーストで重複すると、実行済み判定や provenance が壊れる可能性がある。

設計相談の結果、text 直接編集だけを特別扱いせず、Markdown note 保存時と AI commit 時に同じ Markdown validator を使う方針にした。

## 仕様更新

- `docs/10_要求/10_MVP要件.md`
  - Markdown note 保存時に note 全文を validate することを追記。
  - 同一 note 内の block ID 重複、実行済み output ID 重複を保存 error とすることを追記。
  - AI Markdown commit tool が保存時と同じ validator を使うことを追記。
- `docs/30_設計/10_ブロックJSON設計.md`
  - 保存時 validation 節を追加。
  - `out:` の workspace path は実行前の予定出力なので重複検査対象外とする方針を明記。
- `docs/30_設計/40_InlineAction.md`
  - `insertMarkdownAtCursor` は挿入後の Markdown 全文を validator に通し、NG なら note に反映せず tool result で error を返す方針を明記。

## 実装

- `src/shared/markdownValidation.ts` を追加。
  - `validateMarkdownDocument(markdown, { assetCatalog })` を実装。
  - `itg-notes` fenced block を抽出し、同一 note 内の `id:` 重複を error にする。
  - `out:` / `outputs:` が managed file ID または dataset ID を重複参照している場合に error にする。
  - `formatMarkdownValidationIssues()` を追加。
- `src/renderer/App.tsx`
  - `saveNote()` で `normalizeIntegralBlockInputReferencesInMarkdown()` 後、保存前に共通 validator を実行するようにした。
  - validation error 時は保存せず、status message に error 内容を出す。
- `src/main/aiChatService.ts`
  - `insertMarkdownAtCursor` tool の実行時に、挿入後の候補 Markdown 全文を共通 validator に通すようにした。
  - validation error 時は `{ validationStatus: "error", validationMessage, validationIssues }` を tool result として返し、挿入 text は返さない。
  - inline insertion / Inline Action / promptless continuation の Markdown commit 経路に適用した。
  - AI instructions に、validation error が返った場合は Markdown を修正して再度 `insertMarkdownAtCursor` を呼ぶよう追記した。
- `src/main/aiAgentService.ts`
  - terminal tool stop 条件を「tool call が出たら停止」から「成功扱いの terminal tool result が出たら停止」に変更。
  - `insertMarkdownAtCursor` の validation error result は tool trace 上も error として扱うようにした。

## 確認

- `npm run build` 成功。

## 補足

- validator の対象はまず active Markdown 全文に限定した。
- workspace 全体の block ID 重複検査や自動修復は今回入れていない。
