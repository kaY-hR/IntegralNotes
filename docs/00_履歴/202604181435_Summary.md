# 2026-04-18 14:35 作業サマリ

## 目的

- `original-data` 前提のレガシー実装を `src` から全撤去し、managed-file 中心モデルへ完全に寄せる。

## 実装内容

### shared / IPC 契約

- `src/shared/integral.ts`
  - `IntegralManagedDataEntityType` から `original-data` を削除。
  - `IntegralOriginalDataSummary` と `originalData` catalog を削除。
  - import 結果を `ImportManagedFilesResult` に変更。
  - dataset 作成 request/result を `CreateDatasetRequest` / `CreateDatasetFromWorkspaceEntriesRequest` / `CreateDatasetResult` に整理。
- `src/shared/workspace.ts`
  - API を `createDataset`, `createDatasetFromWorkspaceEntries`, `importManagedFileFiles`, `importManagedFileDirectories` に変更。
  - dataset manifest viewer の member 型を `managedFileId` 基準へ変更。

### main

- `src/main/integralWorkspaceService.ts`
  - `OriginalDataMetadata` 系を削除し、metadata の中核を `ManagedFileMetadata | DatasetMetadata` に整理。
  - import 処理を `importManagedFilePaths()` へ変更。
  - dataset manifest の `memberIds` は generic な managed file ID を持つように統一。
  - dataset materialize も managed file metadata から直接 alias を作るよう変更。
  - reconcile / tracking / dataset member cleanup から `original-data` 分岐を削除。
  - `registerManagedFilePath()` に一本化し、外部 file / folder 取り込みも managed-file metadata を直接作る形にした。
- `src/main/dataNote.ts`
  - `original-data` 向け data-note schema / parser / builder を削除。
  - data-note target は `managed-file | dataset` のみに整理。
- `src/main/workspaceService.ts`
  - data-note sync から `original-data` 分岐を削除。
- `src/main/main.ts`, `src/main/preload.ts`
  - IPC 名を新 API へ差し替え。

### renderer

- `src/renderer/App.tsx`
  - import / dataset 作成導線を新 API 名へ更新。
  - asset catalog の empty state から `originalData` を削除。
- `src/renderer/DataRegistrationDialog.tsx`
  - `ManagedFilePickerDialog` ベースへ更新。
- `src/renderer/IntegralAssetDialogs.tsx`
  - `ManagedFileSelectionDialog` / `ManagedFilePickerDialog` へ改名。
  - import callback も managed file summary ベースへ変更。
- `src/renderer/DatasetManifestFileViewer.tsx`
  - member key を `managedFileId` 基準へ変更。
- `src/renderer/integralCodeBlockFeature.tsx`, `src/renderer/integralPluginRuntime.ts`
  - catalog 初期値から `originalData` を削除。

## 確認結果

- `npm run build` は成功。
- `src` 配下を `rg` で確認し、`original-data` / `originalData` / `CreateSourceDataset` / `importOriginalData` / `source dataset` などの legacy 文字列は残っていない。

- 今回消したのは `src` 内のレガシー実装であり、docs には過去経緯として旧用語がまだ残る。
- 今後は「legacy 互換を残したまま寄せる」のではなく、managed-file / dataset の 2 軸で直接育てていける状態になった。
