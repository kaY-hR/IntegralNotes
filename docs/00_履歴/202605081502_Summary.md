# 2026-05-08 15:02 managed data reconcile 軽量化

## 依頼

- dataset 作成とファイル作成後の重さに対して、まず full reconcile 抑制と dataset 作成時の探索軽量化から手を付ける。
- full reconcile ではなく、`.idts` や `.md` の追加分だけ扱えばよいという方針。

## 対応

- `WorkspaceService.ensureWorkspaceReady()` の data-note 全件同期を、通常は workspace root ごと 1 回だけにした。
  - 明示 `syncWorkspace()` と `syncManagedDataNotes()` では従来どおり同期する。
- `IntegralWorkspaceService.ensureIntegralWorkspaceReady()` から毎回の `reconcileManagedDataMetadata()` を外した。
  - 起動時、workspace open 時、明示 workspace sync 時に `synchronizeManagedDataMetadata()` として full reconcile する。
- workspace mutation の create では、追加された `.md` を managed file として軽量登録し、追加された `.idts` は manifest が valid な場合だけ dataset metadata として登録するようにした。
- `createDatasetFromWorkspaceEntries()` で path ごとに metadata JSON 群を走査していた処理を、最初に `path -> managed file metadata` map を作る方式にした。
  - 未登録 path がある場合もまとめて import し、sync 回数を減らす。
- create mutation が newly registered metadata に当たっても full reconcile へ進まないようにした。

- `npm run build` 成功。
