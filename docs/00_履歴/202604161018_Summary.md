## 会話要約

- 拡張子ごとに専用 viewer を plugin できる統合 plugin system を実装してほしい、という依頼があった。
- plugin system は 1 つのまま、`viewers` contribution を add する形で進めた。
- 既存 plugin への影響は additive に抑え、`plugin viewer -> built-in viewer -> unsupported` の順で解決する方針を実装した。

## 実装

- `src/shared/plugins.ts`
  - `viewers` contribution、viewer renderer message、viewer metadata 型を追加した。
  - manifest parse を `blocks` または `viewers` のどちらかを持てば成立する形へ広げた。
- `src/main/pluginRegistry.ts`
  - viewer ごとの renderer entry を解決できるようにし、`loadViewerDocument(pluginId, viewerId)` を追加した。
  - block を持たない viewer-only plugin も registry に残るようにした。
- `src/main/preload.ts`, `src/main/main.ts`, `src/shared/workspace.ts`
  - `loadPluginViewerDocument()` を IPC / preload / shared API に追加した。
- `src/main/workspaceService.ts`
  - `.md` 以外の workspace file について、拡張子に一致する plugin viewer を優先解決するようにした。
  - plugin viewer 用の file payload を `text` または `data-url` として返せるようにした。
- `src/main/integralWorkspaceService.ts`
  - dataset inspection / renderable 判定でも同じ viewer registry を使うようにした。
  - built-in 対象外でも plugin viewer があれば `renderables` に含めるようにした。
- `src/renderer/ExternalPluginFileViewer.tsx`
  - viewer iframe を generic に扱う renderer component を追加した。
- `src/renderer/WorkspaceFileViewer.tsx`, `src/renderer/IntegralAssetDialogs.tsx`
  - `kind === "plugin"` の file/renderable を新 component で表示するようにした。
- `src/renderer/workspaceEmbedFeature.tsx`
  - plugin viewer 対象 file の note embed は、明示的に未対応メッセージへ流すようにした。
- `src/renderer/PluginManagerDialog.tsx`, `src/renderer/styles.css`
  - plugin manager に viewer 数表示を追加し、plugin iframe style を通常 viewer でも使えるようにした。
