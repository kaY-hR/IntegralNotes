## 背景

- Milkdown の浮遊 UI は suppress で解消したが、chunk viewer 内クリックで block 自体が node selection され、全体が灰色に見える問題は残っていた。
- CSS だけではユーザー体感が変わらなかったため、viewer 内の pointer event を editor selection に渡さない方向へ切り替えた。

## 実装したこと

- `src/renderer/integralCodeBlockFeature.tsx`
  - `stopEvent(...)` の対象を拡張し、以下の領域で発生した event は ProseMirror 側へ渡さないようにした
    - `.integral-renderable-layout`
    - `.integral-renderable-card`
    - `.integral-renderable-card__image`
    - `.integral-renderable-card__text`
    - `iframe`
- これにより、chunk viewer の中身をクリックしたときは renderable viewer の操作を優先し、block 全体の node selection に落ちにくくした

- `npm run build` は成功
