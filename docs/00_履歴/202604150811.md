## 会話要約

- 非表示フォルダ配下の data への Markdown link が開けないため、`.store` など tree に出していない path でも jump できるようにしたい要望があった。

## 実施したこと

- `src/renderer/MilkdownEditor.tsx` の link click 処理を見直した。
- これまでは `workspaceEntries` に含まれる visible file だけを open 対象にしていたため、非表示フォルダ配下の link は click 時点で弾かれていた。
- click 前の visible-path 判定を外し、workspace relative path に解決できた link はそのまま `onOpenWorkspaceFile()` へ流すように変更した。
- これにより `.store/...` など explorer 非表示 path でも、`readWorkspaceFile()` で開ける file は app 内 viewer / editor へ遷移できるようになった。

- `npm run build` が通ることを確認した。
