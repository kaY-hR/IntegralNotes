## 背景

- `general-analysis` / `core-display` の初回実装後、Python Script 登録 UI と chunk viewer の操作系を新設計に合わせて整理する必要があった。
- 特にユーザー要望として、
  - Entry 選択をラベル直下の単一ボタンにまとめること
  - 同梱ファイル追加ボタンを同梱ファイル欄に置くこと
  - input/output slot を行単位で個別追加できること
  - chunk viewer を `select chunk + simple flex-layout` に寄せること
  が明示された。

## 実装したこと

### Python Script 登録

- `src/renderer/PythonScriptDialog.tsx`
  - Entry を「表示兼選択」の単一 picker button に変更
  - 追加ファイルボタンを bundled files セクションに移動
  - Input Slots を `[slot名] [chunk選択]` の複数行編集に変更
  - Output Slots を `[slot名]` の複数行編集に変更
  - 登録時に block markdown を組み立て、選択済み chunk を初期入力に入れた状態で返すよう変更

### editor 連携

- `src/renderer/integralSnippetMenu.ts`
  - block markdown 挿入用 event を追加
- `src/renderer/MilkdownEditor.tsx`
  - active editor のみが block markdown 挿入 event を受けて本文へ挿入するよう変更
- `src/renderer/App.tsx`
  - Python script 登録完了後、
    - dialog を閉じる
    - catalog を更新する
    - 新規 block をアクティブ editor に挿入する
    処理を追加

### chunk viewer

- `src/renderer/integralCodeBlockFeature.tsx`
  - `core-display/chunk-view` は input slot 一覧を前面に出さず、`select chunk` ボタン主体の単純 UI に変更
- `src/renderer/IntegralAssetDialogs.tsx`
  - renderable 表示を `flexlayout-react` のシンプルな tab layout ベースに変更
- `src/renderer/styles.css`
  - 上記 UI 用の picker / row editor / display toolbar / flex layout theme を追加
