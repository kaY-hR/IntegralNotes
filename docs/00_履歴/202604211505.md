# 2026-04-21 15:05 AI Chat runtime fallback / render readiness / MCP表示整理

## 背景

- ユーザーから `Error invoking remote method 'ai-chat:submit': Error: AI agent returned no assistant text.` が出ると報告あり。
- `renderWorkspaceDocument` は success でも、`LC-plot.html` のような描画済み HTML chart を assistant が見えていないように振る舞うケースがあった。
- AI Chat UI では `MCP ready (design)` のように見えていたが、実際の generic MCP client registry / add UI は未実装で、誤解を招いていた。

## 実装

- `src/main/aiAgentService.ts`
  - `ToolLoopAgent` の最終 text が空でも、tool trace が残っていれば例外ではなく fallback assistant text を組み立てて返すように変更。
  - agent instructions に「tool 使用後も final answer を空にしない」旨を追加。
  - `renderWorkspaceDocument` の tool model output に `renderReadiness` を含めるように変更。
  - tool trace 判定で `renderReadiness` が `timeout:*` の場合は `error` 表示に落とすよう変更。
  - tool trace summary に render readiness を表示するよう変更。

- `src/main/workspaceVisualRenderService.ts`
  - rendered screenshot output に `renderReadiness` を追加。
  - fixed sleep のみだった待機をやめ、hidden `BrowserWindow` 上で
    - `document.readyState`
    - `document.fonts`
    - Plotly container (`.js-plotly-plot`)
    - `svg/canvas/img/...` の non-zero rect
    を見て visual readiness を待つように変更。
  - capture 前に resize settle (`window resize + 2x requestAnimationFrame`) を入れて、chart 描画直後の撮影を少し安定化。

- `src/main/aiChatService.ts`
  - `mcpEnabled` を `false` に変更。
  - status notes に「MCP client registry はまだ未実装、add / enable UI もまだ無い」と明記。

- `src/renderer/AIChatPanel.tsx`
  - header pill を `MCP connected / not wired` 表記に変更。

## 確認

- `npm run build` 成功。

## 補足

- `Notes/Data/LC-plot.html` は inline の Plotly bundle を含んでおり、少なくとも「外部 CDN が無いので絶対に描画できない」ケースではなかった。
- そのため今回の主眼は、MCP ではなく `render ready を待たずに capture していた可能性` と `空 assistant text を例外にしていた実装` の是正。
