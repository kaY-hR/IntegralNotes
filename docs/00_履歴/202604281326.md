# 2026-04-28 13:26 会話まとめ

## 目的

inline AI 挿入 (`??`) を、場所が違うだけで AI Chat パネルと同等の workspace agent として扱えるようにする。

## 問題

- `??` の AI 挿入は `useWorkspaceTools: false` のままだった。
- そのため LLM へ渡る context でも `workspace tools mounted: no` となり、モデルが「ファイルの完全な内容を読み込めない」と判断していた。
- 前回追加した `insertPythonBlock` tool が共通 workspace tools 側に入っており、通常 AI Chat / AI 挿入にも露出し得る状態だった。

## 実装内容

- `??` の AI 挿入でも `useWorkspaceTools: true` に変更した。
- AI 挿入では AI Chat パネル相当の workspace tools に加えて、挿入確定用の `insertMarkdownAtCursor` tool だけを追加する形にした。
- `insertPythonBlock` tool は共通 workspace tools から外し、`>>` の inline Python block popup 専用の追加 tool として渡す形へ整理した。
