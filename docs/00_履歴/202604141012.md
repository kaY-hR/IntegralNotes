## 会話要約

- `raw-data` よりも、日本語の感覚としては `元データ` の方がしっくりくる、という話になった。
- `元データ` に自然に対応する英語命名候補を再検討した。

## 判断

- 第一候補は `original data` とした。
  - `raw data` ほど「未加工」であることを強く要求しない
  - 外部で加工済みの CSV / text を取り込む場合でも、「この workspace における元データ」として扱いやすい
  - `dataset` と対比しても意味が崩れにくい
- `source data` は意味としては近いが、
  - source code
  - source chunk / source bundle
  との混線が起きやすいので優先度を下げた。
- `imported data` は operation 名としては自然だが、概念名としては少し弱いと整理した。

## 推奨案

- `blob` -> `original-data`
- `chunk` -> `dataset`
- `artifact note` -> `data-note`
- `Artifacts/` -> `data-catalog`

## 補足

- `original data` は mass noun なので、singular entity を強く出したい箇所だけ
  - `original data item`
  - `data entry`
  などの補助語を使う余地がある。
