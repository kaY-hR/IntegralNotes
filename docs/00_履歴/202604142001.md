# 2026-04-14 20:01

## 会話要約

- `workspace file` link 補完で `[label](/path)` は挿入されるが、Milkdown 上で link として描画されない問題が見つかった。

## 実施したこと

- `src/renderer/MilkdownEditor.tsx` を修正した。
  - 補完時の挿入を `replaceRange(markdown)` ではなく、Milkdown の `linkSchema` を使った `link mark` 直接生成へ切り替えた
  - 挿入後は cursor を link 末尾へ移すようにした

## 検証

- `npm run build` を実行し、renderer / main ともに build が通ることを確認した。

## 補足

- この修正により、補完で選んだ直後から WYSIWYG 上でも anchor として描画される想定。
- file 保存後に再オープンした場合の parse 経路は従来どおり markdown parser に依存する。
