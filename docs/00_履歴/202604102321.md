## 背景

- export 関連ファイルは repo 直下より `scripts` 配下に集約した方が分かりやすい、という整理方針が出た。
- あわせて、共有先で使う `.copyexport-manifest` も root ではなく `scripts` 配下に置きたい要望があった。

## 実施内容

- `scripts/.copyexclude`
  - 除外定義ファイルを repo 直下から `scripts` 配下へ移動した。
  - 除外対象の自己参照も `scripts/.copyexclude`, `scripts/copy-git-files.bat`, `scripts/export-tracked-files.ps1` に更新した。
- `scripts/copy-git-files.bat`
  - 実行用 bat を repo 直下から `scripts` 配下へ移動した。
  - bat は PowerShell 実装の入口だけを担い、本処理は `export-tracked-files.ps1` に委譲する構成を維持した。
- `scripts/export-tracked-files.ps1`
  - `.copyexclude` の参照先を `scripts\.copyexclude` に変更した。
  - manifest の保存先を共有先 root の `.copyexport-manifest` から `scripts\.copyexport-manifest` に変更した。
  - 旧 root manifest が存在する場合は、新しい `scripts\.copyexport-manifest` に切り替えた上で旧ファイルを削除する migration を追加した。
- repo 直下
  - `.copyexclude`
  - `copy-git-files.bat`
  - は削除した。
