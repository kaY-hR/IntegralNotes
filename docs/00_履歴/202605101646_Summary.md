# 2026-05-10 16:46 AI commit validation error の tool result 化

## 背景

Inline Action / `??` auto continue で Markdown validation error は main process の commit tool 内で検出できるようになっていたが、成功 commit がない場合に handler 側が error を throw していた。

そのため、validation error が AI への tool result として再試行されず、ユーザー向け popup error / IPC error として surfaced していた。

## 方針

- Markdown を変更する tool は tool 内で validator を実行する。
- validation NG の場合、note / file へ反映せず、tool result を error 扱いにする。
- terminal tool は「呼ばれたら停止」ではなく「成功したら停止」とする。
- validation NG の tool call は agent loop を止めず、AI に修正版 commit を試させる。
- AI が成功 commit せず終了した場合は、popup を閉じずに会話履歴と tool trace を残す。
