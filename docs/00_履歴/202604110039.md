## 背景

- 当初は「ワークスペース未設定時に自動でフォルダ選択ダイアログを出す」案が出たが、途中で方針を見直した。
- 最終的には、自動ダイアログではなく、未設定時のエクスプローラーパネルを空状態にして `フォルダを開く` ボタンを表示する案で進めることになった。
- あわせて、flex-layout 背景に表示される `Open first note` / `New note` の操作導線は不要という判断になった。

## 実施内容

- `docs\00_履歴\202604110035_UserReq.md`
  - 今回の UI 要望を `UserReq` として整理した。
- `src\main\workspaceService.ts`
  - ワークスペース未設定状態を許容するように変更した。
  - 初期値が未設定なら `null` 相当で待機し、明示的にフォルダが選ばれたときだけワークスペースを確定する構成にした。
  - ワークスペース必須の API では、未設定時に明示的なエラーを返すようにした。
- `src\main\main.ts`
  - 既定で `Documents\IntegralNotes` を自動採用しないように変更した。
  - ワークスペース未設定時はウィンドウタイトルを `IntegralNotes` のみで表示するようにした。
  - フォルダ選択ダイアログの既定パスは、現在のワークスペースがあればその場所、なければ `Documents` にした。
- `src\shared\workspace.ts`
  - `getWorkspaceSnapshot()` が未設定時に `null` を返せる型へ変更した。
- `src\renderer\App.tsx`
  - ワークスペース未設定時はサイドバーを空状態にし、説明文と `フォルダを開く` ボタンを表示するようにした。
  - ワークスペース未設定時はサイドバー上部の `New note / New folder / Sync` を出さないようにした。
  - 中央 placeholder の `Open first note` / `New note` ボタンを削除し、状態説明だけを表示するようにした。
- `src\renderer\styles.css`
  - サイドバー空状態 UI のスタイルを追加した。
  - 不要になった placeholder action 用スタイルを削除した。
