## 会話要約

- viewer plugin system は、当面は拡張子ベースだけでよい、という方針に整理された。
- 拡張子ごとに専用 viewer を plugin できる仕組みが実装可能か、という確認があった。

## 整理内容

- 実装は可能。
- 既存の external plugin には `renderer` と `iframe` 読み込み基盤があるため、それを viewer 側にも転用できる。
- 追加するなら plugin manifest に viewer contribution を足し、`extensions: [".foo", ".bar"]` から viewer を引ける registry を main 側で持つ構成が自然。

## 想定する主な変更点

- `src/shared/plugins.ts`
  - plugin manifest に viewer contribution 型を追加する。
- `src/main/pluginRegistry.ts`
  - installed plugin の viewer 一覧を解決できるようにする。
- `src/main/workspaceService.ts`
  - workspace file open 時に拡張子から plugin viewer を解決する。
- `src/main/integralWorkspaceService.ts`
  - dataset inspection でも renderable file 判定の代わりに viewer candidate 解決を使えるようにする。
- `src/renderer/WorkspaceFileViewer.tsx`
  - built-in viewer の前に plugin viewer を描画できるようにする。
- `src/renderer/IntegralAssetDialogs.tsx`
  - dataset 内 file の表示でも plugin viewer を使えるようにする。

## 補足

- first version は extension-only に限定した方が実装も仕様もかなり単純。
- `.json` や `.md` の schema/frontmatter 判定は後から viewer matcher を拡張すればよい。
