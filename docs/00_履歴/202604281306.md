## 目的

inline AI 挿入 (`??`) も、inline Python block 生成 (`>>`) と同じく popup 内チャットとして扱えるようにする。

## 背景

- `>>` は popup chat 化済みだったが、`??` は単発の挿入テキスト生成のままだった。
- AI挿入では、Markdown本文とカーソル位置を踏まえて、ユーザーが望む内容をその位置へ挿入する必要がある。
- 応答品質改善のため、system prompt / task prompt で「cursor 前後の Markdown」と「挿入は tool 経由」を明確化する必要があった。

## 実装内容

- `SubmitInlineAiInsertionRequest/Result` を popup chat 用に拡張した。
- `insertMarkdownAtCursor` tool を追加し、AI が挿入確定時に exact Markdown を tool call で返す形にした。
- `??` popup でも複数ターン履歴を表示し、追加質問に回答できるようにした。
- AI挿入の prompt を、`Markdown before cursor` / `[CURSOR]` / `Markdown after cursor` を明示する形へ変更した。
- AI挿入でも inline session を既存 AI Chat history に永続化するようにした。
