# 2026-05-09 12:42 link 挿入直後・dataset note link・Backspace 削除修正

## 背景

ユーザーから次の 3 点が報告された。

- link picker で link を挿入した直後に文字入力すると、続きの文字が link 文面に含まれる。
- `.idts` 作成時に自動生成される data-note の link が、括弧を含む path で壊れる。
- 画像や Integral block が、カーソル直後から Backspace で消しにくい。

## 原因

- link 挿入後の ProseMirror stored marks が link mark を保持し、次の入力に link mark が継続していた。
- dataset data-note の Markdown link target 生成で `)` だけを `%29` にしており、`(` が未エンコードのまま残って括弧の対応が崩れていた。
- Milkdown 標準の隣接 node 削除に任せるだけでは、custom node view / atom block の隣接削除が期待通りに働かないケースがあった。

## 変更

- `src/renderer/MilkdownEditor.tsx`
  - link picker 挿入後に `transaction.setStoredMarks([])` を呼び、次の入力が link mark の外になるようにした。
  - `Backspace` / `Delete` で、選択中またはカーソル隣接の image / embed / Integral block を削除する補助処理を追加した。
  - 入力欄、button、select などの UI 操作中は削除補助を発火しないようにした。
- `src/main/dataNote.ts`
  - data-note の Markdown link target 生成で `%`、空白、`(`、`)` をエンコードするようにした。

## 確認

- `npm run build` 成功。
