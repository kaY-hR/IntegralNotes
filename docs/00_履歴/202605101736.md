# 2026-05-10 17:36 Python block script の hidden path 防止

## 背景

package 由来 script を参考に Python 解析 block を作らせると、AI が変更版 script まで `.packages` 配下へ作成してしまうことがあった。

`.packages` は hidden/system-managed path であり、通常の Python block discovery / `>` 挿入の導線では新規生成 script として扱えないため、変更版 script は visible path に置く必要がある。

## 変更

- `Notes/.codex/skills/implement-integral-block/scripts/validate_integral_block.py`
  - 対象 script path の各 component を確認し、`.` 始まりの hidden/system path を error にした。
- `src/main/aiChatService.ts`
  - `insertPythonBlock` tool で script path / callable を validation し、hidden path や未保存 / discovery 不可の callable を tool result error として返すようにした。
  - `createPythonBlockDraft` tool でも hidden path を validation error として返すようにした。
  - `@@make-python-block` / `@@amend-python-block` / `>>` / `@@` 系 prompt に、生成・変更版 script は `scripts/ai_blocks/` など visible path へ置く指示を追加した。
- `src/main/aiAgentService.ts`
  - `insertPythonBlock` / `createPythonBlockDraft` の validation error を tool trace 上も error として扱うようにした。
- `src/shared/aiChat.ts`
  - 既定 system prompt に hidden/system path 配下へ生成しない指示を追加した。
- `Notes/.inline-action/*.md` と `Notes/.codex` / `Notes/.claude` の `implement-integral-block` skill に同じ注意を追加した。
- `docs/30_設計/40_InlineAction.md`
  - make/amend Python block と `createPythonBlockDraft` の hidden path validation 方針を追記した。

## 確認

- `python Notes/.codex/skills/implement-integral-block/scripts/validate_integral_block.py scripts/demo_hello_report.py scripts/demo_dataset_report.py` 成功。
- hidden path の検出は `Notes/.codex/.../validate_integral_block.py` を対象にした validator 実行で `.codex` component を error として検出することを確認した。
