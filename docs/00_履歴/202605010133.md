# 2026-05-01 01:33 py block execution log persistence

## 背景

- Python block の実行時に stdout / stderr 由来の warning や error が block 下の一時 UI には出るが、Markdown source には残らず、LLM から参照しづらかった。
- 仕様では失敗時に `integral-error` fenced code block として block 直下へ反映してよいことになっている。
- fenced code block で全文を note に入れると本文が長くなりすぎるため、log file として永続化し、note には `![]()` で埋め込む方針にした。

## 対応

- `src/main/integralWorkspaceService.ts` で、Python block の stdout / stderr / execution error を `analysis-result/integral-block-logs/{blockId}.log` に保存するようにした。
- log file は managed file metadata に登録し、note には `![](/analysis-result/integral-block-logs/{blockId}.log)` として挿入する。
- 実行失敗も IPC exception ではなく structured result として返し、log embed を note へ反映できるようにした。
- `src/renderer/MilkdownEditor.tsx` では、block 直後の既存 `integral-error` / `integral-log` fenced block または既存 log embed を置換対象にした。
- 実行ログが空の成功時は log file / embed を作らない。

## 確認

- `npm run build`
  - 成功。
  - Vite CJS API deprecated warning と chunk size warning は既存系の警告として残る。
