  - CSV/TSV/JSON などの機械可読・中間出力は別 output slot にし、基本的に `auto_insert_to_work_note` は省略または `False` にする。

## 2026-04-28 02:07 追記

- `.codex` / `.claude` が無い新規 cwd を開いたときに skill folder が生成されない問題を修正した。

- 原因
  - 従来の bootstrap は、開いた workspace 内に `Notes/.codex/skills` / `Notes/.claude/skills` が既に存在する場合だけ、それを cwd 直下の `.codex/skills` / `.claude/skills` へコピーしていた。
  - そのため、任意の新規 cwd では source skill folder が存在せず、何も配置されなかった。

- 修正
  - `src/main/workspaceService.ts` の skill bootstrap source を、workspace 内 legacy source とアプリ同梱 template の二段探索にした。
  - 開発中は repo の `Notes/.codex/skills` / `Notes/.claude/skills` を template として使う。
  - package 後は `process.resourcesPath/workspace-skills/...` を template として使う。
  - `package.json` の `extraResources` に Codex / Claude skill template を追加した。

- 検証
  - `npm run build` 成功。
  - `git diff --check` は whitespace error なし。Git の CRLF 変換 warning のみ。
