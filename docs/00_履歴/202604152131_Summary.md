# 2026-04-15 21:31 Summary

## 会話要約

- `data-note` を「managed data を開いたとき当然見られるもの」にしたい、という UX 相談があった。
- 当初は tab 内分割案もあったが、検討の結果、VS Code の preview に近い感覚で「managed data 側から対応する `data-note` を別タブで開く」方針がよい、という整理になった。
- 具体的には、managed data を app 内で見ているときに右上へ `[ノートを開く]` 相当の action を出し、クリックで対応する `data-note` tab を開く方向を採ることになった。

## 対応

- `add-issue` スキルを使い、`docs/Issues.md` へ新規 Issue を追加した。
- 追加した Issue は `25. managed data を見ている文脈から data-note を別タブで開けるようにしたい`。
- 内容としては、以下を主論点にまとめた。
  - original data / dataset のどちらでも一貫して note を開けること
  - renderable / unsupported を問わず同じ導線で扱うこと
  - hidden path 直参照ではなく `targetId` 起点で note tab を開くこと
  - user-facing 文言や tab 名は hidden file 名ではなく data の表示名を優先すること

## 補足

- Issue 24 が `data-note を entity 起点で扱うための基盤整理` なら、今回の Issue 25 は `viewer / tab UX をどう見せるか` の切り出しにあたる。
