# 2026-04-14 11:03 Summary

## 会話要約

- `dataset` に対応する `data note` があってもよいのでは、という案が出た。

## 整理した判断

- この案は自然で、`data note` を `original data` 専用概念ではなく「人間向けに文脈・preview・補足を書くための面」と再定義できる。
- そうすると、
  - `original data`
  - `dataset`
  の両方に 1:1 対応の `data note` を持たせる設計が見えてくる。

## 利点

- データ種別に関係なく「説明は note にある」という一貫した体験になる。
- dataset 側にも
  - 何を意味する中間結果か
  - どう解釈すべきか
  - 代表グラフや preview
  を残せる。
- provenance は machine-readable に `dataset.json`、人間向け説明は `data note` に分担できる。

## 注意点

- `original data note` と `dataset note` の違いを UI で曖昧にしすぎると、元データと解析結果の区別が薄れる。
- そのため概念としては `data note` を共通化しても、対象が
  - original data なのか
  - dataset なのか
  は明示した方がよい。
