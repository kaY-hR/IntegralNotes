# 2026-05-08 23:40 bash unsupported command hint

## 依頼

- `bash` tool でブラウザ環境では実行できない command に失敗したとき、AI に `runShellCommand` を使うよう reaction として教えたい。
- 全失敗ではなく、未対応系の失敗に限定する。

## 対応

- `src/main/aiAgentService.ts` の `createBashTool()` に `onAfterBashCall` を追加した。
- `bash` の `exitCode !== 0` かつ未対応系 error と判定できる場合だけ、stderr に `runShellCommand` への hint を追記するようにした。
- 判定対象は `command not found`、`not available in browser environments`、Node.js bundle 誘導、browser / sandbox / just-bash 文脈での unsupported / unavailable 系に限定した。
- `buildBashToolInstructions()` にも、browser/sandbox で unavailable / unsupported の場合は `runShellCommand` を使うよう短い指示を追加した。

## 確認

- `npm run build` 成功。

## 注意

- `rg` no match や通常の test failure など、単なる nonzero exit には hint を追加しない方針。
- `node_modules` は変更していない。
