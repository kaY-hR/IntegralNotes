# 2026-05-03 21:00 Integral SDK hidden 配置化

## 依頼

- `integral` SDK が visible な `scripts/integral/` に置かれ、managed metadata JSON の管理対象にもなりうるのは不自然ではないか、という指摘。

## 対応

- workspace SDK の同期先を `.integral-sdk/python/integral/` に変更した。
- app runner / editor 補助設定の import root を `.integral-sdk/python/` に変更した。
- root `scripts/integral/` と template `Notes/scripts/integral/` を hidden な `.integral-sdk/python/integral/` へ移した。
- 旧 `scripts/integral/` と新 `.integral-sdk/` を system-managed SDK path として扱い、自動 managed file 登録・手動登録・Python callable discovery から除外するようにした。
- 既存 metadata JSON が SDK path を指している場合は reconcile 時に metadata を削除し、data-note 同期で orphan note も整理されるようにした。
- SDK 参照を持つ template skill、`.vscode/settings.json`、設計文書を新配置へ更新した。
- root の demo Python script は直接 import 確認用に `.integral-sdk/python/` を `sys.path` へ追加する bootstrap を入れた。
- block 実装 skill と `>>` / fallback prompt に、`.integral-sdk/python/` は hidden/system-managed SDK import root であり、block 実装時に `.integral-sdk` 配下を作成・変更しない、という指示を追加した。

## 確認

- `npm run build` 成功。
- `python -c` で root hidden SDK import と `scripts/demo_hello_report.py` の module load が成功。
- prompt 追記後も `npm run build` 成功。
