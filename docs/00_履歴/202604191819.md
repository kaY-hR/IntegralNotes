## 概要

- 解析 block の UX 改善について、DSL 化ではなく「GUI 編集モード」と「YAML テキスト編集モード」の切替案を中心に実現性を検討した
- 現行実装を確認した結果、保存形式は既存 YAML のまま維持し、block 内テキスト編集に popup 補助を付ける方針が最も安全と判断した

## 確認したこと

- `src/renderer/integralCodeBlockFeature.tsx`
  - 解析 block は Milkdown の custom node view (`atom`, `isolating`) として実装されている
  - `stopEvent()` では `textarea`, `input`, `button` などを許可しており、block 内に独自 textarea を置く実装は可能
- `src/renderer/styles.css`
  - `integral-code-block__editor-shell` / `integral-code-block__editor` の style が既にあり、block 内 text editor UI を載せる前提と相性がよい
- `src/renderer/App.tsx`
  - 既存の `F2` は explorer sidebar rename 用だが、sidebar が focus されている場合だけ反応するため、editor 内 block 用 `F2` とは衝突しにくい
- `src/renderer/simpleYaml.ts`
  - 現行 YAML parser は位置情報を返さないため、popup 補助の文脈判定は AST ではなく caret 行ベースの軽量判定で行うのが現実的

## 設計判断

- 推奨方針
  - 既存 GUI は残す
  - 解析 block にだけ一時的な `テキスト表示` モードを追加する
  - text mode では DSL を新設せず、実 YAML を編集する
  - popup 補助は `in.<slot>`, `out.<slot>.dir`, `out.<slot>.name` に対して出す
- 想定操作
  - `>` から解析 block を挿入した直後は text mode で開く
  - GUI の `テキスト表示` ボタン、または block 上で `F2` でも text mode に入れる
  - block から focus が外れたら、draft が valid な場合のみ YAML を commit して GUI に戻す
  - invalid YAML の場合は text mode を維持して error を出し、自動では戻さない

## 実装上の注意

- text mode は keystroke ごとに即 commit せず、block 内 local draft を持つ必要がある
  - 途中入力では一時的に invalid YAML になるため
- input popup は workspace file 候補を拡張子で絞って表示する
- output popup は 2 段階にする
  - `dir`: cwd 配下の folder 一覧
  - `name`: 推奨名 (slot 名) を含む file name 候補
- popup は Milkdown 全体の `>` 補完を流用せず、block 内 textarea 専用で持つ方が実装しやすい
