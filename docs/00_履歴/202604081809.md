# 2026-04-08 18:09 作業ログ

## 依頼

- `npm run dev` をもう一度ターミナルから実行したら `Port 5173 is already in use` で落ちた
- 再実行しても落ちにくいようにしたい

## 原因

- 以前の dev 実行で起動した Vite サーバーが `127.0.0.1:5173` を保持したまま残っていた
- 既存の `package.json` では `concurrently -k` を使っていたため、renderer がポート衝突で落ちると main/electron もまとめて停止していた

## 対応

- `package.json`
  - `concurrently -k` を `--kill-others-on-fail` に変更
  - `dev:renderer` を専用 helper 経由に変更
  - `dev:electron` の `wait-on` を `tcp` から `http` に変更
- `src/dev/start-renderer.cjs`
  - `127.0.0.1:5173` に既存の HTTP サーバーがいれば再利用して正常終了
  - 空いている場合だけ Vite を起動
  - ポートは開いているが HTTP 応答でなければ、誤接続防止のため異常終了

## 検証

- `npm run dev:renderer` 実行結果:
  - `[dev:renderer] Reusing existing renderer on http://127.0.0.1:5173`
- `npm run build` 成功

## メモ

- これで、既存の Vite が残っている状態でも `npm run dev` の再実行で renderer が即死しにくくなった
