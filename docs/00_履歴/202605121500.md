# 2026-05-12 15:00 shared data-note 反映調査

## 背景

- `動画B_クロマト版/Notes/scripts/plot_chromatogram.py` で `embed_to_shared_note=True` を指定しているのに、結果が挿入されないように見えるという報告を受けた。
- 対象 script の output slot には `auto_insert_to_work_note=True` / `share_note_with_input="source"` / `embed_to_shared_note=True` が設定されていた。

## 確認

- 直近実行 `BLK-4AB53844` は `Data/html_UTW.html` を生成し、作業 note `20260512.md` 直下にも `![](/Data/html_UTW.html)` が追記されていた。
- output metadata `FL-9F5B79CB.json` は `noteTargetId: "FL-AC3513B3"` を持ち、source data-note `data-notes/FL-AC3513B3.md` 末尾にも provenance link と HTML embed が追記済みだった。
- したがって main process の保存処理自体は成功しており、開いている data-note tab が実行後に再読込されないため、UI 上で未挿入に見える可能性が高いと判断した。

## 対応

- `ExecuteIntegralBlockResult` に `updatedReferenceFiles` を追加した。
- `appendProjectedOutputsToDataNotes` が追記した data-note path を返すようにした。
- block 実行成功時に、shared data-note へ追記した path を renderer へ返すようにした。
- renderer 側で `updatedReferenceFiles` があれば、開いている非 dirty tab をディスクから再読込するようにした。
