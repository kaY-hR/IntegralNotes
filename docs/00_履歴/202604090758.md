## 概要

- 「独自 UI 上のボタンから、将来的に装置操作や解析を実行したい」という前提を受け、preview ベース実装から node view ベース実装へ切り替えた
- Markdown 上は引き続き `json` code block を保持しつつ、表示時は interactive block として描画し、ボタン押下は preload 経由で main process に渡す構成に変更した

## 実施した対応

- `src\renderer\integralCodeBlockFeature.tsx`
  - `code_block` node view を上書きし、Integral JSON を interactive UI として描画する custom node view を追加
  - `JSON編集` と `UI表示` を切り替えられるよう実装
  - ボタン押下時に `window.integralNotes.executeIntegralAction(...)` を呼ぶよう実装
- `src\renderer\integralBlockRegistry.tsx`
  - `type` ごとの block 定義を追加
  - `LC.Method.Gradient`
  - `StandardGraphs.Chromatogram`
  - 上記 2 つに対して interactive UI の描画ロジックを整理
- `src\renderer\MilkdownEditor.tsx`
  - Crepe 初期化時に custom code block feature を追加するよう変更
- `src\renderer\integralSnippetMenu.ts`
  - 以前の preview ベース設定を削除し、slash menu による JSON 挿入だけを担う形に整理
- `src\main\main.ts`
  - `integral:executeAction` IPC handler を追加
  - block type / action id ごとの mock executor を追加
  - 実行ログや summary を返すようにした
- `src\main\preload.ts`
  - renderer から action 実行を呼べる API を公開
- `src\shared\workspace.ts`
  - action 実行 request / result 型を追加
- `src\renderer\styles.css`
  - interactive block、実行ボタン、結果表示、JSON editor 用スタイルを追加
- `src\renderer\integralJsonPreview.ts`
  - 旧 preview-only 実装のため削除

## 検証

- `npm run build` 成功

## 設計上の整理

- 今回の実装で「UIボタン -> preload -> main IPC」までは通った
- ただし main 側の executor はまだ mock であり、実際の装置操作や解析実行は未接続
- 将来は main process 側で block type / action id ごとの実 executor registry を持ち、そこから外部プログラムやジョブ実行基盤へ委譲するのが自然
