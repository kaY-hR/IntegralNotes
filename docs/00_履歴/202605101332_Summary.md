# 2026-05-10 13:32 保存 validation エラー表示修正

## 背景

Markdown validation により重複 block id / output id を含む note の保存は拒否できるようになったが、保存失敗時に画面上でエラー内容が見えなかった。

## 原因

`saveNote()` の catch では `setStatusMessage()` にエラー文字列を入れていたが、`statusMessage` は `workspace__layout` の `data-status` 属性として保持されるだけで、通常の可視 UI としては描画されていなかった。

## 変更

- `src/renderer/App.tsx`
  - `WorkspaceNoticeState` を追加した。
  - 保存失敗時に `workspaceNotice` へエラー内容を入れ、workspace 上部に dismiss 可能なエラーバナーを表示するようにした。
  - 保存開始・保存成功・本文編集時には既存のエラー notice を消すようにした。
- `src/renderer/styles.css`
  - `workspace-notice` / `workspace-notice--error` のスタイルを追加した。
  - 複数行の validation エラーを `pre-wrap` で表示するようにした。

## 確認

- `npm run build` 成功。
