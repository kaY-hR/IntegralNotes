- Activity Bar に `Search` を追加する相談を行った。
- 既存実装は `Activity Bar / Sidebar Host / Main Workspace` の 3 列構成で、Explorer と Plugins は built-in sidebar view として載っていることを確認した。
- そのため Search も同じく built-in sidebar view として追加するのが自然、という整理を行った。
- 期待する UI は VS Code の検索ビューに近いが、MVP は次の段階分けが妥当と判断した。
  - まずは query 入力と結果一覧を持つ検索専用 sidebar view を作る
  - `replace`、`include files`、`exclude files` は foldable advanced section として後付けできる形にする
  - replace across files は preview と誤更新防止が必要なので search-only より 1 段重い機能として扱う
- 実装方式としては、UI は既存 React/CSS で十分で、外部 UI ライブラリは必須ではないと判断した。
- 検索エンジン候補は次の 2 つに整理した。
  - Node/Electron 側で workspace を走査して自前検索する
  - `ripgrep` 系を使う
- 推奨は `ripgrep` 系で、理由は大規模 workspace でも速く、VS Code 的な include/exclude と相性がよいため。
- ただし packaged app では binary 同梱が必要なので、採るなら `@vscode/ripgrep` のような実績ある構成を参考にするのが妥当。
- 置換まで含めて VS Code 並みに寄せること自体は可能だが、MVP としては次の順で進めるのが現実的。
  1. Search view を built-in sidebar view として追加
  2. query 入力、debounce、結果一覧、match highlight を実装
  3. include/exclude を foldable advanced controls として追加
  4. replace UI と preview/apply を追加
