## 概要

- ユーザーから、FlexLayout のファイルエディタ部分が表示されず、`md` ファイルを開いても見えないという報告を受けた
- 直前のヘッダー削除時に editor 領域の親レイアウトが崩れ、FlexLayout の表示領域が潰れていた

## 実施した対応

- `src/renderer/styles.css`
  - `.workspace` を再度 `flex-direction: column` に戻した
  - `.workspace__layout` に `flex: 1`, `width: 100%`, `min-width: 0`, `min-height: 0` を追加し、FlexLayout が確実に表示領域を持つように修正した

## 検証

- `npm run build` 成功

## 補足

- `md` を開く処理そのものではなく、表示コンテナ側の CSS 崩れが原因だった
