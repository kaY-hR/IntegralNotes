# 2026-04-09 11:45 Summary

## 概要

- 起動フリーズは解消したが、`itg-notes` が通常の code block として表示される不具合が残っていたため修正した

## 原因

- `src\renderer\integralCodeBlockFeature.tsx` で、Milkdown 標準 code block node view を `nodeViewCtx` から探す際のキーが誤っていた
- `codeBlockSchema.id` を使っていたが、node view 登録で使われるのは `codeBlockSchema.node.id`
- そのため標準 node view を見つけられず、前回追加した fallback で通常 editor 側へ落ちていた

## 実施した対応

- `src\renderer\integralCodeBlockFeature.tsx`
  - `nodeViewCtx` 検索条件を `codeBlockSchema.node.id` に修正

## 検証

- `npm run build` 成功

## 補足

- これで `itg-notes` は custom node view、その他の code block は Milkdown 標準 view という意図した分岐になる見込み
