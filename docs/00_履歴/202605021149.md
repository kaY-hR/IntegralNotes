# 2026-05-02 11:49 workspace template initialization

## 背景

- `.codex/skills` / `.claude/skills` / `scripts/integral` などを個別 bootstrap ではなく、app が持つ workspace template 一式として管理したい。
- `inline-action` は今回の対象外とし、cwd 初期化用 template 展開だけを実装した。

## 対応

- 要求・設計文書へ workspace template 仕様を追記した。
  - 開発時 source は source tree の `Notes/`
  - packaged app source は `process.resourcesPath/workspace-template/`
  - 空扱いの cwd では自動展開
  - 非空 cwd では自動展開しない
  - `初期化/更新` では確認 dialog 後に強制上書き展開
  - MVP では削除同期、merge、user edit 検知、conflict 生成、version 管理はしない
- `WorkspaceService` の個別 skill bootstrap を廃止し、workspace template 展開に置き換えた。
  - `.git`、`.` 始まり、Windows hidden 属性だけの cwd を空扱いする。
  - 展開先が workspace root 外へ出ないことを確認する。
  - symlink など通常 file / directory 以外は skip する。
- UI に `初期化/更新` button を追加した。
  - 実行前に Electron main process の dialog で上書き確認する。
  - 展開後は snapshot / managed metadata / asset catalog / 開いている tab を更新する。
- package 同梱設定を `Notes -> workspace-template` に変更した。
- Python SDK 同期元も `workspace-template/scripts/integral` を優先するようにした。
- `Notes/scripts/integral/__init__.py` を root `scripts/integral` と同じ params schema 対応へ揃えた。

## 確認

- `npm run build` 成功。
- compiled `WorkspaceService` を使い、一時 workspace で以下を確認した。
  - 空 workspace では `.codex` / `.claude` / `.vscode` / `scripts` が自動展開される。
  - visible file がある workspace では自動展開されない。
  - 明示 `applyWorkspaceTemplate()` では既存 file を残したまま template が展開される。
- ユーザー実行の `npm run dev` でも main watch compile が `Found 0 errors` まで到達した。
