# 2026-05-08 23:12 画像解析skillの.icv出力方針調整

## 背景

対象ノート `C:\Users\shimadzu\Desktop\_Root\00_Obsidian\Projects\01_Integral\動画A_白井先生テーマ模擬\Notes\20260508-スクリプト作成.md` と直近の `ai-chat-history.json` を確認した。

生成された `scripts/cell_detection.py` は `image-analysis-tips` の SDK と `.icv` テンプレートを読んでいたが、実際の出力は `overlay` PNG と `results_csv` のみだった。ログ上も `insertMarkdownAtCursor` の `out:` は `overlay` と `results_csv` のみで、`compare_view` は含まれていなかった。

## 判断

従来の skill 文面は `.icv` の存在とヘルパーを案内しているだけで、画像解析で overlay/mask/label/detection のような「元画像と位置合わせして比較する成果物」を作る場合に `.icv` を必ず出す、という判断基準が弱かった。

そのため、今回のような細胞検出では通常の PNG overlay を出せば十分と判断され、`.icv` が省略されたと考えられる。

## 変更

- `Notes/.codex/skills/image-analysis-tips/SKILL.md` に `Default Output Policy` を追加した。
- 位置合わせされた overlay/mask/label/detection/segmentation/before-after/UV-RGB などを作る場合、明示的に拒否されない限り `compare_view` `.icv` output を追加するようにした。
- PNG/HTML など通常の成果物は残し、`.icv` は追加 output とする方針を明記した。
- `.icv` の `auto_insert_to_work_note` は、埋め込み対応が確認できるまでは付けず、PNG/HTML 側を自動挿入する方針にした。
- `references/image-compare-manifest.md` に、overlay PNG と `compare_view` `.icv` を同時に宣言する Python block pattern を追加した。
- 対象ノート側の `Notes\.codex\skills\image-analysis-tips` コピーにも同じ更新を反映した。

## 補足

この変更は skill/document の修正のみで、アプリ本体の build は実行していない。
