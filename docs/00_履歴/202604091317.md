## 2026-04-09 13:17

- 事象
  - `itg-notes` node view の復活後、renderer で以下の例外が発生した。
    - `Unrecognized extension value in extension set`
    - `multiple instances of @codemirror/state are loaded`

- 原因
  - 標準 code block fallback 用に `CodeMirrorBlock` を `node_modules/@milkdown/components/src/...` から直接 import していた。
  - その結果、標準側が生成した CodeMirror extension と custom 側が参照する CodeMirror 実装が別系統になり、instance 判定が壊れていた。

- 対応
  - `src/renderer/integralCodeBlockFeature.tsx`
    - direct import した `CodeMirrorBlock` / `LanguageLoader` を廃止した。
    - `@milkdown/kit/component/code-block` の `codeBlockView.view` を使い、Milkdown 標準が生成した node view constructor をそのまま fallback に利用するように変更した。

- 効果
  - `itg-notes` は custom node view を維持する。
  - 非 `itg-notes` は標準 `code_block` node view を同一インスタンス系のまま利用する。
  - `@codemirror/state` の多重ロード由来の extension error を回避する。

- 確認
  - `npm run build` 成功。
