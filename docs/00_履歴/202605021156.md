# Inline Action 仕様整理と main merge

## 背景

- `??` / `>>` / `@@` の inline AI trigger を、共通の Inline Action 仕組みに寄せる方針を整理した。
- workspace-template 実装は別作業として main 側で進められたため、現在ブランチへ main を取り込む依頼があった。

## 決定事項

- `@@` は Inline Action picker を開く。
- `??` は新仕様では `@@continue` の alias とする。
- `>>` は `@@mkpy` の alias とする。
- 標準 Inline Action は `continue` / `write` / `mkpy` / `ask` とする。
- `@@continue` は prompt なしで即実行し、文脈から保守的に commit する。
- `@@write` は prompt 必須で、Markdown 挿入中心。workspace file 編集/作成は OFF。
- `@@mkpy` は prompt 必須で、workspace file 編集/作成と Python block draft 作成を許可する。
- `@@ask` は prompt 必須で、回答のみ。note は変更しない。
- commit tool は原則 `insertMarkdownAtCursor` に統一する。
- Python block 挿入は commit tool ではなく、`createPythonBlockDraft` helper tool で draft を返し、最終的に Markdown として挿入する方針。
- Inline Action 定義は `cwd/.inline-action/*.md` の frontmatter + Markdown body とする。
- `.inline-action` は workspace-template の一部として展開され、通常 file tree では非表示にする想定。
- skill / `$$` 連携は今回の MVP 対象外とした。

## 対応

- `docs/30_設計/40_InlineAction.md` を追加し、上記仕様を文書化した。
- `git merge main` を実行した。
  - 初回は linked worktree の git metadata 書き込み権限で失敗した。
  - safe.directory 登録後に再実行した結果、local `main` とは `Already up to date` だった。
