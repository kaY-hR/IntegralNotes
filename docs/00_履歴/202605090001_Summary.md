# 2026-05-09 00:01 .icv embed 表示と矢印キー focus 修正

## 背景

`.icv` viewer を plugin 化した後、note 埋め込み表示で stage が縦につぶれること、また矢印キーで 1 回切り替えた後に再クリックしないと次の矢印キーが効かないことが確認された。

## 原因

- embed iframe の幅が 920px 未満になると plugin 内の responsive layout が縦積みに切り替わり、下部操作パネルが高さを消費して stage が極端に低くなっていた。
- plugin 内の `render()` が DOM を差し替えるため、矢印キー操作後に focus されていた要素が消え、`#app` に付けた keydown handler が次の keydown を拾えない場合があった。

## 変更

- `plugins/image-compare-viewer/renderer/index.html`
  - `presentation: "embed"` を state に持たせ、embed 専用 class `viewer--embed` を付けるようにした。
  - embed 表示では 920px 未満でも原則として stage と panel を左右配置にし、狭い場合だけ縦積みにするよう CSS を追加した。
  - keydown handler を `#app` ではなく `window` に付け、iframe 内で focus target が body 等になっても矢印キーを拾えるようにした。
  - layer 切替・mode 切替・layer 選択・reset 後に `.viewer` へ focus を戻すようにした。

## 確認

- plugin renderer inline script の syntax check 成功。
- `npm run plugins:package:image-compare-viewer` 成功。
- `npm run plugins:install:all` で `IntegralNotes-dev` 側へ再 install 済み。

## 補足

今回は plugin HTML/CSS/JS の修正のみで、TypeScript 側の追加変更はない。
