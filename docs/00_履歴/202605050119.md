# 2026-05-05 01:19 global / pwd 解析スクリプトと skill

## 相談

- Python 解析スクリプトと AI skill を、workspace (`pwd`) と user/global の両方から使えるようにしたい。
- vendor plugin の wrapper script や user stock を考えると、manifest / registry まで作るのは MVP では重い。
- MVP では absolute path を `%LocalAppData%` などの token 表記で保存・表示し、workspace へ自動 copy せずに使う方針にした。

## 仕様化

- `docs/30_設計/32_GlobalPwd解析スクリプトとSkill.md` を追加した。
- `docs/30_設計/30_Pythonスクリプト登録と実行.md` に external/global script の扱いを追記した。

## 実装

- `src/main/pathTokens.ts` を追加し、`%LocalAppData%` / `%AppData%` / `%UserProfile%` の短縮・展開を共通化した。
- Python callable discovery で、workspace 内 `.py` に加えて次を走査するようにした。
  - `%LocalAppData%/IntegralNotes/analysis-stock`
  - `%LocalAppData%/IntegralNotes/plugins`
- `run:` の block-type として、workspace relative path と token 短縮済み absolute path の両方を扱えるようにした。
- catalog に無い manually written external `run:` でも、対象 `.py` が読める場合はその場で decorator を読んで実行できる fallback を追加した。
- file tree の `.py` context menu に「ユーザーストックに追加」を追加し、`%LocalAppData%/IntegralNotes/analysis-stock` へ copy するようにした。
- AI skill は次の優先順で project/global を merge して使うようにした。
  1. workspace `.codex/skills`
  2. workspace `Notes/.codex/skills`
  3. `%LocalAppData%/IntegralNotes/skills`
  4. `%UserProfile%/.codex/skills`
- AI Chat の skill 表示にも `project:` / `global:` の出所を出すようにした。
