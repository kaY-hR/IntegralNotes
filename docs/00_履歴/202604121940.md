## 背景

- 解析基盤の議論で、`run` を強いドメイン概念として持つ案が重いことが明確になった。
- 代わりに
  - `blob` は取り込み単位
  - `chunk` は解析契約単位
  - `block` は chunk だけを参照
  - `execution sandbox` は内部実装
  という整理へ寄せ直した。

## 主な判断

- ユーザーが持ち込む元データは `blob`
- `1 blob = 1 artifact.md` を基本とする
- 解析 block と runner は `chunk` だけを見る
- GUI で blob を選んだときは app が自動で source chunk を生成する
- block には `params / inputs / outputs` を持たせ、`inputs / outputs` は chunk ID のみ
- 再実行時は新しい output chunk を生成し、block の最新結果を置き換える
- 過去 chunk は immutable のまま残し、未参照になれば GC 対象とする
- `run` はユーザー向け履歴ではなく、execution sandbox の内部詳細に落とす
