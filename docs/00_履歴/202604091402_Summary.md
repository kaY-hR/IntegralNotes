# 2026-04-09 14:02 Summary

## 概要

- ユーザーから、独自 UI が Milkdown 標準 UI / preview と干渉しており、`LaTeX` 表示や通常 code block が崩れるとの報告を受けた
- 当初は `json` code block を拾っていたため、独自 UI の対象を ` ```itg-notes` に限定する方針へ変更した
- その後、`code_block` 全体を custom node view で包む実装が原因で、起動時ハング、`Math (LaTeX)` 選択時のフリーズ、標準 preview 消失が断続的に発生した
- 最終的に、`itg-notes` だけを扱う専用 node を新設する構成へ切り替えたことで、安定して動作する状態まで改善した

## 実施した対応

- `src/renderer/integralSnippetMenu.ts`
  - Integral 用スニペットの挿入形式を ` ```itg-notes` に統一した
- `src/renderer/integralBlockRegistry.tsx`
  - Integral 独自 block の language 判定を `itg-notes` 前提に整理した
- `src/renderer/integralCodeBlockFeature.tsx`
  - 途中段階では `code_block` wrapper node view による委譲方式、標準 view への fallback、trace 追加などを試した
  - `@codemirror/state` 多重読込を避けるため、標準 code block view の直接 import をやめ、Milkdown 標準経路を使う修正も行った
  - 最終的に `code_block` 全置換を廃止し、`integral_notes_block` 専用 schema / node view を実装した
  - Markdown の ` ```itg-notes` は専用 node に parse し、保存時は同じ ` ```itg-notes` へ serialize するようにした
  - 標準 `code_block` 側は parser 拡張で `itg-notes` だけ除外し、通常 code block と LaTeX は Milkdown 標準 UI へ戻した
- `src/renderer/MilkdownEditor.tsx`
  - 新しい専用 node 構成を editor 初期化時に登録するようにした
- `Notes/Experiments/2026-04-08.md`
  - サンプルの fence を `json` から `itg-notes` へ揃えた

## 調査と判断

- Milkdown 全体に広くログを仕込むのではなく、wrapper 側 lifecycle と `Math (LaTeX)` 周辺の限定 trace を入れる方針にした
- ただし、限定ログで停止地点は追えても、`code_block` 全体を包む設計自体が不安定要因と判断した
- そのため、根本対応として `itg-notes` 専用 node を切り出す方針へ移行した

## 発生した主な問題

- 起動直後に UI 全体が無反応になるハング
- `itg-notes` がただの通常 code block として表示される退行
- `Math (LaTeX)` 選択時のフリーズ
- `Unrecognized extension value in extension set` による CodeMirror runtime error

## 検証

- 各段階で `npm run build` を実行し、最終構成でも成功
- 最終的にユーザーから「安定して動いている」と確認を得た

- 本スレッド中の段階的な変更内容は個別履歴として `docs/00_履歴/202604091127.md` から `docs/00_履歴/202604091331.md` に残してある
- 最終的な安定化の決定打は、`itg-notes` を専用 node として分離し、Milkdown 標準の `code_block` / `LaTeX` 経路に干渉しない構成へ戻した点だった
