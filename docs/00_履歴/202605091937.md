# 2026-05-09 19:37 package / global extension root 実装

## 背景

直近の議論で、Python block、AI skill、runtime plugin、viewer plugin を package 単位で扱う方針を整理した。

合意した主な方針:

- 拡張資産 root は `%LocalAppData%/IntegralNotes` に寄せる。
- runtime plugin は `%LocalAppData%/IntegralNotes/runtime-plugins`。
- package は `%LocalAppData%/IntegralNotes/packages`。
- package 内 component は `integral-package.json` の `exports` に載ったものだけ公開する。
- package 由来 runtime plugin / skill は global 側から直接読む。
- package 由来 Python script は workspace `.packages/{packageId}/scripts/` へ copy してから使う。
- `.packages` は system-managed path として file tree / managed tracking から除外する。
- 旧 `%AppData%/IntegralNotes[-dev]/plugins`、`%LocalAppData%/IntegralNotes/{plugins,analysis-stock,skills}` は削除対象。

## 変更

- `docs/30_設計/55_Package管理.md` を追加し、package manifest、exports、global extension root、`.packages` import 方針を明文化した。
