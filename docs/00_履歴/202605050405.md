# 2026-05-05 04:05 provider built-in web search 接続

## 依頼

- OpenAI と Anthropic の provider built-in web search を AI Chat / inline AI agent から呼べるようにしたい。

## 確認

- 直近履歴 3 件は FlexLayout tab / main 取り込み関連で、AI provider 層とは未関連だった。
- `docs/10_要求` と `docs/30_設計/86_AIChatPanelとAgentTooling.md` を確認し、direct provider runtime が `AiAgentService` に集約されていることを前提にした。
- OpenAI は公式 docs と AI SDK docs / local package source から `openai.tools.webSearch()` を確認した。
- Anthropic は公式 docs と AI SDK docs / local package source から `anthropic.tools.webSearch_20250305()` を確認した。

## 対応

- `src/main/aiAgentService.ts` で direct OpenAI runtime に `web_search: openai.tools.webSearch(...)` を追加した。
- `src/main/aiAgentService.ts` で direct Anthropic runtime に `web_search: anthropic.tools.webSearch_20250305({ maxUses: 5 })` を追加した。
- provider built-in search は direct provider runtime のみで有効化し、AI Gateway runtime は触らない方針にした。
- agent instructions に、current public web information が必要な場合だけ `web_search` を使い、workspace/source-code 調査には原則使わない指示を追加した。
- transcript tool trace 用に `web_search` の input / output summary を追加した。

## 補足

- Anthropic 側は organization Console で web search が有効になっていないと provider 側で失敗する。
- OpenAI 側は Responses API の provider tool として使う。
- 今回は user 指示どおり実装のみ行い、build / test は未実行。
