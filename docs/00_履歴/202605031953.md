# 2026-05-03 19:53 Inline Action の itg-notes 直接 node 挿入修正

## 依頼

- 前回の refresh transaction 追加後、@@continue / @@mkpy で挿入した block が逆にずっと default block のままになった。

## 原因の見立て

- 挿入直後の refresh transaction が editor の serializer / autosave を早く走らせ、itg-notes が標準 code block として残った状態を固定していた可能性がある。
- insert(markdown) や通常 Markdown parser 任せでは、Inline Action の複合挿入時に itg-notes 専用 node 化が安定しない。

## 対応

- src/renderer/MilkdownEditor.tsx から refresh 用 equestAnimationFrame transaction を削除した。
- Inline Action 挿入時、Markdown 内の itg-notes fenced code block を検出し、ProseMirror の integral_notes_block node を直接作って挿入する helper を追加した。
- itg-notes 以外の Markdown 部分は通常 parser に任せ、AI transcript などとの混在は維持する。
