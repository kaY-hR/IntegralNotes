# 2026-05-09 20:53 履歴

## 背景

- `image-analysis-pack` に手動追加した `scripts/detect_cells.py` が `>` picker に出ない問題を確認した。
- installed package 側の `integral-package.json` では `exports.pythonBlocks` が `scripts/detect_cells.py` になっていた。
- 現行仕様では Python block export は callable 単位の `scripts/foo.py:function` が必須のため、この entry は manifest parse 時に除外されていた。
- `detect_cells.py` は `main` に `@integral_block` が付いていたため、正しい export は `scripts/detect_cells.py:main`。

## 対応

- installed package にあった `detect_cells.py` を repo source の `plugins/image-analysis-pack/scripts/detect_cells.py` に取り込んだ。
- `plugins/image-analysis-pack/integral-package.json` の `exports.pythonBlocks` を `["scripts/detect_cells.py:main"]` に更新した。
- package 由来 runtime plugin が Plugin Manager から通常 uninstall されようとして失敗する問題を修正した。
  - `InstalledPluginOrigin` に `package` を追加。
  - package runtime plugin を `origin: "package"` / `packageId` 付きで扱うようにした。
  - Plugin Manager では package 由来 plugin の uninstall button を disabled にし、package 単位管理であることを示す。
  - backend の `uninstallPlugin` でも package 由来 plugin は package 単位管理を促す error にした。
- `docs/30_設計/55_Package管理.md` に package 由来 runtime plugin は package install / uninstall 単位で管理することを追記した。

## 確認

- `npm run build` 成功。
- `npm run plugins:install:all` で `%LocalAppData%/IntegralNotes/packages/image-analysis-pack` を再インストール済み。
- installed manifest は `scripts/detect_cells.py:main` になった。
- built code で `listPackagePythonBlockExports()` を確認し、`.packages/image-analysis-pack/scripts/detect_cells.py:main` が返ることを確認。
- built code で `PluginRegistry.listInstalledPlugins()` を確認し、`image-compare-viewer` が `origin: package`, `packageId: image-analysis-pack` として返ることを確認。

## 注意

- dev app 起動中に package manifest を手動変更した場合、renderer 側の catalog 更新が必要。再起動か Refresh 相当の操作で反映される。
