# Relation Graph 実装サマリー

## 背景

2026-05-03 の `$interview` で、Relation Graph を link picker ranking の補助ではなく、今後のサジェスト機能全般の基盤として扱う方針を決めた。Graph DB は純粋なリンク情報の derived index とし、UI から関係を追加する場合も DB を直接編集せず、将来は Markdown link や source 側の記述へ反映する。

## 実装内容

- `src/shared/relationGraph.ts` を追加し、Relation Graph の node / edge / snapshot / neighborhood API 型を定義した。
- `src/main/relationGraphService.ts` を追加し、workspace 配下の `.store/.integral/relation-index.sqlite` に `nodes / edges / origins` を保持する SQLite index を実装した。
- SQLite 実装は `sql.js` を使用し、`sigma` / `graphology` とあわせて依存関係へ追加した。
- scan は hash / origin 単位で差分更新する構造にした。初回や空 index は全構築し、それ以後は `origins.origin_hash` と current hash を比較して、変更 origin の edge だけを置き換える。
- workspace mutation listener から Relation Graph index を更新し、delete / move 時は該当 path subtree の node / edge / origin を即削除する。
- `workspace:sync`, workspace folder open, template apply 時には Relation Graph の同期も行う。
- renderer preload / shared API に `getRelationGraphSnapshot()` と `getRelationNeighborhood()` を追加した。
- built-in workspace tool として `Relation Graph` を追加し、Activity Bar からタブで開けるようにした。
- Graph View は `sigma.js` + `graphology` で描画し、以下を実装した。
  - workspace 全体 graph の表示
  - edge type filter
  - node search
  - active tab node と 3-hop 近傍の強調
  - node click で対応する note / file / data-note / block / Python callable を開く

## Scan 対象と edge

- folder / file parent-child: `contains`
- Markdown link / image link: `markdown-link`
- `itg-notes` block:
  - note -> block: `contains`
  - block -> Python callable: `block-call`
  - block -> input data: `block-input`
  - block -> output data: `block-output`
- dataset -> member data: `dataset-member`
- Python file -> callable: `defines`

## datafile / datanote の扱い

- managed file / dataset が `noteTargetId` を持つ場合、`data-note:{noteTargetId}` を graph node として扱う。
- 同じ `noteTargetId` を共有する managed file / dataset は同一 node へ合流する。
- data-note path は node metadata に `dataNotePath` として保持し、data-note tab を開いた場合も同じ node を active として強調できるようにした。
- data-note node click は `.store/.integral/data-notes/{noteTargetId}.md` を開く。
