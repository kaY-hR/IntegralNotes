## 会話要約

- ユーザーから、file rename 時に `workspace:renameEntry` で `.md` 制約エラーが出てアプリが止まる、と報告があった。
- 原因は explorer が全 file を扱うようになった後も、`renameEntry()` が「file は note なので .md のみ」という旧前提を持っていたことだった。

## 実装内容

- `src/main/workspaceService.ts`
  - `normalizeEntryName()` に option を追加した。
  - `createEntry()` では従来どおり file 作成時のみ `.md` を default / required にした。
  - `renameEntry()` では既存 file の rename に拡張子制限を掛けないようにした。

## 検証

- `npm run build` が通ることを確認した。

## 補足

- 今回の修正で、explorer 上の既存 file は `.md` 以外でも rename できる。
- 新規作成については従来どおり「note 作成 = `.md`」のまま維持している。
