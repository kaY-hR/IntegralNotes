## 概要

- output slot から input slot の data-note 共有先を指定できる contract (`share_note_with_input`) を実装した。
- 共有先が設定された managed data は、自分専用の data-note を新規作成せず、共有先 note を開く挙動へ変更した。
- Notes 配下のデモ Python script / SDK も新 contract を使うよう更新した。

## 実装

- `src/shared/integral.ts`
  - `IntegralSlotDefinition` に `shareNoteWithInput` を追加。
  - managed data / dataset summary に `noteTargetId` を追加。
- `src/main/integralWorkspaceService.ts`
  - Python decorator parser で `share_note_with_input` を解釈するよう変更。
  - output 生成時に `noteTargetId` を metadata / dataset manifest へ保存。
  - data-note 解決時に `id` 固定ではなく `noteTargetId` を優先するよう変更。
- `src/main/workspaceService.ts`
  - `syncManagedDataNotesInternal` で `noteTargetId != id` の managed data は data-note 自動作成対象から除外。
- `src/main/dataNote.ts`
  - metadata 正規化と frontmatter に `noteTargetId` を反映。
- `src/renderer/App.tsx`
  - `openManagedDataNote` が summary の `noteTargetId` を使って shared note を開くよう変更。
- `Notes/scripts/integral/__init__.py`
  - SDK slot spec に `share_note_with_input` を追加。
- `Notes/src/lc_text_to_chromatogram_json.py`
  - `json` / `plot` output に `share_note_with_input: "source"` を追加。
- `Notes/src/chromatogram_pca.py`
  - `report` output に `share_note_with_input: "source"` を追加。
- `Notes/demo_dataset_report.py`
  - `report` output に `share_note_with_input: "source"` を追加。
