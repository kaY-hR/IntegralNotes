# 2026-05-08 16:25 .idts input SDK helper 反映

## 依頼

- side コマンドで整理した `.idts` input の SDK 化方針を実装へ反映する。
- skill 更新先は `main_ws/.codex/skills` ではなく、workspace template 側の `Notes` 配下 skill とする。

## 対応

- Python block 実行 payload の `.idts` input を readable directory path ではなく `.idts` manifest file path として渡すように変更した。
  - `src/main/integralWorkspaceService.ts` の `resolveInputExecutionPath()` で dataset は `metadata.path` の manifest path を渡す。
- Python SDK に dataset helper を追加した。
  - `resolve_dataset_files(path)`
  - `resolve_dataset_input(path)`
  - `prepare_dataset_output(path)`
  - `IntegralDatasetInput`
  - `IntegralDatasetMember`
- SDK helper は `.idts` manifest と `.store/.integral/{managedFileId}.json` を読んで member file を解決する。
  - `dataPath` がある derived dataset は visible output folder を読む。
  - `memberIds` だけの source dataset は必要に応じて `.store/.integral/materialized-datasets/{datasetId}` へ copy materialize する。
- root 側 `.integral-sdk/python/integral` と template 側 `Notes/.integral-sdk/python/integral` の両方を更新した。
- `scripts/demo_dataset_report.py` を新 SDK helper 使用へ更新した。
- AI prompt と `Notes/.codex` / `Notes/.claude` の `implement-integral-block` skill、および各 reference を新契約へ更新した。
- `docs/10_要求` と関連 architecture / design docs の Python 実行契約を `.idts manifest path + SDK helper` に更新した。
- 誤って触った `main_ws/.codex/skills/implement-integral-block/SKILL.md` は内容差分なしまで戻した。

## 確認

- `python -m py_compile Notes/.integral-sdk/python/integral/__init__.py scripts/demo_dataset_report.py` 成功。
- `resolve_dataset_files()` が実データセット `Data/細胞画像データセット.idts` から 390 files を返すことを確認。
- `scripts/demo_dataset_report.py` を `.idts` manifest input で直接実行し、`index.html` / `files.csv` を生成できることを確認。
- 検証で作成した `tmp_dataset_demo_test` と `__pycache__` は削除済み。
- `npm run build` 成功。

## 注意

- `resolveDatasetReadablePath()` は dataset 表示や inspect 用の app 内部 helper として残している。
- 既存の Python block script が `.idts` input を directory として直接扱っている場合は、新契約では SDK helper へ更新が必要。
