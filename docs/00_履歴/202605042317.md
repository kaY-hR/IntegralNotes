# 2026-05-04 23:17 FlexLayout duplicate editor-main 修正

## 事象

- `npm run dev` 中、tab close 後に Vite client で `duplicate id:editor-main` が出た。
- 表示上の動作には大きな問題は見えていなかった。

## 原因

- FlexLayout は最後の tab / tabset を閉じたあと、root が空になると内部で新しい tabset を再生成する。
- その再生成時に `setOnCreateTabSet` から固定 id `editor-main` を返していた。
- FlexLayout 内部では古い idMap が掃除される前に新しい `TabSetNode` が作られるため、同じ `editor-main` と衝突していた。

## 対応

- `src/renderer/App.tsx` の `setOnCreateTabSet` で、root 再生成時に固定 id を返さないようにした。
- empty tabset の自動削除設定 `enableDeleteWhenEmpty: true` は維持した。
- tab 追加先は既存の `model.getActiveTabset()` fallback で解決できるため、固定 id は不要と判断した。

## 確認

- `npm run build` 成功。
- `flexlayout-react` の model 操作だけで最後の tab を削除し、duplicate id が出ないことを確認した。
