## 会話要約

- 解析 input の割り当てや結果閲覧で `DTS-...` ID を選ぶ前提になっており、UX が悪いので、基本は dataset 名を前面に出したいという依頼があった。
- 特に `.idts` file はユーザーが見て選ぶため file 名も dataset 名ベースにしたい、解析結果の dataset manifest も hidden ではなく `Data/` 配下へ出したい、という方針が示された。
- derived dataset の system 既定名は `{解析名}_{slot名}_yyyyMMddHHmm` にしたい、という naming rule が指定された。

## 調査

- `src/main/integralWorkspaceService.ts` を確認したところ、
  - source dataset は既に `name` を持ち、visible manifest file 名も dataset 名ベースで作成していた
  - 一方 derived dataset は `DTS-...` を `name` に使い、hidden `.store/.integral/datasets/{datasetId}.idts` へ保存していた
  - block source の `out:` にも hidden `.idts` path を書き戻していた
- ただし内部では既に `.idts path -> datasetId -> readable directory path` の解決層があり、manifest path を `Data/` に変えても内部契約は崩れないと判断した。

## 仕様更新

- `docs/10_要求`, `docs/20_アーキテクチャ`, `docs/30_設計` の関連文書を更新した。
- 主な更新点:
  - user-facing な dataset 表示は `name` を主表示し、ID は通常見せない
  - source / derived とも visible manifest の既定保存先は `Data/*.idts`
  - derived dataset の system 既定名は `{解析名}_{slot名}_yyyyMMddHHmm`
  - block source の `out:` には visible `.idts` path を書き戻す

## 実装

- `src/main/integralWorkspaceService.ts`
  - source dataset manifest の既定保存先を `Data/` に統一した
  - derived dataset 作成時に block title と slot 名から system 既定名を生成するようにした
  - derived dataset manifest を `Data/{dataset-name}.idts` へ保存するようにした
  - internal canonical ID と `.store/objects/{datasetId}` は維持した

- `src/renderer/DatasetManifestFileViewer.tsx`
  - summary chip、member 行、note section などから dataset/original data ID の直接表示を外した
