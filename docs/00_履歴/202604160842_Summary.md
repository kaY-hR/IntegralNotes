## 会話要約

- 後から専用 viewer を追加できる viewer plugin system をどう設計するかを検討した。
- viewer の判定方法として、特殊な Markdown、特殊な JSON、拡張子ベースを比較した。
- 当面は拡張子ベースだけでよく、plugin system 自体は分けずに既存 plugin system へ contribution を追加する方針に整理した。

## 合意した方針

- plugin system は 1 つに統合する。
- ただし contribution type は分け、`blocks` と `viewers` を同一 manifest に持てるようにする。
- viewer の first version は extension-only matcher とし、`extensions: [".foo", ".bar"]` で解決する。
- 既存 plugin への影響は additive に抑え、`viewers` を持たない plugin はそのまま動かす。
- viewer 解決順は、`plugin viewer -> built-in viewer -> unsupported / 外部アプリ` を基本とする。
- `.md` の note editor は既存の note tab を維持し、viewer plugin の初版対象には含めない。
