## 会話要約

- managed data を `id / path / hash` 基盤で扱い、`source / derived` や `visible / hidden` を追跡方式とは分離したい、という方針で整理を進めた。
- あわせて、`data-note` は visible な `data-catalog/` file として扱うのをやめ、ID に 1:1 で紐づく system-managed note として `.store/.integral/data-notes/{id}.md` に置きたい、という要望が入った。
- `data-note` は rename / move させず、ユーザーは本文だけを編集する。「実体へのリンク集」ではなく、「この data は何か」を説明する note として扱う方針にした。
