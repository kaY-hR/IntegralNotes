# 2026-05-03 19:40 Inline Action 挿入後の py block 描画修正

## 依頼

- @@continue や @@mkpy で Python block を挿入した直後、Python block として認識されず default の ?block として描画される。
- Sync を押すと再描画されて正常な Python block 表示になる。

## 原因の見立て

- itg-notes block は Markdown 再読込時には専用 node として parse される。
- 一方、Inline Action の挿入処理は parserCtx(markdown) と eplaceRange で独自に slice を挿入していた。
- そのため、挿入直後の editor state では itg-notes 専用 node 化が > 経由の block 挿入と揃わず、Sync による再parseで初めて正しい描画になっていた可能性が高い。

## 対応

- src/renderer/MilkdownEditor.tsx の Inline Action 挿入処理を、手組みの parserCtx + Slice + replaceRange から Milkdown の既存 insert(markdown) 経路へ変更した。
- > で Python block を挿入する既存処理と同じ Markdown 挿入経路に揃えた。
- 不要になった parserCtx と Slice import を削除した。
