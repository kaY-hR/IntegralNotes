## 背景

- `.chunk` だけを例外表示する方針は不自然なのでやめ、workspace レイアウト自体を
  - `blob` -> `.blob`
  - `.chunk` -> `chunk`
  へ整理し直すことになった。
- 既存 workspace には旧レイアウトの実データがある前提なので、見た目だけではなく migration まで含めて合わせる必要があった。

## 実装したこと

### workspace レイアウトの rename

- `src/main/workspaceService.ts`
  - 初期化時の作成先を `Notes / Artifacts / .blob / chunk / .py-scripts` に変更
  - `blob -> .blob`, `.chunk -> chunk` を自動で移行する migration を追加
  - hidden 例外としての `.chunk` 表示ロジックは削除し、`chunk` を通常フォルダとして見せる形へ戻した

### 追随 migration

- `src/main/workspaceService.ts`
  - source chunk の `links.json` を新レイアウトの `../../.blob/.../payload` に再計算して書き直す処理を追加
  - blob artifact note も新配置に合わせて再生成し、
    - ファイル名
    - 見出し
    - payload 相対パス
    を揃えるようにした

### 実行系の参照先更新

- `src/main/integralWorkspaceService.ts`
  - blob/chunk の実ディレクトリ定数を `.blob` / `chunk` に更新
  - chunk 作成、inspect、artifact summary などが新レイアウトを使うように調整

### 文書更新

- `docs/30_設計/20_ワークスペースレイアウト.md`
  - ルート構成、source chunk の `links.json` 例、blob/chunk 配置を新レイアウトに更新
- `docs/30_設計/30_Pythonスクリプト登録と実行.md`
- `docs/20_アーキテクチャ/30_Python汎用解析プラグイン.md`
  - `.chunk` の実行パス例を `chunk` に更新
