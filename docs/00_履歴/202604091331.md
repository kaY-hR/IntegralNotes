## 背景

- `code_block` 全体を custom node view で包む実装が Milkdown / Crepe の標準 UI と干渉し、LaTeX 選択時のフリーズや preview 崩れが発生していた。
- 方針を `itg-notes` 専用 node へ切り替えることにした。

## 実施内容

- `src/renderer/integralCodeBlockFeature.tsx`
  - `code_block` wrapper node view を廃止。
  - `integral_notes_block` の専用 schema を追加。
  - Markdown の ```` ```itg-notes ```` を専用 node に parse し、保存時は再び ```` ```itg-notes ```` へ serialize するようにした。
  - `codeBlockSchema.extendSchema(...)` で標準 `code_block` の Markdown parser から `itg-notes` を除外した。
  - `itg-notes` 専用 node view は従来の Integral preview / 実行 UI を引き続き持つ。
  - trace は `window.__integralMilkdownTrace__` と `console.debug` に継続出力。

- `src/renderer/MilkdownEditor.tsx`
  - 既存の `installIntegralCodeBlockFeature(editor)` 呼び出しをそのまま利用し、新しい schema / view 登録経路で動作するようにした。
