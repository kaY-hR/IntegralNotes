# 2026-04-09 11:42 Summary

## 概要

- 起動直後に UI 全体が反応しなくなる不具合に対処した
- 原因候補として、Milkdown の custom code block feature が標準 node view の初期化順に依存していた点を潰した

## 実施した対応

- `src\renderer\integralCodeBlockFeature.tsx`
  - `codeBlockView.view` への直接依存をやめた
  - `@milkdown/core` の `nodeViewCtx` から既存の code block node view を取得し、それを fallback として包む方式に変更した
- `src\renderer\MilkdownEditor.tsx`
  - Milkdown editor の生成を `try/catch` 化した
  - Integral custom feature 付き editor の生成に失敗した場合、通常 editor で再生成する fallback を追加した

## 検証

- ユーザー環境で起動直後フリーズが解消したことを確認
- `npm run build` 成功

## 補足

- まだ根本原因が custom node view 側の別要因に残っている可能性はある
- ただし今回の変更で、少なくとも起動不能に落ちる経路はかなり潰せている
