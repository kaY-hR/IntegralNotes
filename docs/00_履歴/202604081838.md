# 2026-04-08 18:38 作業ログ

## 依頼

- 最初に編集するときに下記 warning が出る
  - `Warning: Cannot update during an existing state transition (such as within render).`

## 原因

- `src/renderer/App.tsx` で `setOpenTabs(...)` の functional updater の中から `syncTabLabel(...)` を経由して `model.doAction(FlexLayout.Actions.renameTab(...))` を呼んでいた
- そのため、React の state transition 中に FlexLayout 側の更新が重なり、`render` 中更新扱いの warning が発生していた
- 同様に、`closeTabsMatching(...)` でも `setOpenTabs(...)` の流れと FlexLayout の `deleteTab` が密結合していた

## 対応

- `src/renderer/App.tsx`
  - `openTabsRef` を追加して最新のタブ状態を state updater 外でも参照できるようにした
  - `updateTabContent(...)` では React state の更新だけを `setOpenTabs(...)` に閉じ込め、`syncTabLabel(...)` はその外で実行するように変更
  - `closeTabsMatching(...)` でも削除対象の判定と `model.doAction(...)` を state updater の外へ出し、`setOpenTabs(...)` 自体は純粋更新だけに変更

- `npm run build` 成功
