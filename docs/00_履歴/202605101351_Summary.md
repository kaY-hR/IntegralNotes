# 2026-05-10 13:51 `??` auto-continue の最終挿入 validation 修正

## 背景

`??` による auto-continue で、重複 block id / output id を含む Markdown が挿入され、保存時 validation で保存できない状態になる問題が残っていた。

## 原因

`??` は `@@auto-continue` に解決され、基本的には Inline Action の共通経路に乗っていた。

ただし main process の `insertMarkdownAtCursor` validation は `beforeText` / `afterText` / insertion text を連結して候補 Markdown を作っていた。この `beforeText` / `afterText` は ProseMirror の `textBetween()` 由来であり、Markdown source ではないため、fenced code block の ````itg-notes` 構造が失われる。

その結果、保存時 validator では検出できる `itg-notes` block が、AI commit tool 側の早期 validation では見えないケースがあった。

## 変更

- `src/renderer/MilkdownEditor.tsx`
  - AI Markdown 挿入の最終反映直前に、実際に dispatch する ProseMirror transaction 後の document を Milkdown serializer で Markdown に戻すようにした。
  - serializer で得た実際の挿入後 Markdown を `validateMarkdownDocument()` に通し、NG の場合は dispatch せず popup error として返すようにした。
- `docs/10_要求/10_MVP要件.md`
  - renderer final commit validation の要件を追記した。
- `docs/30_設計/40_InlineAction.md`
  - main process の tool validation は早期 feedback、renderer の transaction 後 validation は最終防衛線であることを追記した。
