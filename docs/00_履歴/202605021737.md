# 2026-05-02 17:37 inline 挿入フリーズ調査ログ追加

## 依頼

- `@@` などの inline 挿入時、たまにフリーズし、放置すると画面が真っ白になる。
- DevTools console は見られない。
- 真っ白画面を落とす際に、main process 側で `Render frame was disposed before WebFrameMain could be accessed` が出た。
- 途中で「原因特定前に対策だけ入れてよいか」という確認があり、原因未確定としてログを先に残す方針にした。

## 対応

- main process に app log を追加し、renderer からの診断イベントを `app:rendererLog` IPC で受けるようにした。
- renderer 全体の `window.error` / `unhandledrejection` をログへ送るようにした。
- `MilkdownEditor` の inline AI 経路へ、次の段階ログを追加した。
  - action 開始
  - trigger 削除
  - submit 開始 / 結果 / error
  - stream batch
  - preview 開始 / 完了 / cancel
  - Markdown parse
  - ProseMirror dispatch
  - 挿入完了
- ログには本文や生成内容は出さず、長さ・件数・所要時間・stream ID・action 名などの診断情報だけを残す。
- main process 側で `render-process-gone` / `unresponsive` / `responsive` をログに残すようにした。
- renderer が破棄済みの状態で close 確認や AI stream event を送ろうとした場合、例外にせず `ipc-send-skipped` としてログへ残すようにした。
- inline popup の描画負荷を下げるため、stream 表示を 80ms 単位で batch 反映し、表示上限を設けた。
- 挿入前 preview も長大 Markdown では 12,000 chars まで表示し、実際の挿入内容は full text のまま扱うようにした。

## ログ出力先

- `app.getPath("userData")/logs/integral-notes-yyyyMMdd.jsonl`
- Windows packaged / dev 環境では Electron の userData 配下に出る。

## 確認

- `npm run build` 成功。
- `git diff --check` は CRLF 変換 warning のみで whitespace error なし。

## 備考

- 現時点で根本原因は未確定。
- 次に再発した場合は、ログの最後の `MilkdownEditor.inlineAi` event と main の `render-process-gone` / `renderer-unresponsive` を見れば、stream / preview / parse / dispatch / renderer crash のどこで止まったかを切り分けられる。
