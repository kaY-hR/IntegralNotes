# 2026-04-17 15:15

- 外部 drag/drop は直ったが `Ctrl+V` で落ちる件を修正した。
- 原因は、`src/main/preload.ts` で `electron.clipboard` を直接使っていたことだった。
  - 実行環境では preload 側の `clipboard` が `undefined` となり、`writeBuffer` / `readImage` 呼び出しで例外になっていた。
- 対応:
  - `src/main/main.ts` に clipboard 用 IPC を追加した。
    - `workspace:writeWorkspaceSelectionToClipboard`
    - `workspace:readWorkspaceSelectionFromClipboard`
    - `workspace:writeClipboardText`
    - `workspace:clipboardHasImage`
  - `src/main/preload.ts` は clipboard 直接参照をやめ、上記 IPC の thin bridge に変更した。
  - `src/renderer/App.tsx` は `readWorkspaceSelectionFromClipboard()` と `clipboardHasImage()` を `await` するように修正した。
- これにより、Explorer の
  - app 内 copy/paste
  - 外部 clipboard file/folder paste
  - clipboard image paste
  の判定が main process の clipboard 実装を使う形に揃った。
- `npm run build` を実行し、通過を確認した。
