# 2026-04-21 14:48 AI Chat rendered document inspection tool

## 背景

- ユーザーから、`md` / `html` を読むときに埋め込み HTML チャートなどをグラフィカルに LLM に送りたい、という要望あり。
- 既存実装では:
  - image file path は `readWorkspaceImage` で能動的に見られる
  - しかし markdown / html document 全体を「描画後の見た目」として送る tool は無かった

## 実装

- `src/main/workspaceVisualRenderService.ts` を追加。
  - `WorkspaceService.readWorkspaceFile()` で `markdown` / `html` / `text` document を読む
  - hidden `BrowserWindow` を作成して document を描画
  - `capturePage()` で PNG screenshot を取得
  - AI tool に返せるよう base64 化
- markdown は `micromark + micromark-extension-gfm` を dynamic import で使って HTML 化。
  - `allowDangerousHtml: true` にして raw HTML を保持
  - そのため markdown 内に埋めた HTML chart / iframe / script も screenshot 対象にできる
- workspace-root 形式の `/Data/...` などの URL は file URL に rewrite するようにした。
- `src/main/aiAgentService.ts`
  - `renderWorkspaceDocument` tool を追加
  - tool instructions に
    - `markdown/html layout or embedded charts matter`
    - `renderWorkspaceDocument`
    を明記
  - tool trace summary にも `renderWorkspaceDocument` を追加
- `src/main/main.ts`
  - `WorkspaceVisualRenderService` を生成して `AiAgentService` に注入
- `src/main/aiChatService.ts`
  - runtime note に `renderWorkspaceDocument` の存在を反映

## 使い方の想定

- agent は `find` / `rg` で `report.md` や `chart.html` を発見
- 見た目が重要なら `renderWorkspaceDocument(path)` を呼ぶ
- tool result が image-data として model に返る
- assistant は screenshot を見た上で回答できる

## 制約

- 対応 kind は現時点で `markdown` / `html` / `text`
- 外部 CDN に依存する script / asset は環境によって描画失敗の可能性あり
- 実機での Electron 手動 smoke test は未実施。`npm run build` は成功
