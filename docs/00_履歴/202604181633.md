## 概要

- `.md` managed file は追跡対象のまま残しつつ、data-note の生成対象から除外した
- backend に `hasDataNote` を追加し、renderer 側のノート導線もそれに従うように揃えた
- 要求・アーキテクチャ・設計文書の data-note 記述を `.md` 除外前提へ更新した

## 実装

- `WorkspaceService` の data-note 同期で、`representation = file` かつ `*.md` の managed file は write plan を作らないようにした
- 既存の `.md` 向け data-note は、同期時に active plan から外れるため自動削除される
- `IntegralManagedFileSummary` / `IntegralDatasetSummary` に `hasDataNote` を追加した
- renderer の file viewer / Markdown editor / asset picker / block UI / embed fallback で `hasDataNote === false` の場合は「ノート」導線を出さないようにした
- `openManagedDataNote` は note 非対応 target に対しては open せず、status message を出すようにした
