## 背景

- 独立開発用のローカル配置として `plugin-sdk` と `plugins/*` を切る案が出た。
- その構成が妥当か、特に `plugins/Graphs.Chromatogram` と `plugins/LC.Method` の粒度・命名が適切かを整理する必要があった。

## 実施内容

- `docs/20_アーキテクチャ/pluginシステム.md`
  - ローカル開発ワークスペース案を追記。
  - `plugin-sdk` と `plugins/*` を分ける方向性自体は妥当と明記。
  - `plugins/<name>` は block type 名ではなく plugin package / plugin id 単位にする方がよいと整理。
  - 例として以下を推奨。
    - `plugins/shimadzu-lc`
    - `plugins/standard-graphs`
  - 「1 plugin が複数 block type を持つ」関係を mermaid で可視化。

## 判断

- `plugin-sdk` はそのままでよい。
- `plugins/LC.Method` はまだ許容範囲だが、plugin id としては少し block type 寄り。
- `plugins/Graphs.Chromatogram` は 1 block 専用に見えやすく、将来拡張の観点ではやや粒度が細かい。
- plugin package 名と block type 名は分けて考えるべき。
