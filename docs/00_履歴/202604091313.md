- 調査
  - Milkdown / Crepe の `code_block` node view は `CodeMirrorBlock` に集約されていた。
  - `nodeViewCtx` は node type 単位で 1 つの view に収束するため、`code_block` を差し替える場合は custom view 側で標準 `CodeMirrorBlock` に委譲する必要があると確認した。
  - `LaTeX` preview は `codeBlockConfig.renderPreview` をラップする構成のため、前回の `renderPreview` 上書き方式は相性が悪いと判断した。

- 対応
  - `src/renderer/integralCodeBlockFeature.tsx`
    - `code_block` 用 wrapper node view を実装した。
    - `itg-notes` のときだけ interactive な custom React view を生成し、それ以外は Milkdown 標準 `CodeMirrorBlock` に委譲する。
    - language 変更時に `integral` / `standard` を切り替えて node view を差し替える。
    - `create/update/destroy/switch/execute` の trace を `console.debug` と `window.__integralMilkdownTrace__` に積むようにした。
  - `src/renderer/integralSnippetMenu.ts`
    - `CodeMirror.renderPreview` の上書きを削除した。
  - `src/renderer/MilkdownEditor.tsx`
    - editor 初期化時に custom code block node view を登録するようにした。
