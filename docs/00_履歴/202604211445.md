# 2026-04-21 14:45 AI Chat proactive image inspection tool

## 背景

- ユーザーから「ファイルパスから、LLM が能動的に画像を見られるようにしたい」と要望あり。
- 既存実装では、user prompt に書かれた画像パスを submit 前に attachment 化する処理はあったが、agent が `find` / `rg` などで画像パスを発見した後に、自分でその画像を開く tool は無かった。

## 実装

- `src/main/aiAgentService.ts`
  - `readWorkspaceImage` tool を追加。
  - 入力は workspace-relative path。
  - real workspace から画像ファイルを読み、base64 化して model tool result の `image-data` として返す。
  - 対応 media type:
    - avif / bmp / gif / heic / heif / jpeg / jpg / png / svg / webp
  - 8MB 超の画像は拒否。
  - agent instructions / bash instructions にも `image path を見つけて画像検査が必要なら readWorkspaceImage を使う` と追記。
  - tool trace summary に `readWorkspaceImage` の入力 / 出力要約を追加。

## 関連補足

- 直前に入れた binary placeholder snapshot と組み合わせることで、Windows fallback でも `find` で画像を発見し、その path を `readWorkspaceImage` に渡して中身を視覚入力としてモデルへ戻せるようになった。
