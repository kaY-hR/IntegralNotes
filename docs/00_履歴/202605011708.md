# 2026-05-01 17:08 worktree local dev port config

## 背景

- Playwright MCP だけでなく通常の `npm run dev` でも renderer port `5173` を別 worktree 間で reuse し、main process と renderer bundle が別 worktree 由来になる混乱が起き得る。
- Codex MCP 設定は global なので、worktree ごとに port / userData / artifact dir を分離する必要がある。

## 対応

- worktree root の `.integralnotes-dev.local.json` を dev runtime の local config として導入した。
  - `.gitignore` に追加し、Git 管理対象外にした。
  - `devPort` を通常 `npm run dev` と MCP の共通 renderer port として使う。
  - `playwrightUserDataDir` / `playwrightArtifactDir` を MCP の既定値として使う。
- `src/dev/dev-local-config.cjs` を追加した。
  - local config 読み込み、既定値解決、port 自動採番、config 作成を担当する。
- `src/dev/ensure-local-dev-config.cjs` を追加した。
  - register bat から呼び、local config が無ければ作成する。
- `src/dev/dev-supervisor.cjs` / `src/dev/start-renderer.cjs` を local config の `devPort` 使用に変更した。
  - Vite 起動時は `--host 127.0.0.1 --port <devPort> --strictPort` を渡す。
- `npm run dev` / `npm run dev:renderer` は起動前に `src/dev/ensure-local-dev-config.cjs` を実行するようにした。
- `src/dev/dev-supervisor.cjs` の Electron 起動条件を調整した。
  - `tsc --watch` の出力を pipe し、`Found 0 errors. Watching for file changes.` を確認するまで Electron を起動しない。
  - `dist-electron/main` の出力が 700ms 静かになるまで起動を待つ。
  - これにより、古い `dist-electron/main/main.js` で一度起動した直後、watch 初回 compile の出力で Electron が再起動して window が一瞬消える挙動を抑制する。
- `src/dev/integral-playwright-mcp.mjs` を local config の port / userData / artifact dir 使用に変更した。
- `scripts/register-codex-playwright-mcp.bat` で登録前に local config を準備するようにした。
- `docs/30_設計/95_PlaywrightMCP.md` に worktree local config の説明を追記した。
