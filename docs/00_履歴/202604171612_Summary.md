# 2026-04-17 16:12

- Explorer の `Ctrl+V` が無反応に見える件を追加で調査した。
- 原因候補として、`npm run dev` 時だけ残していた Electron のネイティブメニューが `Ctrl+V` を先に処理し、renderer 側の `keydown` に到達していない可能性が高いと判断した。
- 対応:
  - `src/main/main.ts`
    - `mainWindow.removeMenu()` を常時適用するように変更した。
    - 開発時に困らないよう、`F12` / `Ctrl+Shift+I` で DevTools を開閉するショートカットを `before-input-event` で追加した。
    - `readClipboardExternalPaths()` に `console.info` / `console.error` を追加し、`FileDropList` の取得結果を main 側でも追えるようにした。
  - `src/renderer/App.tsx`
    - Explorer の `ignored shortcut` / `external clipboard probe` / external drop のログを `console.debug` から `console.info` に変更した。
    - これで DevTools の既定設定でもログが見える。
- 期待される効果:
  - 開発時でも `Ctrl+V` が renderer 側の Explorer 貼り付け処理に到達する。
  - まだ貼り付けできない場合でも、renderer/main のどちらで止まっているかログで切り分けできる。
- `npm run build` を実行し、通過を確認した。
