# Relation Graph active node 解決の修正

## 問題

- `test.md` を開いているとき、Relation Graph で `test.md` の note node ではなく、その中の block node が active として扱われていた。

## 原因

- block node は `path: origin.relativePath` を持っているため、`test.md` 内の block node も `path = test.md` になる。
- snapshot の nodes は `kind, label, id` 順で返るため、`block` が `note` より先に並ぶ。
- renderer の `findNodeIdByPath` が `path` 一致の先頭 node を採用していたため、block node が active になっていた。

## 実装

- `src/renderer/RelationGraphView.tsx`
  - active path 解決時に `note -> data-note -> file -> folder -> python-callable -> block` の優先順位で exact path match を選ぶように変更。

## 確認

- `npm run build` 成功。
