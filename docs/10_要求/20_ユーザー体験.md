# ユーザー体験

## 1. 元データを登録する

1. ユーザーはファイルまたはフォルダを選ぶ
2. app は blob を作る
3. app は対応する artifact note を作る
4. ユーザーはノートからその artifact を参照できる

## 2. Python 処理 block を新規に置く

1. ユーザーは「Python 処理 block を挿入」を選ぶ
2. entry の `.py` を選ぶ
3. 同梱ファイルを選ぶ
4. input slot 名を決める
5. output slot 名を決める
6. displayName と description を必要なら入れる
7. app は `PYS-...` を作る
8. app は block をノートへ挿入する

## 3. 既存 Python script から block を置く

1. ユーザーは `.py-scripts/` 登録済み一覧を開く
2. `displayName` と `description` を見て script を選ぶ
3. 各 input slot に対して chunk または blob 群を指定する
4. app は必要なら source chunk を作る
5. block をノートへ挿入する

## 4. input 指定の UX

各 input slot には次の 2 通りの指定方法がある。

- 既存 chunk を 1 つ選ぶ
- blob を複数選ぶ

blob を複数選んだ場合:

- app は新しい source chunk を作る
- block には source chunk の `CNK-...` を書く

## 5. block を実行する

1. app は output slot ごとに新しい output chunk を作る
2. app は `analysis-args.json` を作る
3. plugin が実行される
4. 成功したら block の `outputs` が `CNK-...` に更新される

再実行時:

- output chunk は新しい ID になる
- 古い chunk はそのまま残る
- block の `outputs` は最新結果へ置き換わる

## 6. 結果を見る

結果表示は block を分ける。

- 処理 block
  - `chunk -> chunk`
- 表示 block
  - 入力 chunk を読み、renderable ファイルを表示する

標準表示 block では、chunk 内の

- html
- image
- text

を自動判定して表示する。

## 7. 装置 plugin を使う

装置 plugin でも流れは同じ。

1. block を置く
2. input slot に chunk を割り当てる
3. 必要なら custom UI で設定する
4. 実行する
5. output chunk を得る
6. 別 block で結果を見る

## 8. LLM 利用時の体験

LLM は block JSON を直接書ける。  
そのため block schema は人間と LLM の両方にとって単純である必要がある。

MVP の方針:

- LLM に覚えてもらうのは `plugin`, `block-type`, `inputs`, `outputs` のルールだけ
- Python 固有の複雑な schema は持ち込まない
- 詳細説明は `script.json.description` を見せればよい
