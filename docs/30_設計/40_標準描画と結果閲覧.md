# 標準描画と結果閲覧

## 位置づけ

MVP では、renderable なファイルの閲覧は app の標準機能とする。  
可視化 plugin は「renderable でない chunk を renderable chunk に変換する処理 plugin」として扱う。

## 対象拡張子

### HTML

- `.html`

### Image

- `.png`
- `.jpg`
- `.jpeg`
- `.svg`

### Text

- `.txt`
- `.md`
- `.json`
- `.csv`

## 表示 block

標準表示も共通 block schema で表す。

```json
{
  "id": "BLK-7A8B9C0D",
  "plugin": "core-display",
  "block-type": "chunk-view",
  "params": {},
  "inputs": {
    "source": "CNK-9X4Q2M1A"
  },
  "outputs": {}
}
```

## 表示ルール

- input slot `source` に指定された chunk を読む
- chunk 内ファイルを走査する
- 対象拡張子だけを拾う
- すべて flex-layout で表示する

### 並び順

MVP では固定順でよい。  
特に意味を持たせない。

## 対象ファイルがない場合

- 「表示可能なファイルがありません」を出す
- chunk 内に何があるかの一覧は別途見られるようにしてよい

## 可視化 plugin の考え方

可視化 plugin は次の責務を持つ。

- input chunk を読む
- output chunk に `html / image / text` を書く

つまり、可視化 plugin も本質的には処理 plugin である。  
標準表示 block と組み合わせて結果を見る。

## 装置 plugin の結果閲覧

装置 plugin も同じ整理にする。

- 装置 plugin が output chunk を作る
- output chunk に renderable な結果があれば標準表示できる
- renderable でなければ別の可視化 plugin を使う
