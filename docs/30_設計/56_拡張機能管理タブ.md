# 拡張機能管理タブ

## 目的

runtime plugin、AI skill、Python script、package を 1 つの画面で管理できるようにする。

この画面では、単に install 済み一覧を出すだけではなく、次の仕様が自然に分かる UI にする。

- Python script は workspace へ import してから実行する
- skill は workspace へ import せず、workspace / global / package stock から読む
- runtime plugin は workspace へ import しない
- package は skill / script / runtime / shared をまとめる配布単位であり、操作単位も package とする

## 表示場所

拡張機能管理は left sidebar view ではなく、main workspace の tab として開く。

Activity Bar には built-in workspace-tool item として `Extensions` を置いてよい。選択すると main workspace に `Extensions` tab を開く。

理由:

- package / script / skill / runtime を同時に見せるため、sidebar では横幅が足りない
- `Current Workspace` と `Global` を横並びで比較する画面にしたい
- install / import / delete / repair などの確認付き操作が多く、作業画面として扱う方が自然

## 基本レイアウト

行は component 種別、列は location を表す。

```text
Extensions                                      [Refresh] [Install Package...]

--------------------------------------------------------------------------------

                                     [Open] [Delete] [Stock on Global]

  image-analysis-tips                -                                 available
                                                                       [Open] [Delete]

Scripts
  scripts/local_analysis.py          present                           -
    main()
                                     [Open] [Delete] [Stock on Global]

  normalize_csv.py                   -                                 stock
    main()
                                                                       [Open] [Import] [Delete]

Standalone Runtime
  shimadzu-lc                        -                                 installed
                                                                       [Uninstall]

Packages
  image-analysis-pack                imported                          installed
                                     [Reimport] [Remove Import]        [Uninstall Package]

    skills                           -                                 image-analysis-tips
    scripts                          detect_cells.py                   detect_cells.py
    runtime                          -                                 image-compare-viewer
    shared                           integral_image_compare.py         integral_image_compare.py

  legacy-pack                        imported                          missing
                                     [Remove Import]                   [Install/Repair...]
```

ルール:

- top-level の `Skills` / `Scripts` / `Standalone Runtime` は package に属さない standalone component だけを表示する
- package に属する component は `Packages` 行の child として read-only 表示する
- package child row には操作 button を出さない
- package の import / reimport / remove / uninstall は package 行にだけ出す
- `Current Workspace` へ import できるのは standalone global script と package だけ
- workspace に追加できない component には import button を出さない

## location

### Current Workspace

現在開いている workspace (`pwd`) に存在するもの。

```text
Workspace/
  .codex/skills/
  Notes/.codex/skills/
  scripts/
  .packages/
    <packageId>/
      integral-package.json
      scripts/
      shared/
```

扱い:

- workspace skill は既に存在するものだけを表示する
- global skill を workspace へ追加する UI は持たない
- workspace script は workspace 内の standalone `.py` file を表示する
- `.packages` 配下の script は standalone script には混ぜず、package child として表示する
- workspace package import の有無は `.packages/<packageId>/integral-package.json` の存在で判定する

### Global

IntegralNotes が管理する user-wide な拡張資産。

```text
%LocalAppData%/IntegralNotes/
  global/
    skills/
    scripts/
  runtime-plugins/
  packages/
```

扱い:

- `global/skills` は standalone global skill
- `global/scripts` は workspace へ copy して使う standalone script stock
- `runtime-plugins` は package に属さない standalone runtime plugin
- `packages` は package stock

## 表示単位

### skill

standalone skill は skill directory 単位で表示する。

workspace 側:

- `workspace/.codex/skills/<skillId>`
- `workspace/Notes/.codex/skills/<skillId>`

global 側:

- `%LocalAppData%/IntegralNotes/global/skills/<skillId>`

package skill は package manifest の `exports.skills` に載ったものだけを、package child として表示する。

### script

standalone script は `.py` file 単位で表示する。

script file 内に discovery 可能な callable がある場合は、child detail として `main()` のように表示してよい。

global standalone script を import すると、workspace `scripts/` 配下へ copy する。

```text
%LocalAppData%/IntegralNotes/global/scripts/normalize_csv.py
-> Workspace/scripts/normalize_csv.py
```

global scripts 配下に subdirectory がある場合は、`scripts/` 以下に relative path を保って copy する。

```text
%LocalAppData%/IntegralNotes/global/scripts/image/detect_cells.py
-> Workspace/scripts/image/detect_cells.py
```

copy 先に同名 file がある場合は確認を出し、許可された場合だけ上書きする。

script が複数 file の helper に依存する場合、standalone script stock ではなく package 化する。

### standalone runtime

standalone runtime plugin は global にだけ存在する。

workspace へ import する UI は持たない。

### package

package は package ID 単位で 1 行表示する。

Current Workspace:

- `.packages/<packageId>/integral-package.json` があれば `imported`
- 無ければ `not imported`

Global:

- `%LocalAppData%/IntegralNotes/packages/<packageId>/integral-package.json` があれば `installed`
- 無ければ `missing`

package child には manifest の `exports` と、workspace copy の有無を説明として表示する。

```text
skills   global package stock の exported skill
scripts  workspace copy と global package stock の exported Python block
runtime  global package stock の exported runtime plugin
shared   workspace copy と global package stock の shared helper
```

package import の copy 対象は `docs/30_設計/55_Package管理.md` に従う。

- `integral-package.json`
- `scripts/`
- `shared/`

copy しないもの:

- `skills/`
- `runtime-plugins/`

## 操作

### Refresh

workspace と global を再走査し、表示を更新する。

### Install Package...

`integral-package.json` を含む package folder または package zip を選び、`%LocalAppData%/IntegralNotes/packages/<packageId>` に install する。

同じ package ID が既に installed の場合は確認を出し、許可された場合だけ上書きする。

### Open

対象 file / folder を app 内 editor または Explorer で開く。

### Delete

対象の standalone skill / script を削除する。

必ず確認を出す。workspace script の削除は user-created file の削除になり得るため、対象 path を確認 UI に明示する。

package child には `Delete` を出さない。package import の削除は package 行の `Remove Import`、global package の削除は package 行の `Uninstall Package` で行う。

### Stock on Global

workspace standalone skill / script を global stock へ copy する。

移動ではなく copy とする。workspace 側の file / skill は残す。

copy 先:

```text
workspace skill -> %LocalAppData%/IntegralNotes/global/skills/<skillId>
workspace script -> %LocalAppData%/IntegralNotes/global/scripts/<relativePath>
```

copy 先が既に存在する場合は確認を出し、許可された場合だけ上書きする。

### Import

global standalone script を workspace へ copy する。

copy 先:

```text
Workspace/scripts/<relativePath>
```

skill / runtime plugin には import button を出さない。

package には standalone script の `Import` ではなく、package 行の `Import` / `Reimport` を出す。

### Reimport

global package stock から workspace `.packages/<packageId>` へ再 copy する。

copy 先に既存 file がある場合は確認を出し、許可された場合だけ上書きする。

### Remove Import

workspace `.packages/<packageId>` を削除する。

必ず確認を出す。削除後、既存 note の `run: .packages/<packageId>/...` は解決できなくなる可能性がある。

### Uninstall Package

global package stock `%LocalAppData%/IntegralNotes/packages/<packageId>` を削除する。

必ず確認を出す。workspace import 済み package がある場合、Python block 実行は workspace copy だけで成立し得るが、package skill / runtime plugin / reimport は使えなくなる。

### Install/Repair...

workspace import は存在するが global package stock が missing の場合に表示する。

package folder または zip を選択して global package stock を install する。

## 状態表示

### package imported + installed

通常状態。

### package imported + missing

別 PC から workspace を持ってきた場合などに起こる。

- workspace `.packages/<packageId>` は存在する
- global package stock は存在しない

UI は warning として表示し、`Install/Repair...` を出す。

### package not imported + installed

global package stock だけが存在する状態。

package 行に `Import` を出す。

### standalone script imported

global script stock から workspace へ copy された script は、copy 後は通常の workspace script と同じ扱いにする。

standalone script には個別 manifest を持たせないため、import 元の global script との永続的な link は管理しない。

## 非目標

MVP では次はやらない。

- skill を global から workspace へ import する UI
- runtime plugin を workspace へ import する UI
- package child component の個別 import / delete
- standalone script の依存 helper 自動推定
- package dependency resolver
- global stock と workspace copy の自動同期
