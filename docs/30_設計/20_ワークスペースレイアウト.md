# ワークスペースレイアウト

## 全体

```text
Workspace/
  <user-managed notes, scripts, folders>
  Data/
  .store/
```

## ノート配置

- 通常ノートはユーザーが任意のフォルダに配置する
- 初回ワークスペース設定時に `Notes/` は自動作成しない
- `Notes/` という名前のフォルダを手動で使うこと自体は妨げないが、system 管理ディレクトリではない
- Python script も通常の workspace file として任意フォルダへ置ける

## data-note

data-note は managed data に紐づく system-managed Markdown とする。

例:

```text
.store/
  .integral/
    data-notes/
      ORD-7K2M9Q4D.md
      DTS-9X4Q2M1A.md
```

方針:

- original data の data-note は自動作成
- dataset の data-note も source / derived を問わず自動作成
- file 名は target ID 固定とし、ユーザーに管理させない
- rename / move は system 管理とし、ユーザーにはさせない
- app 上では本文だけを編集・表示する
- owner である managed data を管理対象から外したときは、対応する metadata JSON と data-note も同時に削除する

## Data

user-facing path の default と、note 本文へ添付した画像の既定保存先に使ってよい。

例:

```text
Data/
  sample01.lcd
  samples.idts
  20260414-1730-123.png
```

方針:

- `Data/` は special directory ではなく default location である
- original data も dataset-json も、workspace 内の任意 path を current path にしてよい
- note image attachment は ordinary workspace file であり、managed data metadata は自動作成しない
- 起動時または Sync 時には `cwd` を scan し、未管理の非 Markdown file を original data として自動登録してよい
- ただし `.` または `_` で始まる directory 配下、`.` で始まる file、`.store` などの system-managed directory 配下は自動登録対象から除外する

## .store

```text
.store/
  .integral/
    ORD-7K2M9Q4D.json
    DTS-9X4Q2M1A.json
    datasets/
      DTS-9X4Q2M1A.idts
    data-notes/
      ORD-7K2M9Q4D.md
      DTS-9X4Q2M1A.md
    runtime/
      BLK-1F8E2D0A/
        analysis-args.json
        stdout.log
        stderr.log
  objects/
    DTS-9X4Q2M1A/
      result.html
      table.csv
```

補足:

- original data の実体を `.store/objects/` に複製する構成は採らない
- `.store/objects/` は derived dataset の実データ配置に使う

### original data metadata 例

```json
{
  "id": "ORD-7K2M9Q4D",
  "entityType": "original-data",
  "representation": "file",
  "displayName": "sample01.lcd",
  "createdAt": "2026-04-13T10:00:00+09:00",
  "path": "Data/sample01.lcd",
  "hash": "sha256:8A91...",
  "visibility": "visible",
  "provenance": "source"
}
```

### source dataset

```text
datasets/
  samples.idts
```

`samples.idts` 例:

```json
{
  "datasetId": "DTS-1A2B3C4D",
  "name": "samples",
  "kind": "source-bundle",
  "memberIds": ["ORD-7K2M9Q4D", "ORD-9X4Q2M1A"]
}
```

ルール:

- source dataset の current path は visible `.idts` を default にしてよい
- source / derived を問わず、dataset の entrypoint は `.idts` で統一する
- source dataset が参照する original data を管理対象から外した場合は、対応する `.idts` manifest の `memberIds` からもその ID を除去し、dataset metadata を再同期する

### derived dataset

```text
.store/
  .integral/
    datasets/
      DTS-9X4Q2M1A.idts
  objects/
    DTS-9X4Q2M1A/
      result.html
      preview.png
```

`DTS-9X4Q2M1A.idts` 例:

```json
{
  "datasetId": "DTS-9X4Q2M1A",
  "name": "DTS-9X4Q2M1A",
  "kind": "scripts/pca.py:main.score",
  "dataPath": ".store/objects/DTS-9X4Q2M1A"
}
```

`DTS-9X4Q2M1A.json` 例:

```json
{
  "id": "DTS-9X4Q2M1A",
  "entityType": "dataset",
  "representation": "dataset-json",
  "displayName": "DTS-9X4Q2M1A",
  "kind": "scripts/pca.py:main.score",
  "path": ".store/.integral/datasets/DTS-9X4Q2M1A.idts",
  "hash": "sha256:93BC...",
  "visibility": "hidden",
  "provenance": "derived",
  "createdByBlockId": "BLK-1F8E2D0A",
  "createdAt": "2026-04-13T10:10:00+09:00"
}
```

## runtime metadata

```text
.store/
  .integral/
    runtime/
      BLK-1F8E2D0A/
        analysis-args.json
        stdout.log
        stderr.log
```

### ルール

- 実行補助 file や log は `.store/.integral/runtime/` に置く
- Python source file 自体は workspace 内の user-managed path をそのまま使う
- 実行のたびに runtime 配下を上書きまたは入れ替えてよい
- derived dataset の visible/hidden 差は `.idts` manifest の置き場所だけに寄せる

## GC 対象

MVP では、次を GC 対象にできる。

- どこからも参照されない dataset
- 不要になった runtime log

一方、次は GC 対象にしない。

- original data
- user-managed な `.py` file
- data-note

補足:

- data-note は単独 GC しないが、owner である managed data を明示的に管理対象から外したときは一緒に削除してよい

## 補足

- `.integral` を無視すれば `.store` は普通の filesystem として読める
- app からは `.store` 自体を非表示にしてよい
