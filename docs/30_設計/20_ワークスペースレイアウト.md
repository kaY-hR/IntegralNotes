# ワークスペースレイアウト

## 全体

```text
Workspace/
  <user-managed notes and folders>
  Data/
  .store/
  .py-scripts/
```

## ノート配置

- 通常ノートはユーザーが任意のフォルダに配置する
- 初回ワークスペース設定時に `Notes/` は自動作成しない
- `Notes/` という名前のフォルダを手動で使うこと自体は妨げないが、system 管理ディレクトリではない

## data-note

data-note は managed data に紐づく system-managed Markdown とする。

例:

```text
.store/
  .integral/
    data-notes/
      ORD-7K2M9Q4D.md
      DTS-9X4Q2M1A.md
```

方針:

- original data の data-note は自動作成
- dataset の data-note も source / derived を問わず自動作成
- file 名は target ID 固定とし、ユーザーに管理させない
- rename / move は system 管理とし、ユーザーにはさせない
- app 上では本文だけを編集・表示する
- identity や target 情報は file 名と metadata 側で管理し、editable frontmatter に載せない
- data-note は canonical data への link 集ではなく、「この data は何か」を記す説明 note として扱う

例:

```md
# sample01.lcd

LC 条件 A の生測定データ。
2026-04-14 の 1 本目。
```

dataset data-note 例:

```md
# samples

PCA 前処理後の入力束。
欠損行を除去済み。
```

## Data

visible path の default と、note 本文へ添付した画像の既定保存先に使ってよい。

例:

```text
Data/
  sample01.lcd
  samples.idts
  20260414-1730-123.png
```

方針:

- `Data/` は special directory ではなく default location である
- original data も dataset-json も、workspace 内の任意 path を current path にしてよい
- note image attachment:
  - Milkdown から挿入した画像は既定で `Data/yyyyMMdd-HHmm-RRR.ext` に保存する
  - これは ordinary workspace file であり、managed data metadata は自動作成しない
  - 必要なら後から明示的に original data として登録できる
- note image attachment:
  - `Data/` 以外の workspace path を手書きで参照してもよい

## .store

```text
.store/
  .integral/
    ORD-7K2M9Q4D.json
    DTS-9X4Q2M1A.json
    data-notes/
      ORD-7K2M9Q4D.md
      DTS-9X4Q2M1A.md
  objects/
    ORD-1A2B3C4D.csv
    DTS-9X4Q2M1A/
      result.html
      table.csv
```

### original data metadata 例

```json
{
  "id": "ORD-7K2M9Q4D",
  "entityType": "original-data",
  "representation": "file",
  "displayName": "sample01.lcd",
  "createdAt": "2026-04-13T10:00:00+09:00",
  "path": "Data/sample01.lcd",
  "hash": "sha256:8A91...",
  "visibility": "visible",
  "provenance": "source"
}
```

### hidden original data 例

```text
.store/
  objects/
    ORD-7K2M9Q4D.csv
    ORD-1A2B3C4D/
      run-001/
        a.csv
        b.csv
```

### source dataset

```text
datasets/
  samples.idts
```

`samples.idts` 例:

```json
{
  "datasetId": "DTS-1A2B3C4D",
  "name": "samples",
  "kind": "source-bundle",
  "memberIds": ["ORD-7K2M9Q4D", "ORD-9X4Q2M1A"]
}
```

ルール:

- source dataset は `dataset-json` を既定にしてよい
- current path は visible / hidden のどちらでもよい
- 実行時には app が `memberIds` から directory path を resolve する

### derived dataset

```text
.store/
  objects/
    DTS-9X4Q2M1A/
      result.html
      preview.png
```

`DTS-9X4Q2M1A.json` 例:

```json
{
  "id": "DTS-9X4Q2M1A",
  "entityType": "dataset",
  "representation": "directory",
  "displayName": "DTS-9X4Q2M1A",
  "kind": "PYS-7K2M9Q4D.result",
  "path": ".store/objects/DTS-9X4Q2M1A",
  "hash": "tree:93BC...",
  "visibility": "hidden",
  "provenance": "derived",
  "createdByBlockId": "BLK-1F8E2D0A",
  "createdAt": "2026-04-13T10:10:00+09:00"
}
```

## .py-scripts

```text
.py-scripts/
  PYS-7K2M9Q4D/
    script.json
    pca.py
    helper.py
    config.json
    analysis-args.json
    stdout.log
    stderr.log
```

### ルール

- 実行場所は `PYS-...` フォルダ自身
- entry ファイル名は元のまま保持する
- 同梱ファイルはフラット配置する
- `analysis-args.json`, `stdout.log`, `stderr.log` は上書きされてよい

## GC 対象

MVP では、次を GC 対象にできる。

- どこからも参照されない dataset

一方、次は GC 対象にしない。

- original data
- `.py-scripts/PYS-...`
- data-note

## 補足

- `.integral` を無視すれば `.store` は普通の filesystem として読める
- app からは `.store` 自体を非表示にしてよい
