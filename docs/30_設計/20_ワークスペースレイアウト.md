# ワークスペースレイアウト

## 全体

```text
Workspace/
  <user-managed notes and folders>
  data-catalog/
  Data/
  .store/
  .py-scripts/
```

## ノート配置

- 通常ノートはユーザーが任意のフォルダに配置する
- 初回ワークスペース設定時に `Notes/` は自動作成しない
- `Notes/` という名前のフォルダを手動で使うこと自体は妨げないが、system 管理ディレクトリではない

## data-catalog

ユーザーがノートから参照する Markdown facade。

例:

```text
data-catalog/
  sample01.lcd_BLB-7K2M9Q4D.md
```

方針:

- original data の data note は自動作成
- dataset の data note は必要時のみ作成
- metadata は frontmatter に保持する
- app 上では frontmatter を直接見せず、本文だけを編集・表示する
- metadata 同期時は本文を保持し、差分があるときだけ file を更新する

例:

```md
---
integralNoteType: "data-note"
originalName: "sample01.lcd"
originalDataId: "BLB-7K2M9Q4D"
sourceKind: "file"
createdAt: "2026-04-14T04:17:00.000Z"
aliasRelativePath: "Data/sample01.lcd"
storeRelativePath: ".store/BLB-7K2M9Q4D.lcd"
---
# sample01.lcd_BLB-7K2M9Q4D
```

## Data

`cwd` 外から能動登録した original data の visible alias を置く。

例:

```text
Data/
  sample01.lcd
  raw-run-001/
```

方針:

- file は hard link
- directory は junction
- 同名衝突時は `_{originalDataId}` を suffix として付ける
- `cwd` 内に最初からあった path を登録した場合は、`Data/` ではなく元の path 自体を alias とする

## .store

```text
.store/
  .integral/
    BLB-7K2M9Q4D.json
    CNK-9X4Q2M1A.json
  BLB-7K2M9Q4D.csv
  BLB-1A2B3C4D/
  CNK-9X4Q2M1A/
    result.html
    table.csv
```

### original data metadata 例

```json
{
  "aliasRelativePath": "Data/sample01.lcd",
  "originalDataId": "BLB-7K2M9Q4D",
  "originalName": "sample01.lcd",
  "createdAt": "2026-04-13T10:00:00+09:00",
  "sourceKind": "file",
  "storeRelativePath": ".store/BLB-7K2M9Q4D.lcd"
}
```

### original data 実体

```text
.store/
  BLB-7K2M9Q4D.csv
  BLB-1A2B3C4D/
    run-001/
      a.csv
      b.csv
```

### source dataset

```text
.store/
  .integral/
    CNK-1A2B3C4D.json
  CNK-1A2B3C4D/
    joined_analytical_data.csv
    Current.html
    run-001/
```

`CNK-1A2B3C4D.json` 例:

```json
{
  "datasetId": "CNK-1A2B3C4D",
  "kind": "source-bundle",
  "createdByBlockId": null,
  "createdAt": "2026-04-13T10:05:00+09:00",
  "sourceMembers": [
    {
      "originalDataId": "BLB-7K2M9Q4D",
      "entryName": "joined_analytical_data.csv"
    },
    {
      "originalDataId": "BLB-9X4Q2M1A",
      "entryName": "Current.html"
    }
  ]
}
```

ルール:

- file original data は hard link で置く
- directory original data は junction で置く
- source dataset 直下には custom manifest を置かない
- name collision がある場合は `_{originalDataId}` を suffix として付けて解消する

### derived dataset

```text
.store/
  .integral/
    CNK-9X4Q2M1A.json
  CNK-9X4Q2M1A/
    result.html
    preview.png
```

`CNK-9X4Q2M1A.json` 例:

```json
{
  "datasetId": "CNK-9X4Q2M1A",
  "kind": "PYS-7K2M9Q4D.result",
  "createdByBlockId": "BLK-1F8E2D0A",
  "createdAt": "2026-04-13T10:10:00+09:00"
}
```

## .py-scripts

```text
.py-scripts/
  PYS-7K2M9Q4D/
    script.json
    pca.py
    helper.py
    config.json
    analysis-args.json
    stdout.log
    stderr.log
```

### ルール

- 実行場所は `PYS-...` フォルダ自身
- entry ファイル名は元のまま保持する
- 同梱ファイルはフラット配置する
- `analysis-args.json`, `stdout.log`, `stderr.log` は上書きされてよい

## GC 対象

MVP では、次を GC 対象にできる。

- どこからも参照されない dataset

一方、次は GC 対象にしない。

- original data
- `.py-scripts/PYS-...`
- data note

## 補足

- `.integral` を無視すれば `.store` は普通の filesystem として読める
- app からは `.store` 自体を非表示にしてよい
